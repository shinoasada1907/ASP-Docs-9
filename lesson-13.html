CHƯƠNG 13

MVC — Componentization & Interactivity

Tái sử dụng giao diện bằng Partial View và View Component, kết hợp JavaScript để tạo tương tác động mà không cần tải lại toàn bộ trang.

1 Lý thuyết Componentization

Bài 1 · Componentization theory

Componentization (tách giao diện thành các thành phần) là kỹ thuật chia nhỏ một trang web lớn thành những mảnh độc lập, mỗi mảnh đảm nhận một trách nhiệm riêng. Mặc dù tài liệu liệt kê nhiều lợi ích (maintainability, scalability, reusability…), hai lợi ích quan trọng nhất trong thực tế là:

Lợi íchÝ nghĩa
Separation of Concerns Mỗi thành phần chỉ làm một việc. Khi cần sửa phần navbar, bạn chỉ mở file navbar, không phải lùng sục trong một file layout khổng lồ.
Reusability Một component được viết một lần, dùng lại ở nhiều nơi. Thay đổi một chỗ, toàn ứng dụng đều được cập nhật.

Vấn đề khi không component hoá

Hình dung một trang web có navbar, sidebar, phần nội dung chính và footer. Nếu nhét tất cả vào một file layout, file đó sẽ: cồng kềnh, khó đọc, khó bảo trì và không thể tái sử dụng từng phần.

🎯Nguyên tắc cốt lõi: mỗi component nên tự đủ nghĩa (self-contained) — đọc tên file là biết ngay nó làm gì, không cần đọc toàn bộ code.

Hai cơ chế của ASP.NET Core MVC

ASP.NET Core MVC cung cấp hai cơ chế để component hoá giao diện:

  • Partial View — một file .cshtml được nhúng vào trong một view khác. Dữ liệu đến từ view cha (parent view) truyền xuống.
  • View Component — một lớp C# độc lập kết hợp với view riêng, tự xử lý logic và lấy dữ liệu của mình, không phụ thuộc vào view cha.

2 Component hoá bằng Partial View

Bài 2 · Componentize with Partial View

Một Partial View là một file .cshtml thông thường, nhưng nó được nhúng bên trong một Razor view khác chứ không phản hồi trực tiếp HTTP request. Tên file partial thường bắt đầu bằng dấu gạch dưới (_TênFile.cshtml) để phân biệt với full view.

Ví dụ thực tế: tách Navbar ra khỏi Layout

Thay vì viết toàn bộ HTML navbar trong _Layout.cshtml, ta tạo riêng _Navbar.cshtml trong thư mục Views/Shared/:

Views/Shared/_Navbar.cshtmlrazor
<nav class="navbar navbar-expand-lg bg-body-tertiary">
    <div class="container-fluid">
        <a class="navbar-brand" href="#">Departments</a>
        <div class="collapse navbar-collapse" id="navbarSupportedContent">
            <ul class="navbar-nav me-auto mb-2 mb-lg-0">
                <li class="nav-item">
                    <a class="nav-link" href="/">Home</a>
                </li>
                <li class="nav-item">
                    <a class="nav-link" href="/departments">Departments</a>
                </li>
            </ul>
        </div>
    </div>
</nav>

Trong _Layout.cshtml, thay khối HTML navbar bằng một dòng nhúng partial:

Views/Shared/_Layout.cshtmlrazor
<html>
  <head>
    <title>@ViewBag.Title</title>
    <!-- Bootstrap CSS -->
  </head>
  <body>
    @await Html.PartialAsync("_Navbar")

    <div class="container">
        @RenderSection("title_section", false)
        @RenderBody()
    </div>
    <!-- Bootstrap JS -->
  </body>
</html>

ℹ️Html.PartialAsync("_Navbar") tìm file _Navbar.cshtml theo thứ tự: thư mục view hiện tại → Views/Shared/. Luôn dùng await để tránh deadlock.

Khi nào đặt partial ở thư mục nào?

  • Views/Shared/ — partial dùng chung toàn ứng dụng (navbar, footer, searchbar…).
  • Views/[Controller]/ — partial chỉ dùng trong một controller nhất định.

3 Truyền dữ liệu vào Partial View

Bài 3 · Pass data to partial view

Một partial view có thể có model riêng giống như full view. View cha truyền model xuống thông qua tham số thứ hai của Html.PartialAsync. Đây là cơ chế để tránh trùng lặp code.

Bài toán: form Create và Edit dùng chung

Trong ví dụ quản lý phòng ban (Departments CRUD), form tạo mới và form chỉnh sửa gần như giống hệt nhau. Ta tạo partial _DepartmentDetail.cshtml để dùng ở cả hai nơi:

<form method="post" action="/departments/@ViewBag.Action">

    {

    <div class="row mb-3">
        <div class="col-2">
            <label class="col-form-label">Name</label>
        </div>
        <div class="col-6">
            <input type="text" name="Name" value="@Model.Name" class="form-control" />
        </div>
    </div>

    <div class="row mb-3">
        <div class="col-2">
            <label class="col-form-label">Description</label>
        </div>
        <div class="col-6">
            <input type="text" name="Description" value="@Model.Description" class="form-control" />

4 Bài tập 9 — Component hoá danh sách

Bài 4 · Assignment 9 — Componentize a list

Đây là bài tập thực hành. Trước khi xem lời giải ở bài tiếp theo, hãy tự phân tích trang Departments/Index và trả lời các câu hỏi sau:

  1. Xác định các loại chức năng khác nhau có mặt trong trang danh sách phòng ban. Gợi ý: nhìn vào phần hiển thị danh sách và từng hàng trong bảng.
  2. Đặt tên component cho từng chức năng theo nguyên tắc Single Responsibility (mỗi component chỉ làm một việc).
  3. Tạo các Partial View tương ứng và cập nhật Index.cshtml để nhúng chúng vào.
  4. Kiểm tra trang vẫn hiển thị đúng sau khi refactor.

ℹ️Bài tập này thực hành nested partial views — partial view gọi lại một partial view khác bên trong. Đây là pattern phổ biến trong dự án thực tế.

6 Kết hợp Partial View & JavaScript (Lý thuyết)

Bài 6 · Combine Partial View and Javascript for Interactivity (Theory)

Mô hình request-response truyền thống của MVC có một hạn chế: mỗi lần người dùng tương tác (click tìm kiếm, lọc dữ liệu…), trình duyệt phải gửi request và server phải trả về toàn bộ trang HTML mới. Điều này gây trải nghiệm giật, chậm và không cần thiết.

Giải pháp: Fetch API + Partial View

Ý tưởng: thay vì tải lại cả trang, JavaScript dùng Fetch API gọi một endpoint riêng, endpoint đó trả về chỉ fragment HTML (từ một Partial View), rồi JS chèn fragment đó vào đúng vị trí trên trang:

  1. Người dùng nhập từ khóa & nhấn Search.
  2. JavaScript gọi fetch("/department-list/<filter>") — không tải lại trang.
  3. Server chạy action SearchDepartments, lọc dữ liệu, trả về PartialView("_DepartmentList", data).
  4. Response là HTML thuần (chỉ các hàng <tr>), không có layout hay header.
  5. JavaScript nhận HTML, gán vào innerHTML của <tbody>.

🎯Điểm mấu chốt: controller action gọi return PartialView(...) thay vì return View(...). Khi đó ASP.NET Core render partial mà không bọc layout bên ngoài, trả về fragment HTML gọn nhẹ.

7 Kết hợp Partial View & JavaScript (Triển khai)

Bài 7 · Combine Partial View and Javascript for Interactivity (Implementation)

Triển khai tính năng tìm kiếm phòng ban mà không tải lại trang, sử dụng Partial View + Fetch API.

Bước 1 — Tạo Partial View cho thanh tìm kiếm

Vì thanh tìm kiếm có thể dùng ở nhiều nơi, đặt nó trong Views/Shared/:

Views/Shared/_SearchBar.cshtmlrazor
<div class="row">
    <div class="col-6">
        <div class="input-group mt-1 mb-1">
            <input id="search-filter" type="text"
                   class="form-control" placeholder="Search" />
            <div class="input-group-append">
                <button id="search-button" class="btn btn-primary" type="button">
                    Search
                </button>
            </div>
        </div>
    </div>
</div>

Bước 2 — Cập nhật Index.cshtml

<tbody> có id="department-list" để JavaScript có thể nhắm mục tiêu. Ban đầu bảng rỗng, dữ liệu được nạp qua fetch:

Views/Departments/Index.cshtmlrazor
@{
    ViewBag.Title = "Departments";
}

@section title_section {
    <h3>Departments</h3>
}

@await Html.PartialAsync("_SearchBar")

<table class="table table-striped">
    <thead>
        <tr>
            <th>Name</th>
            <th>Description</th>
            <th></th>
        </tr>
    </thead>
    <tbody id="department-list">
        <!-- dữ liệu được điền bằng JavaScript -->
    </tbody>
</table>

<br />
<a class="btn btn-primary" href="/departments/create">Add</a>

<script>
    document.querySelector("#search-button").addEventListener("click", async () => {
        var filter = document.querySelector("#search-filter").value;

        var response = await fetch(`/department-list/${filter}`);
        var departmentsHTML = await response.text();

        document.querySelector("#department-list").innerHTML = departmentsHTML;
    });
</script>

Bước 3 — Thêm action SearchDepartments vào Controller

Controllers/DepartmentsController.cscsharp
[Route("/department-list/{filter?}")]
public IActionResult SearchDepartments(string? filter)
{
    var departments = DepartmentsRepository.GetDepartments(filter);
    return PartialView("_DepartmentList", departments);
}

⚠️Lưu ý: action này trả về PartialView, không phải View. Nếu trả về View, response sẽ bao gồm cả layout HTML đầy đủ, làm hỏng kết quả khi chèn vào innerHTML.

Luồng hoạt động hoàn chỉnh

  1. Trang Index load → <tbody> rỗng.
  2. Người dùng nhập "HR" vào ô tìm kiếm, nhấn Search.
  3. JS gọi fetch("/department-list/HR").
  4. Server trả về HTML các thẻ <tr> lọc theo "HR".
  5. JS gán HTML vào <tbody id="department-list">.
  6. Trang cập nhật tức thì — không reload.

8 View Component vs Partial View

Bài 8 · ViewComponent vs PartialView

Partial View giải quyết tốt vấn đề tách giao diện, nhưng nó có một hạn chế quan trọng: dữ liệu phải đến từ view cha. Partial View không tự lấy dữ liệu được.

Hạn chế của Partial View

Xem lại ví dụ _DepartmentDetail.cshtml: partial này nhận Department model từ Create.cshtml hoặc Details.cshtml. Nếu muốn dùng partial này ở một trang khác, action của trang đó phải lấy và truyền model Department xuống — dù trang đó không liên quan gì đến Department.

⚠️Vấn đề: khi một partial cần dữ liệu phức tạp (ví dụ: sidebar hiển thị thống kê, widget thông báo…), mọi action ở mọi trang đều phải chuẩn bị dữ liệu đó — làm nặng controller và phá vỡ separation of concerns.

Giải pháp: View Component

View Component là component tự chủ hoàn toàn:

Tiêu chíPartial ViewView Component
Nguồn dữ liệu Từ view cha truyền xuống Tự lấy trong lớp C# riêng
Logic nghiệp vụ Không có Có — trong phương thức Invoke()
Độ phụ thuộc Phụ thuộc vào view cha Hoàn toàn độc lập
Khi nào dùng Tái sử dụng UI đơn giản, dữ liệu đã có sẵn Widget phức tạp, cần tự lấy dữ liệu

9 Các thành phần cốt lõi của View Component

Bài 9 · Essential parts of View Component

Một View Component gồm hai phần bắt buộc:

  1. Lớp C# (ViewComponent class) — chứa logic nghiệp vụ, phương thức Invoke() trả về IViewComponentResult.
  2. File Razor view — nằm ở đường dẫn cố định, tên mặc định là Default.cshtml.

Cấu trúc thư mục chuẩn

Views/
  Shared/
    Components/
      DepartmentList/          ← tên component (không có hậu tố ViewComponent)
        Default.cshtml         ← view của component
        DepartmentListViewComponent.cs   ← class (có thể đặt ở thư mục gốc)

ℹ️Class DepartmentListViewComponent.cs có thể đặt ở bất kỳ đâu trong project (ví dụ: thư mục ViewComponents/). Vị trí trong ví dụ trên là để tiện quản lý cùng view tương ứng.

Lớp ViewComponent

Views/Shared/Components/DepartmentList/DepartmentListViewComponent.cscsharp
using Microsoft.AspNetCore.Mvc;
using WebApp.Models;

namespace WebApp.Views.Shared.Components.DepartmentList
{
    [ViewComponent]
    public class DepartmentListViewComponent : ViewComponent
    {
        public IViewComponentResult Invoke()
        {
            // Tự lấy dữ liệu — không phụ thuộc vào view cha
            var departments = DepartmentsRepository.GetDepartments();
            return View(departments);
        }
    }
}

Default.cshtml — view của component

Views/Shared/Components/DepartmentList/Default.cshtmlrazor
@model List<Department>

@if (Model is not null && Model.Count > 0)
{
    @foreach (var department in Model)
    {
        @await Html.PartialAsync("_DepartmentListItem", department)
    }
}

Gọi View Component trong view

Views/Departments/Index.cshtmlrazor
@{
    ViewBag.Title = "Departments";
}

@section title_section {
    <h3>Departments</h3>
}

<table class="table table-striped">
    <thead>
        <tr>
            <th>Name</th>
            <th>Description</th>
            <th></th>
        </tr>
    </thead>

10 Truyền tham số vào View Component

Bài 10 · Pass parameters to ViewComponent

Một View Component hoàn toàn tự chủ nhưng vẫn có thể nhận tham số từ bên ngoài. Đây là cách làm cho component linh hoạt: có thể dùng lại với những yêu cầu lọc/hiển thị khác nhau.

Cập nhật lớp ViewComponent — nhận tham số filter

DepartmentListViewComponent.cscsharp
using Microsoft.AspNetCore.Mvc;
using WebApp.Models;

namespace WebApp.Views.Shared.Components.DepartmentList
{
    [ViewComponent]
    public class DepartmentListViewComponent : ViewComponent
    {
        // Thêm tham số filter — có thể null (hiển thị tất cả)
        public IViewComponentResult Invoke(string? filter)
        {
            var departments = DepartmentsRepository.GetDepartments(filter);
            return View(departments);
        }
    }
}

Truyền tham số khi gọi component

Sử dụng anonymous object (tương tự route values) để truyền tham số:

Views/Departments/Index.cshtmlrazor
@{
    ViewBag.Title = "Departments";
}

@section title_section {
    <h3>Departments</h3>
}

<table class="table table-striped">
    <thead>
        <tr>
            <th>Name</th>
            <th>Description</th>
            <th></th>
        </tr>
    </thead>
    <tbody id="department-list">
        @* Truyền filter = null để lấy tất cả khi tải trang *@
        @await Component.InvokeAsync("DepartmentList", new { filter = (string?)null })
    </tbody>
</table>

<a class="btn btn-primary" href="/departments/create">Add</a>

Dùng Tag Helper thay cho InvokeAsync

Sau khi thêm @addTagHelper *, Microsoft.AspNetCore.Mvc.TagHelpers vào _ViewImports.cshtml, có thể gọi View Component bằng tag HTML:

Cú pháp Tag Helperrazor
<!-- Không truyền tham số -->
<vc:department-list />

<!-- Truyền tham số filter -->
<vc:department-list filter="IT" />

ℹ️Tên tag helper dùng kebab-case: DepartmentList → vc:department-list. Đây là cú pháp sạch hơn, thân thiện với editor và dễ đọc hơn InvokeAsync.
