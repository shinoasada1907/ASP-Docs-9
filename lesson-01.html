Introduction

Tổng quan khóa học, hiểu ASP.NET Core là gì, cách một ứng dụng ASP.NET Core được cấu thành và tạo ứng dụng web đầu tiên.

1 Giới thiệu khóa học

Bài 1 · Introduction

Chào mừng bạn đến với khóa học ASP.NET Core Deep Dive. Mục tiêu của khóa học là giúp bạn hiểu tường tận cách ASP.NET Core hoạt động và khai thác hết sức mạnh của framework này thông qua các tình huống thực tế và hướng dẫn từng bước.

Lộ trình khóa học

Khóa học được chia thành các phần lớn, xây dựng kiến thức theo từng lớp:

  • Nền tảng: các khối kiến thức cơ bản của một ứng dụng web — giao thức HTTP, HttpContext, và middleware (các thành phần phần mềm xếp thành một pipeline xử lý request/response). Bạn sẽ viết code ngay từ phần đầu tiên.
  • Minimal API: routing, model binding, model validation, tạo kết quả trả về và tổ chức code.
  • MVC: xây dựng ứng dụng web theo mô hình MVC — routing, binding, validation, Razor Views, layout, tái sử dụng giao diện bằng Partial View và View Component, Tag Helpers, làm việc với form.
  • Web API: tiêu thụ (consume) và tự xây dựng API bằng cả Controller lẫn Minimal API, tài liệu hóa với Swagger và versioning.
  • Dữ liệu: truy cập cơ sở dữ liệu với Entity Framework Core, CRUD và migrations.
  • Vận hành: cấu hình (options pattern), logging (kèm Serilog), xử lý ngoại lệ.
  • Bảo mật: authentication & authorization, Identity, và bảo mật API bằng JWT.

🎯Triết lý khóa học: không chỉ học cách dùng, mà hiểu tại sao mọi thứ hoạt động như vậy. Kiến thức được tích lũy dần — mỗi phần là nền tảng cho phần kế tiếp.

2 ASP.NET Core là gì

Bài 2 · What is ASP.NET Core

ASP.NET Core là một framework phát triển web giúp lập trình viên xây dựng ứng dụng web. Để hiểu nó, trước tiên cần hiểu một ứng dụng web (web application) là gì.

Mô hình Client – Server

Về bản chất, một ứng dụng web là chương trình giao tiếp giữa hai phía: client và server, giao tiếp với nhau qua giao thức HTTP.

  • Client gửi một HTTP request đến server. Client thường là trình duyệt web, nhưng cũng có thể là ứng dụng di động hoặc một server khác — bất cứ thứ gì có thể gửi HTTP request.
  • Server (còn gọi là web server) lắng nghe các request, xử lý và gửi lại một HTTP response.

ASP.NET Core giúp chúng ta xây dựng phía server của ứng dụng web.

ℹ️ ASP.NET Core là mã nguồn mở, đa nền tảng (Windows, Linux, macOS), hiệu năng cao và là thế hệ kế thừa hiện đại của ASP.NET truyền thống. Phiên bản dùng trong khóa học là .NET 9.

3 Giải phẫu ứng dụng ASP.NET Core

Bài 3 · ASP.NET Core Anatomy

Một ứng dụng ASP.NET Core thực chất là một ứng dụng console được khởi động từ phương thức Main. Từ .NET 6 trở đi, với top-level statements, toàn bộ phần khởi tạo được gói gọn trong file Program.cs.

Các thành phần chính

Thành phầnVai trò
WebApplicationBuilderTạo bởi WebApplication.CreateBuilder(args); nơi đăng ký dịch vụ (services) vào DI container và cấu hình.
WebApplication (app)Đối tượng ứng dụng được builder.Build() tạo ra; nơi cấu hình middleware pipeline và định nghĩa endpoint.
app.Run()Khởi động web server (Kestrel) và bắt đầu lắng nghe request.

Cấu trúc thư mục tiêu biểu

  • Program.cs — điểm khởi đầu, cấu hình services & pipeline.
  • appsettings.json — cấu hình ứng dụng (chuỗi kết nối, logging…).
  • *.csproj — file project, khai báo SDK và các package NuGet.
  • Properties/launchSettings.json — cấu hình khi chạy ở môi trường phát triển (port, môi trường…).
  • wwwroot/ — chứa tài nguyên tĩnh (CSS, JS, hình ảnh).

4 Ứng dụng Web đầu tiên

Bài 4 · Our first Web App

Hãy tạo ứng dụng web đầu tiên. Có thể dùng Visual Studio (template ASP.NET Core Empty) hoặc dòng lệnh .NET CLI.

Tạo project bằng .NET CLI

Terminalbash
dotnet new web -n MyFirstApp
cd MyFirstApp
dotnet run

Nội dung Program.cs tối giản

Template web (Empty) sinh ra một file Program.cs ngắn gọn như sau:

Program.cscsharp
var builder = WebApplication.CreateBuilder(args);
var app = builder.Build();

app.MapGet("/", () => "Hello World!");

app.Run();

Giải thích từng dòng:

  1. WebApplication.CreateBuilder(args) tạo builder, nạp cấu hình mặc định và DI container.
  2. builder.Build() dựng đối tượng app để cấu hình pipeline.
  3. app.MapGet("/", ...) định nghĩa một endpoint: khi có request GET /, trả về chuỗi "Hello World!".
  4. app.Run() khởi động server và bắt đầu lắng nghe.

💡 Khi chạy dotnet run, terminal sẽ in ra địa chỉ dạng http://localhost:5xxx. Mở địa chỉ này trên trình duyệt để thấy dòng chữ Hello World! — ứng dụng web đầu tiên của bạn đã chạy.
