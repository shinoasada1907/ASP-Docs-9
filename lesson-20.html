Consume API (Tiêu thụ API)

Cách gọi API từ ứng dụng ASP.NET Core — sử dụng HttpClient, IHttpClientFactory, Named Client, hiểu cơ chế bên trong của HttpClient, và dùng Polly để xử lý lỗi mạng tạm thời.

1 Dùng IHttpClientFactory gọi API

Bài 1 · Use HttpClientFactory to access API endpoints

Khi ứng dụng ASP.NET Core cần gọi một Web API bên ngoài (hoặc một service khác trong cùng hệ thống), ta cần một HTTP client để gửi request và nhận response. .NET cung cấp lớp HttpClient cho mục đích này, nhưng cách tốt nhất để sử dụng nó là thông qua IHttpClientFactory.

Tại sao không dùng new HttpClient() trực tiếp?

Tạo HttpClient bằng new trong mỗi request là một anti-pattern phổ biến. Dù HttpClient implements IDisposable, việc gọi Dispose() thường xuyên không đóng ngay socket bên dưới — socket vẫn ở trạng thái TIME_WAIT trong vài phút, dẫn đến tình trạng cạn kiệt socket (socket exhaustion). Ngược lại, giữ một instance duy nhất thì lại không phản ứng kịp khi DNS thay đổi.

IHttpClientFactory giải quyết cả hai vấn đề: nó quản lý vòng đời của các HttpMessageHandler bên trong (pool và tái dùng), đồng thời tự động làm mới handler sau mỗi 2 phút để phản ánh thay đổi DNS.

Đăng ký IHttpClientFactory

Chỉ cần gọi AddHttpClient() trong Program.cs — sau đó IHttpClientFactory sẵn sàng được inject vào bất kỳ class nào.

Program.cscsharp
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllersWithViews();
builder.Services.AddRazorPages();

// Đăng ký IHttpClientFactory vào DI container
builder.Services.AddHttpClient();

var app = builder.Build();
// ... middleware pipeline
app.Run();

Sử dụng trong Controller

Inject IHttpClientFactory vào constructor, gọi CreateClient() để lấy một HttpClient, sau đó đặt BaseAddress và thực hiện request.

Controllers/DepartmentsController.cscsharp
public class DepartmentsController : Controller
{
    private readonly IHttpClientFactory httpClientFactory;

    public DepartmentsController(IHttpClientFactory httpClientFactory)
    {
        this.httpClientFactory = httpClientFactory;
    }

    [HttpGet]
    public async Task<IActionResult> Details(int id)
    {
        var client = httpClientFactory.CreateClient();
        client.BaseAddress = new Uri("http://localhost:5065/");

        var response = await client.GetAsync($"departments/{id}");
        response.EnsureSuccessStatusCode();

        var responseString = await response.Content.ReadAsStringAsync();
        var department = JsonSerializer.Deserialize<Department>(
            responseString,
            new JsonSerializerOptions { PropertyNameCaseInsensitive = true });

        return View(department);
    }
}

ℹ️ response.EnsureSuccessStatusCode() sẽ ném ra HttpRequestException nếu status code không nằm trong khoảng 2xx. Đây là cách nhanh nhất để phát hiện lỗi từ API phía server.

Deserialize JSON response

API thường trả về JSON với tên property theo kiểu camelCase (vd: departmentId), trong khi model C# dùng PascalCase. Cần đặt PropertyNameCaseInsensitive = true khi deserialize để tránh lỗi ánh xạ.

💡 Từ .NET 7 trở đi, bạn cũng có thể dùng phương thức tiện lợi hơn: await response.Content.ReadFromJsonAsync<Department>() (yêu cầu System.Net.Http.Json) thay vì đọc chuỗi rồi deserialize thủ công.

2 Named Client với IHttpClientFactory

Bài 2 · Create a Named Client with HttpClientFactory

Ở bài trước, mỗi lần dùng CreateClient() ta phải set lại BaseAddress và các header mặc định — điều này vi phạm nguyên tắc DRY (Don't Repeat Yourself). Named Client cho phép khai báo cấu hình một lần ở Program.cs rồi dùng lại bằng tên ở mọi nơi.

Đăng ký Named Client

Program.cscsharp
// Đăng ký Named Client tên "ApiEndpoints"
builder.Services.AddHttpClient("ApiEndpoints", (HttpClient client) =>
{
    client.BaseAddress = new Uri("http://localhost:5065/");
    // Có thể thêm header mặc định:
    // client.DefaultRequestHeaders.Add("Accept", "application/json");
});

Sử dụng Named Client

Truyền đúng tên đã đăng ký vào CreateClient(). Lúc này factory sẽ tạo HttpClient đã được cấu hình sẵn — không cần set BaseAddress lại nữa.

Controllers/DepartmentsController.cscsharp
// Dùng Named Client — BaseAddress đã được cấu hình sẵn
var client = httpClientFactory.CreateClient("ApiEndpoints");

var response = await client.GetAsync($"departments/{id}");
response.EnsureSuccessStatusCode();

So sánh các cách dùng IHttpClientFactory

Cách Đặc điểm Khi nào dùng
Basic — CreateClient() Trả về client chưa cấu hình, phải set thủ công Khi gọi nhiều API khác nhau, mỗi nơi một cấu hình riêng
Named Client — CreateClient("tên") Cấu hình tập trung trong Program.cs, tái dùng qua tên Khi nhiều chỗ gọi cùng một API với cùng cấu hình
Typed Client Gói HttpClient vào một class strongly-typed Khi muốn encapsulate logic gọi API vào service riêng

🔑 Named Client giúp cấu hình HttpClient (base URL, headers, timeout) tập trung một chỗ, dễ thay đổi và không bị lặp code ở nhiều controller/service.

3 Gọi API từ lớp Repository

Bài 3 · Call API endpoints from our repository class

Thay vì để logic gọi API nằm trong Controller (vi phạm Single Responsibility), ta nên tách ra thành một lớp Repository riêng. Repository này đóng gói toàn bộ giao tiếp với API — controller chỉ cần gọi phương thức của repository mà không cần biết HTTP bên trong.

Interface Repository

Định nghĩa contract — những thao tác nào cần thiết với API departments:

Models/IDepartmentsApiRepository.cscsharp
namespace WebApp.Models
{
    public interface IDepartmentsApiRepository
    {
        Task<Department?> GetDepartmentByIdAsync(int id);
        Task<List<Department>> GetDepartmentsAsync(string? filter = null);
        Task AddDepartmentAsync(Department? department);
        Task<bool> UpdateDepartmentAsync(Department? department);
        Task<bool> DeleteDepartmentAsync(Department? department);
        Task<bool> DepartmentExistsAsync(int departmentId);
    }
}

Triển khai Repository

Inject IHttpClientFactory vào repository, dùng Named Client để gọi API:

Models/DepartmentsApiRepository.cscsharp
using System.Text.Json;

namespace WebApp.Models
{
    public class DepartmentsApiRepository : IDepartmentsApiRepository
    {
        private readonly IHttpClientFactory httpClientFactory;

        public DepartmentsApiRepository(IHttpClientFactory httpClientFactory)
        {
            this.httpClientFactory = httpClientFactory;
        }

        public async Task<Department?> GetDepartmentByIdAsync(int id)
        {
            var client = httpClientFactory.CreateClient("ApiEndpoints");

            var response = await client.GetAsync($"departments/{id}");
            response.EnsureSuccessStatusCode();

            var responseString = await response.Content.ReadAsStringAsync();
            return JsonSerializer.Deserialize<Department>(
                responseString,
                new JsonSerializerOptions { PropertyNameCaseInsensitive = true });
        }

        // ... các phương thức khác
    }
}

Đăng ký trong Program.cs

Program.cscsharp
// Đăng ký repository với lifetime Transient (mỗi lần inject = 1 instance mới)
builder.Services.AddTransient<IDepartmentsApiRepository, DepartmentsApiRepository>();

builder.Services.AddHttpClient("ApiEndpoints", (HttpClient client) =>
{
    client.BaseAddress = new Uri("http://localhost:5065/");
});

Controller sạch hơn nhờ Repository

Controllers/DepartmentsController.cscsharp
public class DepartmentsController : Controller
{
    private readonly IDepartmentsApiRepository departmentsApiRepository;

    public DepartmentsController(IDepartmentsApiRepository departmentsApiRepository)
    {
        this.departmentsApiRepository = departmentsApiRepository;
    }

    [HttpGet]
    public async Task<IActionResult> Details(int id)
    {
        // Controller không biết gì về HttpClient — chỉ gọi repository
        var department = await departmentsApiRepository.GetDepartmentByIdAsync(id);
        return View(department);
    }
}

💡 Pattern này rất phổ biến trong các ứng dụng thực tế. Khi cần chuyển từ gọi API sang truy cập DB trực tiếp (hoặc ngược lại), bạn chỉ cần tạo một implementation mới của interface và thay đổi đăng ký trong Program.cs — controller hoàn toàn không bị ảnh hưởng.

4 HttpClient hoạt động như thế nào

Bài 4 · How HttpClient works

HttpClient không gửi request trực tiếp đến server — bên trong nó sử dụng một pipeline các HttpMessageHandler, tương tự middleware pipeline của ASP.NET Core nhưng dành cho phía client (outgoing request).

Kiến trúc DelegatingHandler

Mỗi handler trong pipeline nhận request, có thể xử lý/thay đổi nó, rồi chuyển cho handler tiếp theo bằng base.SendAsync(). Handler cuối cùng (HttpClientHandler) thực sự gửi request qua mạng và nhận response. Response đi ngược lại qua pipeline.

Sơ đồ pipelinetext
HttpClient.GetAsync(url)
    |
    v
[DelegatingHandler 1]  <-- có thể thêm header, log, validate...
    |
    v
[DelegatingHandler 2]  <-- có thể retry, circuit breaker...
    |
    v
[HttpClientHandler]    <-- gửi request qua mạng thật
    |
    v  (response trả ngược lại qua các handler)
HttpResponseMessage

Tự viết DelegatingHandler

Kế thừa DelegatingHandler và override SendAsync. Ví dụ dưới đây kiểm tra header X-API-KEY trước khi cho request đi tiếp:

MessageHandlers/ValidateApiHeaderHandler.cscsharp
namespace WebApp.MessageHandlers
{
    public class ValidateApiHeaderHandler : DelegatingHandler
    {
        protected override async Task<HttpResponseMessage> SendAsync(
            HttpRequestMessage request,
            CancellationToken cancellationToken)
        {
            // Kiểm tra header bắt buộc
            if (!request.Headers.Contains("X-API-KEY"))
            {
                return new HttpResponseMessage(System.Net.HttpStatusCode.BadRequest)
                {
                    Content = new StringContent("The API key header X-API-KEY is required.")
                };
            }

            // Chuyển tiếp cho handler tiếp theo
            return await base.SendAsync(request, cancellationToken);
        }
    }
}

Gắn Handler vào Named Client

Program.cscsharp
// Đăng ký handler vào DI (phải là Transient)
builder.Services.AddTransient<ValidateApiHeaderHandler>();

builder.Services.AddHttpClient("ApiEndpoints", (HttpClient client) =>
{
    client.BaseAddress = new Uri("http://localhost:5065/");
})
.AddHttpMessageHandler<ValidateApiHeaderHandler>(); // gắn vào pipeline

ℹ️ DelegatingHandler phải được đăng ký là Transient (không phải Singleton hay Scoped) để tránh các vấn đề về thread-safety và vòng đời khi được resolve bởi factory.

Ứng dụng thực tế của DelegatingHandler

  • Tự động thêm Authorization header (Bearer token) vào mọi request.
  • Logging toàn bộ request/response mà không cần thay đổi code gọi.
  • Retry tự động khi gặp lỗi mạng (thường dùng Polly — xem bài tiếp).
  • Circuit Breaker — ngừng gọi API khi phát hiện lỗi liên tiếp, cho API thời gian hồi phục.
  • Ghi đè URL hoặc thêm query string tự động.

5 Dùng Polly xử lý lỗi tạm thời

Bài 5 · Use Polly to deal with temporary errors

Khi ứng dụng gọi một API bên ngoài, các lỗi mạng tạm thời là điều không thể tránh khỏi — timeout, mất kết nối ngắn hạn, server đang restart, v.v. Thay vì để người dùng thấy lỗi ngay lập tức, Polly cho phép định nghĩa các resilience policy (chính sách phục hồi) như retry, circuit breaker, fallback.

Cài đặt package

Terminalbash
dotnet add package Microsoft.Extensions.Http.Polly

Thêm Retry Policy vào Named Client

Gọi AddTransientHttpErrorPolicy() trên kết quả của AddHttpClient(). Policy này tự động retry khi gặp lỗi mạng hoặc status code 5xx/408:

Program.cscsharp
using Polly;

builder.Services.AddHttpClient("ApiEndpoints", (HttpClient client) =>
{
    client.BaseAddress = new Uri("http://localhost:5065/");
})
.AddTransientHttpErrorPolicy(policy =>
{
    // Retry 3 lần, mỗi lần chờ tăng dần: 100ms, 200ms, 300ms
    return policy.WaitAndRetryAsync(new[]
    {
        TimeSpan.FromMilliseconds(100),
        TimeSpan.FromMilliseconds(200),
        TimeSpan.FromMilliseconds(300),
    });
});

ℹ️ AddTransientHttpErrorPolicy chỉ áp dụng cho lỗi tạm thời (transient errors): lỗi mạng (HttpRequestException), timeout (TaskCanceledException), và HTTP 5xx hoặc 408. Nó không retry với HTTP 4xx (như 401, 404) vì đó là lỗi logic, không phải lỗi hạ tầng.

AddStandardResilienceHandler (.NET 8+)

Từ .NET 8, Microsoft giới thiệu Microsoft.Extensions.Http.Resilience — một cách hiện đại hơn để cấu hình resilience với Polly v8:

Program.cs (.NET 9 — cách hiện đại)csharp
// Cài: dotnet add package Microsoft.Extensions.Http.Resilience
builder.Services.AddHttpClient("ApiEndpoints", client =>
{
    client.BaseAddress = new Uri("http://localhost:5065/");
})
.AddStandardResilienceHandler(options =>
{
    // Tuỳ chỉnh retry (mặc định đã có retry + circuit breaker)
    options.Retry.MaxRetryAttempts = 3;
    options.Retry.Delay = TimeSpan.FromMilliseconds(200);
});

Các Policy phổ biến của Polly

Policy Mô tả
Retry Tự động thử lại sau khi gặp lỗi, có thể cấu hình số lần và thời gian chờ
WaitAndRetry Retry với thời gian chờ tăng dần (exponential backoff) để tránh quá tải server
Circuit Breaker Ngừng gọi API sau N lần lỗi liên tiếp, chờ một khoảng thời gian rồi thử lại
Timeout Hủy request nếu không nhận được response trong thời gian cho phép
Fallback Trả về giá trị dự phòng khi mọi retry đều thất bại

⚠️ Khi dùng Retry, hãy đảm bảo API phía server idempotent — tức là gọi nhiều lần cùng request cho kết quả như nhau. Retry một request POST tạo dữ liệu có thể tạo ra bản ghi trùng lặp!

6 Bài tập 12 — Gọi API trong DepartmentsController

Bài 6 · Assignment 12 - Invoke API in DepartmentsController

Bài tập này củng cố toàn bộ kiến thức chương: thay thế repository truy cập dữ liệu trực tiếp bằng API repository để mọi action trong DepartmentsController đều gọi qua HTTP.

Yêu cầu bài tập

Ở các bài trước, DepartmentsController dùng IDepartmentsRepository (truy cập list in-memory). Nhiệm vụ là:

  1. Hoàn thiện interface IDepartmentsApiRepository với đầy đủ 6 phương thức (Get, GetById, Add, Update, Delete, Exists).
  2. Triển khai DepartmentsApiRepository — mỗi phương thức gọi endpoint tương ứng trên Web API.
  3. Sửa DepartmentsController để inject IDepartmentsApiRepository thay vì IDepartmentsRepository.
  4. Đổi tất cả action từ đồng bộ sang async Task<IActionResult>.
  5. Đăng ký IDepartmentsApiRepository và Named Client trong Program.cs.

💡 Các HTTP method ánh xạ tự nhiên với CRUD: GET = đọc, POST = tạo, PUT = cập nhật toàn bộ, DELETE = xóa. Kiểm tra API project để biết đúng endpoint URL của từng thao tác.

Gợi ý — gửi JSON body với POST/PUT

Để gửi một object dưới dạng JSON body, dùng StringContent với content type application/json:

Gợi ý gửi POST requestcsharp

using static System.Net.Mime.MediaTypeNames;

var departmentJson = new StringContent(
    JsonSerializer.Serialize(department),
    Encoding.UTF8,
    Application.Json);           // "application/json"

var response = await client.PostAsync("/departments", departmentJson);
response.EnsureSuccessStatusCode();

DepartmentsApiRepository (đầy đủ)

Models/DepartmentsApiRepository.cscsharp
using System.Text;
using System.Text.Json;
using static System.Net.Mime.MediaTypeNames;

namespace WebApp.Models
{
    public class DepartmentsApiRepository : IDepartmentsApiRepository
    {
        private readonly IHttpClientFactory httpClientFactory;

        public DepartmentsApiRepository(IHttpClientFactory httpClientFactory)
        {
            this.httpClientFactory = httpClientFactory;
        }

        public async Task<Department?> GetDepartmentByIdAsync(int id)
        {
            var client = httpClientFactory.CreateClient("ApiEndpoints");
            var response = await client.GetAsync($"departments/{id}");
            response.EnsureSuccessStatusCode();

            var responseString = await response.Content.ReadAsStringAsync();
            return JsonSerializer.Deserialize<Department>(
                responseString,
                new JsonSerializerOptions { PropertyNameCaseInsensitive = true });
        }

        public async Task<List<Department>> GetDepartmentsAsync(string? filter = null)
        {
            var client = httpClientFactory.CreateClient("ApiEndpoints");
            var response = await client.GetAsync($"departments/{filter}");
            response.EnsureSuccessStatusCode();

            var responseString = await response.Content.ReadAsStringAsync();
            var departments = JsonSerializer.Deserialize<List<Department>>(
                responseString,
                new JsonSerializerOptions { PropertyNameCaseInsensitive = true });

            return departments ?? new List<Department>();
        }

        public async Task AddDepartmentAsync(Department? department)
        {
            var client = httpClientFactory.CreateClient("ApiEndpoints");

            var departmentJson = new StringContent(
                JsonSerializer.Serialize(department),
                Encoding.UTF8,
                Application.Json);

            var response = await client.PostAsync("/departments", departmentJson);
            response.EnsureSuccessStatusCode();
        }

        public async Task<bool> UpdateDepartmentAsync(Department? department)
        {
            if (department is null) return false;

            var client = httpClientFactory.CreateClient("ApiEndpoints");

            var departmentJson = new StringContent(
                JsonSerializer.Serialize(department),
                Encoding.UTF8,
                Application.Json);

            var response = await client.PutAsync($"/departments/{department.Id}", departmentJson);
            response.EnsureSuccessStatusCode();
            return true;
        }

        public async Task<bool> DeleteDepartmentAsync(Department? department)
        {
            if (department is null) return false;

            var client = httpClientFactory.CreateClient("ApiEndpoints");
            var response = await client.DeleteAsync($"/departments/{department.Id}");
            response.EnsureSuccessStatusCode();
            return true;
        }

        public async Task<bool> DepartmentExistsAsync(int departmentId)
        {
            var client = httpClientFactory.CreateClient("ApiEndpoints");
            var response = await client.GetAsync($"departments/{departmentId}/exists");
            response.EnsureSuccessStatusCode();

            var responseString = await response.Content.ReadAsStringAsync();
            return bool.Parse(responseString);
        }
    }
}

DepartmentsController — hoàn toàn async, dùng API Repository

Controllers/DepartmentsController.cscsharp
using Microsoft.AspNetCore.Mvc;
using WebApp.Models;

namespace WebApp.Controllers
{
    public class DepartmentsController : Controller
    {
        private readonly IDepartmentsApiRepository departmentsApiRepository;

        public DepartmentsController(IDepartmentsApiRepository departmentsApiRepository)
        {
            this.departmentsApiRepository = departmentsApiRepository;
        }

        [HttpGet]
        public IActionResult Index() => View();

        [HttpGet]
        public async Task<IActionResult> Details(int id)
        {
            var department = await departmentsApiRepository.GetDepartmentByIdAsync(id);
            return View(department);
        }

        [HttpGet]
        public IActionResult Create() => View(new Department());

        [HttpPost]
        public async Task<IActionResult> Create(Department department)
        {
            await departmentsApiRepository.AddDepartmentAsync(department);
            return RedirectToAction(nameof(Index));
        }

        [HttpPost]
        public async Task<IActionResult> Edit(Department department)
        {
            await departmentsApiRepository.UpdateDepartmentAsync(department);
            return RedirectToAction(nameof(Index));
        }

        [HttpPost]
        public async Task<IActionResult> Delete(int id)
        {
            var department = await departmentsApiRepository.GetDepartmentByIdAsync(id);
            await departmentsApiRepository.DeleteDepartmentAsync(department);
            return RedirectToAction(nameof(Index));
        }

        [HttpGet]
        public async Task<IActionResult> GetDepartments()
        {
            var departments = await departmentsApiRepository.GetDepartmentsAsync();
            return Json(departments);
        }
    }
}

Program.cs hoàn chỉnh

using WebApp.Filters;
using WebApp.MessageHandlers;
using WebApp.Models;

    options.Filters.Add<WriteToConsoleResourceFilter>();

{
    return policy.WaitAndRetryAsync(new[]

        TimeSpan.FromMilliseconds(100),
        TimeSpan.FromMilliseconds(200),
        TimeSpan.FromMilliseconds(300),
    });
});

var app = builder.Build();

app.UseStaticFiles();
