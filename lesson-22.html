Minimal API — Documentation & Versioning

Tài liệu hóa Minimal API với OpenAPI tích hợp sẵn của .NET 9, tích hợp Swagger UI, bổ sung metadata chi tiết cho từng endpoint, và quản lý versioning với Asp.Versioning.Http.

1 Tài liệu hóa Minimal API với OpenAPI

Bài 1 · Use OpenAPI to document Minimal APIs

Khi xây dựng API — dù là Minimal API hay Web API với Controllers — việc tài liệu hóa (documentation) là bắt buộc nếu API đó được tiêu thụ bởi nhiều nhóm phát triển khác nhau, thậm chí bởi các công ty bên ngoài. Người dùng API cần biết:

  • Có những URL (endpoint) nào và phương thức HTTP nào (GET, POST, PUT, DELETE…)?
  • Mỗi endpoint nhận những tham số nào, kiểu dữ liệu là gì?
  • Phản hồi trả về có cấu trúc ra sao, kiểu dữ liệu nào?

OpenAPI (trước đây gọi là Swagger Specification) là một chuẩn mở để mô tả API RESTful dưới dạng tài liệu JSON/YAML có thể đọc bởi cả máy lẫn người. Từ bản tài liệu này, các công cụ như Postman, Swagger UI có thể tự động tạo giao diện kiểm thử mà không cần viết thêm code.

🎯.NET 9 tích hợp sẵn OpenAPI: Từ .NET 9, ASP.NET Core đã có package Microsoft.AspNetCore.OpenApi tích hợp trực tiếp vào SDK, cho phép sinh tài liệu OpenAPI mà không cần Swashbuckle để generate document. Chỉ cần gọi builder.Services.AddOpenApi() và app.MapOpenApi().

Cài đặt và cấu hình cơ bản

Package Microsoft.AspNetCore.OpenApi đã có sẵn trong SDK .NET 9. Với project ASP.NET Core Web API (hoặc Empty), chỉ cần thêm package nếu chưa có:

Terminalbash
dotnet add package Microsoft.AspNetCore.OpenApi

Tiếp theo cấu hình trong Program.cs:

Program.cscsharp
var builder = WebApplication.CreateBuilder(args);

// Đăng ký dịch vụ sinh tài liệu OpenAPI
builder.Services.AddOpenApi();

var app = builder.Build();

// Ánh xạ endpoint phục vụ tài liệu OpenAPI dưới dạng JSON
// Mặc định truy cập tại: /openapi/v1.json
app.MapOpenApi();

// Định nghĩa các Minimal API endpoint
app.MapGet("/departments", () => Results.Ok(new[] { "HR", "IT", "Finance" }))
   .WithName("GetDepartments")
   .WithSummary("Lấy danh sách phòng ban");

app.MapGet("/departments/{id}", (int id) => Results.Ok(new { Id = id, Name = "HR" }))
   .WithName("GetDepartmentById")
   .WithSummary("Lấy phòng ban theo ID");

app.Run();

Sau khi chạy ứng dụng, truy cập http://localhost:PORT/openapi/v1.json để xem tài liệu OpenAPI dạng JSON.

ℹ️Phân biệt với Swashbuckle: Swashbuckle (Swashbuckle.AspNetCore) là giải pháp bên thứ ba, bao gồm cả phần sinh document (Swashbuckle.AspNetCore.SwaggerGen) lẫn giao diện Swagger UI. Trong .NET 9, phần sinh document được thay thế bởi Microsoft.AspNetCore.OpenApi, còn Swagger UI vẫn có thể dùng từ Swashbuckle hoặc Scalar.

Hai lý do chính cần tài liệu hóa API

Lý doMô tả
Cung cấp specification Cho phép các nhóm khác (thậm chí ở công ty khác, ở bất kỳ nơi nào trên thế giới) biết cách gọi đúng API mà không cần đọc source code.
Kiểm thử API Các công cụ như Postman, Swagger UI đọc tài liệu OpenAPI và tự tạo giao diện giúp kiểm thử từng endpoint trực tiếp trên trình duyệt mà không cần viết code client.

2 OpenAPI hoạt động như thế nào

Bài 2 · How OpenAPI Works

Để hiểu cách OpenAPI hoạt động trong ASP.NET Core .NET 9, hãy nhìn vào middleware pipeline. Khi gọi app.MapOpenApi(), một endpoint đặc biệt được đăng ký trong hệ thống routing.

Luồng xử lý request tài liệu OpenAPI

  1. Client gửi request đến GET /openapi/v1.json.
  2. Request đi qua middleware pipeline thông thường (Routing, Authentication,…).
  3. Routing nhận ra đây là endpoint tài liệu OpenAPI và chuyển đến handler do MapOpenApi() đăng ký.
  4. Handler tự động duyệt qua tất cả endpoint đã được đăng ký trong ứng dụng (thông qua API Explorer), thu thập metadata: URL, phương thức HTTP, tham số, kiểu trả về.
  5. Handler sinh ra tài liệu JSON theo chuẩn OpenAPI 3.x và trả về cho client.

ℹ️Tài liệu OpenAPI được sinh động (on-the-fly) — mỗi lần request đến /openapi/v1.json, hệ thống đọc lại thông tin endpoint tại thời điểm đó. Điều này đảm bảo tài liệu luôn đồng bộ với code thực tế.

Cơ chế khám phá endpoint (API Explorer)

ASP.NET Core sử dụng API Explorer — một abstraction nằm bên trong framework — để thu thập thông tin về tất cả endpoint. Khi bạn gọi MapGet, MapPost… và bổ sung metadata (WithName, Produces, WithSummary…), các thông tin này được lưu vào endpoint metadata. Middleware OpenAPI đọc metadata đó và chuyển hóa thành cấu trúc OpenAPI chuẩn.

Ví dụ cấu trúc tài liệu JSON (rút gọn)json
{
  "openapi": "3.0.1",
  "info": { "title": "MyApi", "version": "1.0" },
  "paths": {
    "/departments": {
      "get": {
        "summary": "Lấy danh sách phòng ban",
        "operationId": "GetDepartments",
        "responses": {
          "200": { "description": "OK" }
        }
      }
    }
  }
}

3 Xem tài liệu OpenAPI bằng Postman

Bài 3 · Use Postman to Visualize OpenAPI documents

Tài liệu OpenAPI dạng JSON thuần có thể khó đọc đối với người dùng thông thường. Postman có khả năng import tài liệu OpenAPI và tự động tạo một Collection gồm tất cả các request tương ứng với từng endpoint — giúp việc kiểm thử API trở nên trực quan hơn nhiều.

Các bước import vào Postman

  1. Chạy ứng dụng ASP.NET Core, truy cập http://localhost:PORT/openapi/v1.json và sao chép toàn bộ nội dung JSON.
  2. Mở Postman → chọn File → Import.
  3. Chọn tab Raw text, dán JSON vào và nhấn Continue → Import.
  4. Postman sẽ tạo một Collection với các thư mục và request tương ứng với từng endpoint và tag.

💡Base URL variable: Sau khi import, Postman tự tạo biến baseUrl trỏ về http://localhost:PORT. Khi deploy lên môi trường production, bạn chỉ cần thay đổi giá trị biến này một lần — tất cả request trong collection sẽ tự cập nhật.

Lưu ý về tính thời điểm

Collection được tạo từ Postman chỉ là một snapshot tại thời điểm import. Nếu API có thay đổi (thêm endpoint, đổi tham số…), bạn cần import lại tài liệu OpenAPI mới nhất để Postman Collection phản ánh đúng thực tế.

ℹ️Ngoài Postman, bạn cũng có thể dùng Scalar — một công cụ UI hiện đại được Microsoft khuyến nghị cho .NET 9, tích hợp trực tiếp vào ứng dụng qua package Scalar.AspNetCore — như một thay thế cho Swagger UI.

4 Hiển thị tài liệu bằng Swagger UI

Bài 4 · Use Swagger to visualize the OpenAPI document

Swagger UI là giao diện web tương tác, đọc tài liệu OpenAPI và hiển thị dưới dạng trang HTML trực quan. Người dùng có thể xem danh sách endpoint, mô tả, và gọi thử API trực tiếp trên trình duyệt mà không cần Postman hay bất kỳ công cụ nào khác.

Trong .NET 9, Microsoft.AspNetCore.OpenApi chỉ sinh ra tài liệu JSON. Để có giao diện Swagger UI, cần cài thêm package Swashbuckle.AspNetCore.SwaggerUi (chỉ lấy phần UI, không lấy phần generate document của Swashbuckle).

Cài đặt

Terminalbash
dotnet add package Swashbuckle.AspNetCore.SwaggerUi

Cấu hình trong Program.cs

Program.cscsharp
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddOpenApi(); // Sinh tài liệu OpenAPI JSON (.NET 9 built-in)

var app = builder.Build();

// Middleware sinh tài liệu OpenAPI JSON tại /openapi/v1.json
app.MapOpenApi();

// Middleware Swagger UI — đọc tài liệu từ endpoint MapOpenApi()
// và hiển thị giao diện tại /swagger
app.UseSwaggerUI(options =>
{
    options.SwaggerEndpoint("/openapi/v1.json", "Department Management API v1");
    options.RoutePrefix = "swagger"; // truy cập tại /swagger
});

app.MapGet("/departments", () => Results.Ok(new[] { "HR", "IT", "Finance" }));
app.MapPost("/departments", (Department dept) => Results.Created($"/departments/1", dept));

app.Run();

Sau khi cấu hình, truy cập http://localhost:PORT/swagger để thấy giao diện Swagger UI.

💡Chỉ bật ở môi trường Development: Trong production, không nên expose tài liệu API công khai trừ khi API là public. Nên bọc trong điều kiện if (app.Environment.IsDevelopment()) để chỉ bật khi phát triển.

Program.cs — Chỉ bật khi Developmentcsharp
if (app.Environment.IsDevelopment())
{
    app.MapOpenApi();
    app.UseSwaggerUI(options =>
    {
        options.SwaggerEndpoint("/openapi/v1.json", "My API v1");
    });
}

5 Bổ sung chi tiết cho tài liệu OpenAPI

Bài 5 · Provide details to OpenAPI document

Mặc định, MapOpenApi() tự động phát hiện URL, phương thức HTTP và kiểu tham số từ code. Tuy nhiên nhiều thông tin quan trọng phải bổ sung thủ công để tài liệu trở nên đầy đủ và dễ hiểu hơn với người dùng API.

Các extension method bổ sung metadata

MethodÝ nghĩa
WithName(string)Đặt operationId — định danh duy nhất cho endpoint trong tài liệu OpenAPI
WithSummary(string)Mô tả ngắn hiển thị ngay trên thanh tiêu đề endpoint trong Swagger UI
WithDescription(string)Mô tả chi tiết, hỗ trợ Markdown
WithTags(string[])Nhóm endpoint theo tag — Swagger UI hiển thị theo nhóm
Produces<T>(int statusCode)Khai báo kiểu dữ liệu trả về và HTTP status code tương ứng
ProducesProblem(int statusCode)Khai báo trường hợp trả về lỗi ProblemDetails
WithOpenApi()Kích hoạt tùy chỉnh OpenAPI nâng cao (bắt buộc khi dùng attribute trên delegate)

Ví dụ đầy đủ

Program.cscsharp
using Microsoft.AspNetCore.Http.HttpResults;

var builder = WebApplication.CreateBuilder(args);
builder.Services.AddOpenApi();

var app = builder.Build();
app.MapOpenApi();
app.UseSwaggerUI(o => o.SwaggerEndpoint("/openapi/v1.json", "API v1"));

// --- Nhóm Department ---
var deptGroup = app.MapGroup("/departments").WithTags("Departments");

deptGroup.MapGet("/", () => Results.Ok(new[] { new Department(1, "HR"), new Department(2, "IT") }))
    .WithName("GetDepartments")
    .WithSummary("Lấy tất cả phòng ban")
    .WithDescription("Trả về danh sách toàn bộ phòng ban hiện có trong hệ thống.")
    .Produces<Department[]>(StatusCodes.Status200OK);

deptGroup.MapGet("/{id:int}", (int id) =>
    {
        var dept = new Department(id, "HR");
        return Results.Ok(dept);
    })
    .WithName("GetDepartmentById")
    .WithSummary("Lấy phòng ban theo ID")
    .Produces<Department>(StatusCodes.Status200OK)
    .ProducesProblem(StatusCodes.Status404NotFound);

deptGroup.MapPost("/", (Department dept) => Results.Created($"/departments/{dept.Id}", dept))
    .WithName("CreateDepartment")
    .WithSummary("Tạo phòng ban mới")
    .Produces<Department>(StatusCodes.Status201Created)
    .ProducesValidationProblem();

deptGroup.MapPut("/{id:int}", (int id, Department dept) => Results.Ok(dept))
    .WithName("UpdateDepartment")
    .WithSummary("Cập nhật phòng ban");

deptGroup.MapDelete("/{id:int}", (int id) => Results.NoContent())
    .WithName("DeleteDepartment")
    .WithSummary("Xóa phòng ban");

app.Run();

record Department(int Id, string Name);

Sử dụng attribute trên endpoint handler

Nếu muốn dùng attribute (giống cách làm với Controller), cần thêm .WithOpenApi() vào cuối chuỗi và đặt attribute trực tiếp trên lambda:

Program.cs — Dùng attributecsharp
app.MapGet("/employees", [Tags("Employees")] () =>
    {
        return Results.Ok(new[] { new Employee(1, "Alice"), new Employee(2, "Bob") });
    })
    .WithName("GetEmployees")
    .WithSummary("Lấy danh sách nhân viên")
    .Produces<Employee[]>(200)
    .WithOpenApi(); // bắt buộc khi dùng attribute trên delegate

record Employee(int Id, string FullName);

💡Nên ưu tiên dùng extension method (WithSummary, WithTags…) thay vì attribute trên lambda — cú pháp nhất quán, dễ đọc hơn và không cần WithOpenApi().

Kết quả trong Swagger UI

Sau khi bổ sung đầy đủ metadata:

  • Các endpoint được nhóm theo tag (Departments, Employees).
  • Mỗi endpoint hiển thị summary, description, các status code có thể trả về.
  • Schema của request body và response body được hiển thị tự động.

6 Versioning cho Minimal API

Bài 6 · Minimal API Versioning

API Versioning là kỹ thuật duy trì nhiều phiên bản của cùng một API cùng lúc, để các client đang dùng phiên bản cũ không bị ảnh hưởng khi API được cập nhật (thay đổi breaking).

Tại sao cần versioning?

Giả sử bạn có endpoint GET /departments đang được nhiều ứng dụng mobile (version 1) sử dụng. Nếu bạn thay đổi cấu trúc phản hồi, các ứng dụng cũ sẽ bị lỗi. Giải pháp: tạo v2 của endpoint với URL riêng (/api/v2/departments) — client cũ vẫn dùng v1, client mới dùng v2.

Cài đặt package

Terminalbash
dotnet add package Asp.Versioning.Http

Cấu hình versioning

Program.cscsharp
using Asp.Versioning;
using Asp.Versioning.Builder;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddOpenApi();

// Đăng ký dịch vụ API Versioning
builder.Services.AddApiVersioning(options =>
{
    options.DefaultApiVersion = new ApiVersion(1, 0);
    options.AssumeDefaultVersionWhenUnspecified = true;
    options.ReportApiVersions = true; // trả về header api-supported-versions
});

var app = builder.Build();
app.MapOpenApi();
app.UseSwaggerUI(o => o.SwaggerEndpoint("/openapi/v1.json", "API v1"));

// Tạo ApiVersionSet — tập hợp các phiên bản được hỗ trợ
ApiVersionSet versionSet = app.NewApiVersionSet()
    .HasApiVersion(new ApiVersion(1, 0))
    .HasApiVersion(new ApiVersion(2, 0))
    .ReportApiVersions()
    .Build();

// Tạo route group có prefix version trong URL: /api/v{version}/...
var apiGroup = app.MapGroup("/api/v{version:apiVersion}")
                  .WithApiVersionSet(versionSet);

// Version 1 — phản hồi đơn giản
apiGroup.MapGet("/departments", () =>
    Results.Ok(new[] { new DeptV1(1, "HR"), new DeptV1(2, "IT") }))
    .MapToApiVersion(new ApiVersion(1, 0))
    .WithName("GetDepartmentsV1")
    .WithSummary("Lấy phòng ban (v1)");

// Version 2 — phản hồi mở rộng thêm trường
apiGroup.MapGet("/departments", () =>
    Results.Ok(new[] { new DeptV2(1, "HR", 50), new DeptV2(2, "IT", 30) }))
    .MapToApiVersion(new ApiVersion(2, 0))
    .WithName("GetDepartmentsV2")
    .WithSummary("Lấy phòng ban (v2) — bao gồm số nhân viên");

app.Run();

record DeptV1(int Id, string Name);
record DeptV2(int Id, string Name, int EmployeeCount);

Cách gọi API theo version

URLKết quả
GET /api/v1/departmentsTrả về danh sách DeptV1
GET /api/v2/departmentsTrả về danh sách DeptV2 (thêm trường)

ℹ️Các chiến lược versioning: Ngoài URL segment (/v1/), Asp.Versioning.Http còn hỗ trợ versioning qua query string (?api-version=1.0) và HTTP header (X-Api-Version: 1.0). Có thể kết hợp nhiều chiến lược cùng lúc.

⚠️Khi dùng MapGroup với WithApiVersionSet, tất cả endpoint trong group đều phải được gắn MapToApiVersion() rõ ràng, nếu không framework sẽ không biết endpoint đó thuộc version nào.

7 Tài liệu hóa nhiều phiên bản API

Bài 7 · Documenting multiple versions for minimal APIs

Sau khi cấu hình versioning, nếu truy cập /openapi/v1.json hoặc /openapi/v2.json, bạn sẽ thấy thông báo lỗi: "No OpenAPI document was found with the name 'v2'". Nguyên nhân là mặc định AddOpenApi() chỉ tạo một document tên v1, chưa biết về các version khác.

Cấu hình nhiều OpenAPI document

Cần gọi AddOpenApi() cho từng version, kèm theo cách lọc endpoint thuộc version đó:

Program.cscsharp
using Asp.Versioning;
using Asp.Versioning.Builder;
using Microsoft.AspNetCore.OpenApi;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddApiVersioning(options =>
{
    options.DefaultApiVersion = new ApiVersion(1, 0);
    options.AssumeDefaultVersionWhenUnspecified = true;
    options.ReportApiVersions = true;
});

// Đăng ký document OpenAPI cho Version 1
builder.Services.AddOpenApi("v1", options =>
{
    options.ShouldInclude = (description) =>
        description.GroupName == "v1";
});

// Đăng ký document OpenAPI cho Version 2
builder.Services.AddOpenApi("v2", options =>
{
    options.ShouldInclude = (description) =>
        description.GroupName == "v2";
});

var app = builder.Build();

// Ánh xạ 2 endpoint tài liệu riêng biệt
app.MapOpenApi("/openapi/{documentName}.json");

// Swagger UI — liệt kê cả hai version
app.UseSwaggerUI(options =>
{
    options.SwaggerEndpoint("/openapi/v1.json", "API Version 1");
    options.SwaggerEndpoint("/openapi/v2.json", "API Version 2");
});

// Tạo ApiVersionSet
ApiVersionSet versionSet = app.NewApiVersionSet()
    .HasApiVersion(new ApiVersion(1, 0))
    .HasApiVersion(new ApiVersion(2, 0))
    .ReportApiVersions()
    .Build();

var apiGroup = app.MapGroup("/api/v{version:apiVersion}")
                  .WithApiVersionSet(versionSet);

// Endpoint v1 — gắn GroupName để lọc vào document "v1"
apiGroup.MapGet("/departments", () =>
    Results.Ok(new[] { new DeptV1(1, "HR") }))
    .MapToApiVersion(new ApiVersion(1, 0))
    .WithName("GetDepartmentsV1")
    .WithSummary("Lấy danh sách phòng ban (v1)")
    .WithGroupName("v1");

// Endpoint v2 — gắn GroupName để lọc vào document "v2"
apiGroup.MapGet("/departments", () =>
    Results.Ok(new[] { new DeptV2(1, "HR", 50) }))
    .MapToApiVersion(new ApiVersion(2, 0))
    .WithName("GetDepartmentsV2")
    .WithSummary("Lấy danh sách phòng ban (v2) — có số nhân viên")
    .WithGroupName("v2");

apiGroup.MapGet("/departments/{id:int}", (int id) =>
    Results.Ok(new DeptV1(id, "HR")))
    .MapToApiVersion(new ApiVersion(1, 0))
    .WithName("GetDepartmentByIdV1")
    .WithGroupName("v1");

apiGroup.MapGet("/departments/{id:int}", (int id) =>
    Results.Ok(new DeptV2(id, "HR", 50)))
    .MapToApiVersion(new ApiVersion(2, 0))
    .WithName("GetDepartmentByIdV2")
    .WithGroupName("v2");

app.Run();

record DeptV1(int Id, string Name);
record DeptV2(int Id, string Name, int EmployeeCount);

Truy cập tài liệu từng version

URLNội dung
/openapi/v1.jsonTài liệu OpenAPI chỉ chứa endpoint v1
/openapi/v2.jsonTài liệu OpenAPI chỉ chứa endpoint v2
/swaggerSwagger UI với dropdown chọn v1 hoặc v2

💡Dropdown version trong Swagger UI: Khi bạn đăng ký nhiều SwaggerEndpoint, Swagger UI sẽ hiển thị một dropdown selector ở góc phải trên cùng, cho phép chuyển đổi qua lại giữa tài liệu v1 và v2 ngay trong trình duyệt.

Tóm tắt toàn bộ cấu hình

  1. Cài Microsoft.AspNetCore.OpenApi (sinh tài liệu JSON — built-in .NET 9) và Asp.Versioning.Http (versioning).
  2. Cài Swashbuckle.AspNetCore.SwaggerUi nếu cần giao diện Swagger UI trong trình duyệt.
  3. Gọi AddApiVersioning() và AddOpenApi("v1") + AddOpenApi("v2") trong phần đăng ký dịch vụ.
  4. Gọi app.MapOpenApi("/openapi/{documentName}.json") và app.UseSwaggerUI() sau builder.Build().
  5. Tạo ApiVersionSet, dùng MapGroup với WithApiVersionSet.
  6. Gắn MapToApiVersion() và WithGroupName() cho từng endpoint.

🎯Điểm mấu chốt: WithGroupName("v1") liên kết endpoint với tài liệu OpenAPI có tên "v1" — hai tên này phải khớp nhau. Đây là cơ chế mà ShouldInclude dùng để lọc endpoint đưa vào từng document.
