MVC — Layout Views

Tái sử dụng HTML dùng chung qua Layout file, điều phối layout tự động bằng _ViewStart, tập trung các directive với _ViewImports, truyền dữ liệu từ view lên layout qua ViewBag, và mở rộng layout linh hoạt với Sections.

1 Layout Razor View

Bài 1 · Layout razor view

Khi xây dựng ứng dụng MVC với nhiều view, mỗi view đều phải chứa phần khung HTML đầy đủ: thẻ <html>, <head>, link CSS, <body>… Điều này dẫn đến việc lặp lại (duplicate) rất nhiều code HTML giống hệt nhau ở mọi file view.

Layout file là giải pháp của ASP.NET Core MVC để giải quyết vấn đề này — tương tự như khái niệm master page hay template. Một layout file là một file .cshtml bình thường, nhưng chứa phần HTML dùng chung và có một "lỗ trống" đặc biệt là @RenderBody() — nơi nội dung của từng view riêng lẻ sẽ được chèn vào khi render.

Quy ước đặt tên và vị trí

  • Theo convention, layout file được đặt tên bắt đầu bằng dấu gạch dưới: _Layout.cshtml.
  • Vị trí phổ biến nhất là thư mục Views/Shared/ để tất cả các view trong ứng dụng đều tìm thấy.
  • Tên file thực ra có thể tùy ý — điều quan trọng là view phải chỉ định đúng tên layout khi tham chiếu tới nó.

Cách hoạt động

  1. View riêng lẻ (vd: Index.cshtml) khai báo Layout = "_Layout" trong code block Razor.
  2. Razor engine nạp file _Layout.cshtml, render phần HTML chung.
  3. Tại vị trí @RenderBody() trong layout, engine chèn toàn bộ nội dung HTML của view riêng lẻ vào.
  4. Kết quả HTML hoàn chỉnh được trả về cho browser.

Layout file (_Layout.cshtml)

Views/Shared/_Layout.cshtmlrazor
<html>
    <head>
        <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
              rel="stylesheet" />
    </head>
    <body>
        <nav class="navbar navbar-expand-lg bg-body-tertiary">
            <div class="container-fluid">
                <a class="navbar-brand" href="#">Departments</a>
                <div class="collapse navbar-collapse">
                    <ul class="navbar-nav me-auto mb-2 mb-lg-0">
                        <li class="nav-item">
                            <a class="nav-link" href="/">Home</a>
                        </li>
                        <li class="nav-item">
                            <a class="nav-link" href="/departments">Departments</a>
                        </li>
                    </ul>
                </div>
            </div>
        </nav>

        <div class="container">
            @RenderBody()
        </div>

        <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>
    </body>
</html>

View sử dụng layout

Mỗi view cần khai báo layout nào nó muốn dùng thông qua thuộc tính Layout trong một code block Razor @{ }. Chỉ cần viết tên (không cần đường dẫn đầy đủ và không cần extension .cshtml).

Views/Departments/Index.cshtmlrazor
@using WebApp.Models
@model List<Department>

@{
    Layout = "_Layout";
}

<h3>Departments</h3>

@if (Model is not null && Model.Count > 0)
{
    <table class="table table-striped">
        <thead>
            <tr>
                <th>Name</th>
                <th>Description</th>
                <th></th>
            </tr>
        </thead>
        <tbody>
            @foreach (var department in Model)
            {
                <tr>
                    <td>@department.Name</td>
                    <td>@department.Description</td>
                    <td><a class="btn btn-link" href="/departments/details/@department.Id">Edit</a></td>
                </tr>
            }
        </tbody>
    </table>
}

<a class="btn btn-primary" href="/departments/create">Add</a>

💡 Razor engine tìm layout theo thứ tự: trước tiên trong cùng thư mục với view, nếu không thấy thì tìm trong Views/Shared/. Đây là lý do tại sao đặt layout trong Shared/ là convention phổ biến nhất — mọi view đều tìm thấy nó.

ℹ️ Nếu muốn view không dùng layout nào cả, đặt Layout = null trong code block. Điều này hữu ích cho các partial response hoặc trang đặc biệt như trang login không cần thanh navbar.

2 Multiple Layouts & Nested Layouts

Bài 2 · Multiple Layouts and Nested Layouts

Một ứng dụng không nhất thiết chỉ có một layout duy nhất. Bạn có thể tạo nhiều layout file khác nhau và để từng nhóm view sử dụng layout phù hợp với mình.

Multiple Layouts (Nhiều layout)

Giả sử ứng dụng có hai khu vực: khu vực công khai và khu vực admin. Khu vực admin có thể dùng một layout riêng với sidebar, trong khi khu vực công khai dùng layout đơn giản hơn.

Cơ chế tìm layout của Razor engine:

  1. Razor tìm layout trong cùng thư mục với view hiện tại trước.
  2. Nếu không tìm thấy, nó leo lên thư mục Views/Shared/.

Điều này cho phép bạn tạo một layout đặc biệt chỉ cho một nhóm view bằng cách đặt file layout đó vào cùng thư mục. Ví dụ: tạo Views/Departments/_Layout.cshtml — tất cả view trong thư mục Departments/ khi khai báo Layout = "_Layout" sẽ ưu tiên dùng file này thay vì file layout trong Shared/.

Views/Departments/_Layout.cshtmlrazor
<!-- Layout đặc biệt chỉ dành cho khu vực Departments -->
<html>
    <head>
        <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
              rel="stylesheet" />
    </head>
    <body>
        <h1>Departments Section</h1>
        <hr />
        <div class="container">
            @RenderBody()
        </div>
    </body>
</html>

Nested Layouts (Layout lồng nhau)

Một layout file cũng có thể sử dụng layout khác làm cha của nó — đây là kỹ thuật nested layout. Điều này cho phép xây dựng cấu trúc layout phân cấp, tránh lặp lại HTML chung giữa các layout.

Ví dụ: _AdminLayout.cshtml sử dụng _Layout.cshtml làm layout cha, và thêm một sidebar đặc biệt cho khu vực admin.

Views/Shared/_AdminLayout.cshtmlrazor
@{
    Layout = "_Layout";   <!-- layout này cũng dùng _Layout làm cha -->
}

<!-- Nội dung bổ sung của admin layout -->
<div class="row">
    <div class="col-2">
        <!-- Admin Sidebar -->
        <ul>
            <li><a href="/admin/users">Users</a></li>
            <li><a href="/admin/settings">Settings</a></li>
        </ul>
    </div>
    <div class="col-10">
        @RenderBody()
    </div>
</div>

Các view trong khu vực admin chỉ cần khai báo Layout = "_AdminLayout". Kết quả render sẽ là: nội dung view → chèn vào @RenderBody() của _AdminLayout → kết quả đó lại được chèn vào @RenderBody() của _Layout cha.

🔑 Thứ tự tìm layout: cùng thư mục → Views/Shared/. Nested layout render theo thứ tự từ ngoài vào trong: layout cha render trước, sau đó chèn nội dung của layout con vào, rồi mới chèn nội dung view.

3 _ViewStart — Tự động áp dụng Layout

Bài 3 · _ViewStart razor view

Sau khi thêm layout, bạn nhận thấy rằng mọi view đều phải lặp lại đoạn code sau:

(mỗi view đều phải có)razor
@{
    Layout = "_Layout";
}

Đây vẫn là dạng lặp code. _ViewStart.cshtml được tạo ra để giải quyết vấn đề này.

_ViewStart là gì?

_ViewStart.cshtml là một file Razor đặc biệt được Razor engine tự động render trước bất kỳ view nào trong cùng thư mục hoặc thư mục con. Đây là nơi lý tưởng để đặt việc chỉ định layout mặc định — mọi view sẽ kế thừa setting này mà không cần khai báo lại.

Vị trí đặt file

Theo convention, _ViewStart.cshtml được đặt trong thư mục Views/ (root của thư mục Views), không phải trong Shared/. Nó áp dụng cho tất cả view bên dưới thư mục đó.

Views/_ViewStart.cshtmlrazor
@{
    Layout = "_Layout";
}

Sau khi có file này, tất cả các view không cần khai báo Layout nữa — chúng sẽ tự động dùng _Layout.

Views/Home/Index.cshtml (sau khi có _ViewStart)razor
@{
    var days = new DateTime(DateTime.Today.Year, 12, 25) - DateTime.Today;

    if (DateTime.Today > new DateTime(DateTime.Today.Year, 12, 25))
    {
        days = new DateTime(DateTime.Today.Year + 1, 12, 25) - DateTime.Today;
    }

    string color = days.Days % 2 == 0 ? "green" : "red";
}

@if (days.Days <= 35)
{
    <h1>Christmas is around the corner!</h1>
    @:Happy holidays!
}
else
{
    <h1 style="color:@color">Welcome home!</h1>
    @days.Days <text> days before Christmas!</text>
}

💡 Nếu một view riêng lẻ muốn ghi đè layout mặc định từ _ViewStart, nó vẫn có thể khai báo Layout = "TênLayoutKhác" hoặc Layout = null trong code block của mình — view luôn thắng _ViewStart.

Thứ tự áp dụng _ViewStart

_ViewStart cũng hoạt động theo cơ chế phân cấp thư mục. Nếu có nhiều file _ViewStart.cshtml ở các cấp thư mục khác nhau, tất cả đều được thực thi — từ thư mục con lên thư mục cha. Điều này cho phép ghi đè layout ở cấp thư mục cụ thể.

Vị tríÁp dụng cho
Views/_ViewStart.cshtmlTất cả view trong Views/ và thư mục con
Views/Departments/_ViewStart.cshtmlChỉ các view trong Views/Departments/

4 _ViewImports — Tập trung các Directive

Bài 4 · _ViewImports to centralize directives

Tương tự như vấn đề với Layout, các view cũng thường phải lặp lại các directive như @using, @model namespace, hay @addTagHelper. Ví dụ, mỗi view trong thư mục Departments/ cần @using WebApp.Models để nhận biết class Department.

_ViewImports.cshtml là file Razor đặc biệt cho phép bạn khai báo các directive dùng chung một lần duy nhất, áp dụng cho tất cả view trong cùng thư mục và thư mục con.

_ViewImports khác _ViewStart như thế nào?

Đặc điểm_ViewStart_ViewImports
Mục đíchThực thi code trước mỗi view (thường set layout)Chia sẻ directive (using, namespace…)
Nội dung hợp lệCode Razor bất kỳChỉ các directive (@using, @addTagHelper, @namespace…)
Vị trí conventionViews/Views/

File _ViewImports thực tế

Views/_ViewImports.cshtmlrazor
@using WebApp.Models
@addTagHelper *, Microsoft.AspNetCore.Mvc.TagHelpers

Sau khi có file này, các view không cần khai báo @using WebApp.Models nữa. Ví dụ, Index.cshtml trong thư mục Departments được rút gọn:

Views/Departments/Index.cshtml (sau khi có _ViewImports)razor
@model List<Department>

<!-- Không cần @using WebApp.Models nữa -->

@if (Model is not null && Model.Count > 0)
{
    <table class="table table-striped">
        <thead>
            <tr><th>Name</th><th>Description</th><th></th></tr>
        </thead>
        <tbody>
            @foreach (var department in Model)
            {
                <tr>
                    <td>@department.Name</td>
                    <td>@department.Description</td>
                    <td><a class="btn btn-link" href="/departments/details/@department.Id">Edit</a></td>
                </tr>
            }
        </tbody>
    </table>
}

<a class="btn btn-primary" href="/departments/create">Add</a>

ℹ️ _ViewImports cũng hoạt động theo cơ chế phân cấp. Các directive được cộng dồn từ nhiều cấp _ViewImports (không phải ghi đè). File ở thư mục con được hợp nhất với file ở thư mục cha.

Các directive thường dùng trong _ViewImports

  • @using Namespace — import namespace để dùng các class không cần gõ tên đầy đủ.
  • @addTagHelper *, Microsoft.AspNetCore.Mvc.TagHelpers — kích hoạt Tag Helpers tích hợp sẵn của ASP.NET Core.
  • @namespace MyApp.Views — đặt namespace mặc định cho tất cả Razor Page trong thư mục.

5 ViewData / ViewBag giao tiếp với Layout

Bài 5 · Use ViewData or ViewBag to communicate with layout files

Layout file được dùng chung cho nhiều view. Vậy làm thế nào để layout có thể hiển thị thông tin khác nhau tùy theo view đang được render? Ví dụ điển hình nhất là tiêu đề trang (page title) — mỗi trang cần tiêu đề khác nhau trong thẻ <title>.

Giải pháp: dùng ViewData hoặc ViewBag để truyền dữ liệu từ view riêng lẻ lên layout file.

ViewData vs ViewBag

ViewDataViewBag
KiểuViewDataDictionary (dictionary)Dynamic wrapper của ViewData
Cú pháp đọc/ghiViewData["Key"]ViewBag.Key
Type safetyCần ép kiểu khi đọcDynamic, không cần ép kiểu
Nền tảngLà nền tảng thực sựViewBag là alias của ViewData

Cả hai đều chia sẻ cùng một storage — ghi vào ViewBag.Title rồi đọc bằng ViewData["Title"] đều cho kết quả như nhau.

Ví dụ thực tế: tiêu đề động

Bước 1: Trong layout, dùng @ViewBag.Title (hoặc ViewData["Title"]) tại thẻ <title>:

Views/Shared/_Layout.cshtmlrazor
<html>
    <head>
        <title>@ViewBag.Title</title>
        <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
              rel="stylesheet" />
    </head>
    <body>
        <nav class="navbar navbar-expand-lg bg-body-tertiary">
            <!-- ... navbar content ... -->
        </nav>
        <div class="container">
            @RenderBody()
        </div>
    </body>
</html>

Bước 2: Trong mỗi view riêng lẻ, set giá trị cho ViewBag.Title:

Views/Departments/Index.cshtmlrazor
@model List<Department>

@{
    ViewBag.Title = "Departments";
}

<!-- Phần còn lại của view -->
<h3>Departments</h3>
<!-- ... -->
Views/Home/Index.cshtmlrazor
@{
    ViewBag.Title = "Home";
    <!-- ... logic khác ... -->
}

<h1>Welcome home!</h1>

Kết quả: thẻ <title> sẽ hiện "Departments" khi render trang danh sách, và "Home" khi render trang chủ — trong khi layout vẫn là file dùng chung.

⚠️ ViewData và ViewBag không có type safety — lỗi chính tả tên key chỉ phát sinh lúc runtime, không bị bắt lúc compile. Dùng chúng cho các trường hợp đơn giản như page title là hợp lý; với dữ liệu phức tạp hơn, hãy ưu tiên strongly-typed ViewModel.

6 Sections — Các Placeholder Bổ Sung

Bài 6 · Sections as additional placeholders

@RenderBody() chỉ cung cấp một vị trí (placeholder) trong layout để chèn nội dung của view. Nhưng đôi khi bạn cần nhiều vị trí hơn — ví dụ: view muốn chèn nội dung vào phần navbar, hoặc thêm CSS/JS đặc thù vào phần <head>, hoặc đặt một tiêu đề ở một khu vực riêng ngoài body chính.

Sections là cơ chế của Razor cho phép định nghĩa thêm các placeholder tùy ý trong layout. View riêng lẻ có thể cung cấp nội dung cho các section đó.

Định nghĩa Section trong Layout

Dùng @RenderSection("tên_section", required: bool) trong layout để đánh dấu một vị trí placeholder có tên:

  • required: true — view bắt buộc phải cung cấp nội dung cho section này, nếu không sẽ gây lỗi runtime.
  • required: false — section là tùy chọn; nếu view không cung cấp, vị trí đó sẽ bỏ trống.
Views/Shared/_Layout.cshtml (với sections)razor
<html>
    <head>
        <title>@ViewBag.Title</title>
        <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
              rel="stylesheet" />
    </head>
    <body>
        <nav class="navbar navbar-expand-lg bg-body-tertiary">
            <div class="container-fluid">
                <a class="navbar-brand" href="#">Departments</a>
                <div class="collapse navbar-collapse">
                    <ul class="navbar-nav me-auto mb-2 mb-lg-0">
                        <li class="nav-item"><a class="nav-link" href="/">Home</a></li>
                        <li class="nav-item"><a class="nav-link" href="/departments">Departments</a></li>
                    </ul>
                    @* Section tùy chọn trong navbar — view có thể thêm nút/link vào đây *@
                    @RenderSection("a_section", false)
                </div>
            </div>
        </nav>

        <div class="container">
            @* Section tùy chọn cho tiêu đề trang — đặt trên body chính *@
            @RenderSection("title_section", false)
            <br />

            @* Placeholder chính — nội dung view chèn vào đây *@
            @RenderBody()
        </div>

        <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>
    </body>
</html>

Cung cấp nội dung cho Section từ View

Trong view riêng lẻ, dùng cú pháp @section tên_section { ... } để cung cấp nội dung cho section tương ứng:

Views/Departments/Index.cshtmlrazor
@model List<Department>

@{
    ViewBag.Title = "Departments";
}

@* Nội dung chèn vào section "title_section" trong layout *@
@section title_section {
    <h3>Departments</h3>
}

@* Phần còn lại chèn vào @RenderBody() *@
@if (Model is not null && Model.Count > 0)
{
    <table class="table table-striped">
        <thead>
            <tr>
                <th>Name</th>
                <th>Description</th>
                <th></th>
            </tr>
        </thead>
        <tbody>
            @foreach (var department in Model)
            {
                <tr>
                    <td>@department.Name</td>
                    <td>@department.Description</td>
                    <td><a class="btn btn-link" href="/departments/details/@department.Id">Edit</a></td>
                </tr>
            }
        </tbody>
    </table>
}

<a class="btn btn-primary" href="/departments/create">Add</a>
Views/Departments/Details.cshtmlrazor
@model Department

@{
    ViewBag.Title = "Department Details";
}

@section title_section {
    <h3>Department</h3>
}

<form method="post" action="/departments/edit">

    <div class="row mb-3">

            <label class="col-form-label">Name</label>
        </div>
        <div class="col-6">

    </div>

    <div class="row mb-3">
        <div class="col-2">
            <label class="col-form-label">Description</label>
        </div>

Tổng quan cơ chế Section

Cú pháp trong LayoutCú pháp trong ViewÝ nghĩa
@RenderBody()(toàn bộ HTML còn lại)Placeholder chính, bắt buộc có trong layout
@RenderSection("name", true)@section name { ... }Placeholder có tên, view bắt buộc cung cấp
@RenderSection("name", false)@section name { ... } (tùy chọn)Placeholder có tên, view không bắt buộc

🔑 Khi nào dùng Section thay vì ViewBag? Dùng ViewBag khi chỉ cần truyền dữ liệu đơn giản (chuỗi, số) lên layout. Dùng @section khi cần chèn HTML/markup phức tạp vào nhiều vị trí khác nhau trong layout — như script, style đặc thù trang, hoặc widget.

💡 Lưu ý đặt script cuối trang: Sections đặc biệt hữu ích để chèn JavaScript đặc thù của từng trang vào cuối <body>. Bạn có thể tạo một section scripts trong layout và để các view tự quyết định có thêm script đặc thù hay không — đây là pattern rất phổ biến trong các dự án ASP.NET Core MVC.
