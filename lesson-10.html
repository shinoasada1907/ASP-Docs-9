MVC Controllers — Producing Results

Khám phá các kiểu kết quả mà MVC Controller có thể trả về — ContentResult, JsonResult, File Results, Redirect Results — và xây dựng giao diện CRUD hoàn chỉnh cho Departments bằng ContentResult thuần HTML.

1 Sự khác biệt giữa MVC Controller và Minimal API khi trả kết quả

Bài 1 · Main differences in MVC controllers

Trong phần Minimal API chúng ta đã học về việc trả kết quả — NotFoundResult, OkResult, các status-code result… Những khái niệm đó vẫn áp dụng được trong MVC Controller. Tuy nhiên, MVC Controller chủ yếu được dùng để xây dựng giao diện người dùng (UI), nên kiểu kết quả quan trọng nhất và thường gặp nhất là ViewResult — kết quả HTML được sinh ra từ Razor View.

Bức tranh tổng quan các kiểu kết quả trong MVC Controller

Kiểu kết quảLớp / HelperKhi nào dùng
ViewResult View() Trả về trang HTML từ file .cshtml (Razor View). Đây là kết quả chính trong MVC UI.
ContentResult Content(...) Trả về chuỗi văn bản thuần hoặc HTML tự tạo, có thể chỉ định Content-Type.
JsonResult Json(...) Buộc kết quả trả về là JSON bất kể Accept header của client.
FileResult File(...) / PhysicalFile(...) Trả về nội dung file để hiển thị hoặc tải xuống trong trình duyệt.
RedirectResult RedirectToAction(...) / LocalRedirect(...) / Redirect(...) Chuyển hướng người dùng đến action khác, URL nội bộ, hoặc URL bên ngoài.

ℹ️ ViewResult sẽ được học ở chương tiếp theo khi chúng ta nghiên cứu Razor Views. Trong chương này chúng ta tập trung vào bốn kiểu còn lại: ContentResult, JsonResult, FileResult và RedirectResult.

Tất cả đều implement IActionResult

Mọi kiểu kết quả trong MVC đều implement interface IActionResult (hoặc kế thừa ActionResult đã implement nó). Vì vậy kiểu trả về phổ biến nhất của một action method là:

DepartmentsController.cscsharp
public IActionResult Index()
{
    // Có thể trả bất kỳ kiểu kết quả nào
    return Content("Hello from MVC!");
}

2 ContentResult — Trả về nội dung văn bản / HTML

Bài 2 · ContentResult

ContentResult là kiểu kết quả dùng để trả về một chuỗi văn bản bất kỳ — có thể là plain text, HTML, hay thậm chí JSON — kèm theo Content-Type tùy chọn. Đây là cách đơn giản nhất để tạo nội dung giao diện thủ công mà chưa cần Razor View.

Cách khởi tạo trực tiếp (instantiate)

DepartmentsController.cscsharp
public IActionResult Index()
{
    // Khởi tạo ContentResult thủ công
    return new ContentResult()
    {
        Content = "<h1>Welcome to the Homepage</h1>",
        ContentType = "text/html"
    };
}

Dùng helper method Content() — cách ngắn gọn hơn

Controller có sẵn phương thức trợ giúp Content() để tạo ContentResult nhanh hơn:

DepartmentsController.cscsharp
// Trả về plain text (Content-Type mặc định: text/plain)
public IActionResult PlainText()
{
    return Content("Hello, World!");
}

// Trả về HTML
public IActionResult HtmlPage()
{
    return Content("<h1>Welcome</h1><p>This is a paragraph.</p>", "text/html");
}

💡 Helper method vs. khởi tạo thủ công: Kết quả hoàn toàn tương đương. Hầu hết lập trình viên dùng Content(...) vì ngắn gọn hơn. Trong Minimal API, helper tương đương nằm trong class Results; trong MVC Controller, helper nằm ngay trong class Controller nên bạn dùng trực tiếp không cần viết tên class.

Lưu ý về Content-Type

Content-TypeÝ nghĩa
text/plainTrình duyệt hiển thị như văn bản thuần; thẻ HTML không được diễn giải.
text/htmlTrình duyệt diễn giải và render HTML đầy đủ.
application/jsonTrả JSON; tuy nhiên nên dùng JsonResult thay thế.

3 JsonResult — Ép kiểu kết quả về JSON

Bài 3 · JsonResult

Nếu bạn trả về một object từ action method với kiểu trả về là IActionResult, ASP.NET Core sẽ dùng Content Negotiation (thương lượng nội dung) để quyết định format. Khi client gửi header Accept: application/xml, server sẽ trả XML thay vì JSON — nếu Output Formatter XML đã được cài đặt.

Vấn đề: Content Negotiation không phải lúc nào cũng mong muốn

DepartmentsController.cscsharp
// Nếu client gửi Accept: application/xml,
// action này có thể trả XML thay vì JSON!
public IActionResult GetDepartment()
{
    return Ok(new Department(1, "Sales", "Sales Department"));
}

Giải pháp: dùng JsonResult để luôn trả JSON

DepartmentsController.cscsharp
// Khởi tạo JsonResult thủ công
public IActionResult GetDepartmentJson()
{
    var dept = new Department(1, "Sales", "Sales Department");
    return new JsonResult(dept);
}

// Hoặc dùng helper method Json() — ngắn gọn hơn
public IActionResult GetDepartmentJsonHelper()
{
    var dept = new Department(1, "Sales", "Sales Department");
    return Json(dept);   // Luôn trả JSON bất kể Accept header
}

🎯 JsonResult bỏ qua Content Negotiation. Kể cả khi client yêu cầu XML qua header Accept: application/xml, server vẫn trả JSON và đặt Content-Type: application/json trong response. Đây là hành vi cố ý khi bạn muốn đảm bảo format JSON.

JsonResult với tùy chỉnh serializer

DepartmentsController.cscsharp
public IActionResult GetDepartmentCustomJson()
{
    var dept = new Department(1, "Engineering", "Engineering Department");

    // Overload thứ hai cho phép cấu hình JsonSerializerOptions
    return Json(dept, new System.Text.Json.JsonSerializerOptions
    {
        PropertyNamingPolicy = System.Text.Json.JsonNamingPolicy.CamelCase,
        WriteIndented = true
    });
}

ℹ️ Trong thực tế, MVC Controller xây dựng UI không thường dùng JsonResult vì Controller UI trả View. JsonResult phổ biến hơn ở Controller API. Nhưng hiểu cơ chế này rất quan trọng để biết khi nào cần ép buộc format JSON.

4 File Results — Trả về file từ server

Bài 4 · File Results

Đôi khi ứng dụng web cần cho phép người dùng xem hoặc tải xuống một file — PDF, hình ảnh, tài liệu… ASP.NET Core MVC cung cấp ba loại File Result tùy vào nguồn gốc của file.

LoạiLớpNguồn file
Virtual VirtualFileResult File nằm trong thư mục wwwroot (đường dẫn tương đối từ wwwroot).
Physical PhysicalFileResult File nằm bất kỳ đâu trên đĩa cứng server (đường dẫn tuyệt đối).
Content FileContentResult Nội dung file đã được nạp vào bộ nhớ dưới dạng byte[] (thường lấy từ database hoặc remote API).

1. VirtualFileResult — file trong wwwroot

Cần có thư mục wwwroot và file readme.txt bên trong. Lưu ý: cách này không cần cấu hình Static Files Middleware — file được trả về qua action endpoint.

DepartmentsController.cscsharp
[Route("/download_vf")]
public IActionResult ReturnVirtualFile()
{
    // Đường dẫn tương đối từ wwwroot
    // Helper File() tạo VirtualFileResult
    return File("/readme.txt", "text/plain");
}

ℹ️ Nếu bạn truy cập /readme.txt trực tiếp (không qua action), Static Files Middleware mới phục vụ file đó. Action /download_vf ở đây hoạt động độc lập với middleware đó.

2. PhysicalFileResult — file ở đường dẫn tuyệt đối

DepartmentsController.cscsharp
[Route("/download_pf")]
public IActionResult ReturnPhysicalFile()
{
    // Đường dẫn tuyệt đối trên máy chủ
    return PhysicalFile(@"c:\temp\sample.pdf", "application/pdf");
}

3. FileContentResult — từ byte array trong bộ nhớ

DepartmentsController.cscsharp
[Route("/download_cf")]
public IActionResult ReturnContentFile()
{
    // Đọc toàn bộ file vào byte[] (thực tế có thể lấy từ DB)
    byte[] bytes = System.IO.File.ReadAllBytes(@"c:\temp\sample.pdf");

    return File(bytes, "application/pdf");
}

Hiển thị trong trình duyệt vs. Tải xuống

Hành vi của trình duyệt phụ thuộc vào MIME type:

  • application/pdf, text/plain, image/png… → trình duyệt mở trực tiếp trong tab.
  • application/octet-stream → trình duyệt tải xuống vì không biết cách mở.
DepartmentsController.cscsharp
[Route("/force_download")]
public IActionResult ForceDownload()
{
    byte[] bytes = System.IO.File.ReadAllBytes(@"c:\temp\sample.pdf");

    // application/octet-stream buộc trình duyệt tải xuống
    return File(bytes, "application/octet-stream", "report.pdf");
}

💡 Tham số thứ ba của File() là fileDownloadName — tên file mà người dùng thấy khi tải xuống. Nếu bỏ qua, trình duyệt tự đặt tên.

5 Redirect Results — Chuyển hướng người dùng

Bài 5 · Redirect Results

Trong ứng dụng web có UI, sau khi xử lý một form (thêm, sửa, xóa dữ liệu), thông lệ tốt là chuyển hướng người dùng đến trang khác thay vì render nội dung ngay lập tức. Điều này tránh vấn đề "submit lại form khi refresh" (Post/Redirect/Get pattern). ASP.NET Core MVC cung cấp ba loại redirect result.

Tổng quan ba loại Redirect Result

LoạiLớp / HelperMô tả
RedirectToActionResult RedirectToAction() Chuyển hướng đến action method khác (cùng hoặc khác controller). Cách được ưa dùng nhất trong MVC UI.
LocalRedirectResult LocalRedirect() Chuyển hướng đến URL nội bộ (cùng ứng dụng) được chỉ định trực tiếp bằng chuỗi.
RedirectResult Redirect() Chuyển hướng đến URL bất kỳ, kể cả URL bên ngoài ứng dụng (vd: google.com).

1. RedirectToAction — cách ưa dùng

DepartmentsController.cscsharp
[HttpGet]
public IActionResult Details(int id)
{
    // ... logic xử lý ...

    // Chuyển hướng sang action "GetEmployeesByDepartment" của EmployeesController
    // Truyền route value qua anonymous object
    return RedirectToAction(
        actionName: "GetEmployeesByDepartment",
        controllerName: "Employees",
        routeValues: new { id = id }
    );
}

// Chuyển hướng trong cùng controller — không cần chỉ định controllerName
[HttpPost]
public IActionResult Edit(Department department)
{
    DepartmentsRepository.UpdateDepartment(department);
    return RedirectToAction(nameof(Index));   // Về trang danh sách
}

💡 Dùng nameof(Index) thay vì chuỗi "Index" để tránh lỗi typo và được refactoring tool hỗ trợ. Tương tự, nameof(EmployeesController) kết hợp với .Replace("Controller", "") giúp lấy tên controller không cần hard-code.

Cơ chế hoạt động: HTTP 302 Found

Khi action trả về RedirectToAction, server gửi response với status code 302 Found và header Location chứa URL đích. Trình duyệt đọc header này và tự động gửi một GET request đến URL mới.

HTTP Response Headershttp
HTTP/1.1 302 Found
Location: /employees/getemployeesbydepartment/99

2. LocalRedirect — chỉ định URL trực tiếp

DepartmentsController.cscsharp
[HttpGet]
public IActionResult Details(int id)
{
    // Chỉ định URL trực tiếp — phải là URL nội bộ
    return LocalRedirect($"/employees/getemployeesbydepartment/{id}");
}

3. Redirect — chuyển hướng ra ngoài ứng dụng

DepartmentsController.cscsharp
[HttpGet]
public IActionResult GoToGoogle()
{
    // Chuyển hướng sang URL bên ngoài
    return Redirect("https://www.google.com");
}

Redirect vĩnh viễn — HTTP 301 Moved Permanently

Mặc định, các redirect trả về HTTP 302 (tạm thời). Khi muốn chỉ định chuyển hướng vĩnh viễn (hữu ích khi retire API cũ), dùng phiên bản Permanent:

DepartmentsController.cscsharp
// 301 Moved Permanently
return RedirectToActionPermanent(nameof(Index));

return LocalRedirectPermanent("/departments");

return RedirectPermanent("https://newsite.example.com");

⚠️ 301 được trình duyệt và search engine lưu cache vĩnh viễn. Chỉ dùng khi bạn chắc chắn URL cũ sẽ không bao giờ được dùng lại. 302 là lựa chọn an toàn hơn cho hầu hết tình huống trong ứng dụng UI.

6 Bài tập 6 — Xây dựng CRUD UI cho Departments

Bài 6 · Assignment 6 — Implement CRUD UI for Departments

Đây là bài tập thực hành tổng hợp: xây dựng giao diện CRUD hoàn chỉnh cho Departments bằng cách kết hợp ContentResult (HTML tự tạo), RedirectToAction sau POST, và Model Binding / Validation đã học ở chương trước — chưa cần Razor View.

Yêu cầu giao diện

  1. Danh sách (GET /departments): Hiển thị danh sách phòng ban, mỗi mục có link vào trang Chi tiết. Có nút "Add Department".
  2. Chi tiết (GET /departments/details/{id}): Hiển thị form điền sẵn thông tin phòng ban — có nút Update, nút Cancel, và nút Delete.
  3. Thêm mới (GET /departments/create): Form trống để nhập thông tin phòng ban mới, nút Add.
  4. Xử lý Update (POST /departments/edit): Nhận dữ liệu form, validate, cập nhật repository, redirect về danh sách.
  5. Xử lý Create (POST /departments/create): Nhận dữ liệu form, validate, thêm vào repository, redirect về danh sách.
  6. Xử lý Delete (POST /departments/delete/{id}): Xóa phòng ban, redirect về danh sách.

Cấu hình Program.cs

Program.cscsharp
var builder = WebApplication.CreateBuilder(args);
builder.Services.AddControllers();

var app = builder.Build();
app.UseRouting();

app.MapControllerRoute(
    name: "default",
    pattern: "{controller=Home}/{action=Index}/{id?}");

app.Run();

Model và Repository (từ file đính kèm)

Models/Department.cscsharp
using System.ComponentModel.DataAnnotations;

namespace WebApp.Models
{
    public class Department
    {
        public Department() { }

        public Department(int id, string name, string? description = "")
        {
            Id = id;
            Name = name;
            Description = description;
        }

        public int Id { get; set; }

        [Required]
        public string? Name { get; set; }

        [StringLength(500)]
        public string? Description { get; set; }
    }
}
Models/DepartmentsRepository.cscsharp
namespace WebApp.Models
{
    public static class DepartmentsRepository
    {
        private static List<Department> Departments = new()
        {
            new Department(1, "Sales", "Sales Department"),
            new Department(2, "Engineering", "Engineering Department"),
            new Department(3, "QA", "Quality Assurance")
        };

        public static List<Department> GetDepartments() => Departments;

        public static Department? GetDepartmentById(int id) =>
            Departments.FirstOrDefault(x => x.Id == id);

        public static void AddDepartment(Department? dept)
        {
            if (dept is not null)
            {
                dept.Id = Departments.Max(x => x.Id) + 1;
                Departments.Add(dept);
            }
        }

        public static bool UpdateDepartment(Department? dept)
        {
            if (dept is null) return false;
            var existing = Departments.FirstOrDefault(x => x.Id == dept.Id);
            if (existing is null) return false;
            existing.Name = dept.Name;
            existing.Description = dept.Description;
            return true;
        }

        public static bool DeleteDepartment(Department? dept)
        {
            if (dept is null) return false;
            Departments.Remove(dept);
            return true;
        }
    }
}

🎯 Mục tiêu bài tập: Hiểu rằng giao diện web có thể được xây dựng bằng ContentResult (trả chuỗi HTML) ngay trong Controller — không cần Razor View. Đây là bước đệm để bạn nhận ra tại sao Razor View cần thiết: code HTML inline trong C# rất khó bảo trì.

DepartmentsController — CRUD đầy đủ

Controllers/DepartmentsController.cscsharp
using Microsoft.AspNetCore.Mvc;
using WebApp.Models;

namespace WebApp.Controllers
{
    public class DepartmentsController : Controller
    {
        // GET /departments — Danh sách phòng ban
        [HttpGet]
        public IActionResult Index()
        {
            var departments = DepartmentsRepository.GetDepartments();

            var html = $@"
                <h1>Departments</h1>
                <ul>
                    {string.Join("", departments.Select(x => $@"
                        <li>
                            <a href='/departments/details/{x.Id}'>{x.Name} ({x.Description})</a>
                        </li>
                    "))}
                </ul>
                <br/>
                <a href='/departments/create'>Add Department</a>";

            return Content(html, "text/html");
        }

        // GET /departments/details/{id} — Chi tiết + form sửa + nút xóa
        [HttpGet]
        public IActionResult Details(int id)
        {
            var department = DepartmentsRepository.GetDepartmentById(id);

            if (department == null)
                return Content("<h3 style='color:red'>Department not found.</h3>", "text/html");

            var html = $@"
                <h1>Department Details</h1>
                <form method='post' action='/departments/edit'>
                    <input type='hidden' name='Id' value='{department.Id}' />
                    <label>Name: <input type='text' name='Name' value='{department.Name}' /></label><br />
                    <label>Description: <input type='text' name='Description' value='{department.Description}' /></label><br />
                    <br/>
                    <a href='/departments'>Cancel</a>
                    <button type='submit'>Update</button>
                </form>

                <form method='post' action='/departments/delete/{department.Id}'>
                    <button type='submit' style='background-color:red;color:white'>Delete</button>
                </form>";

            return Content(html, "text/html");
        }

        // POST /departments/edit — Cập nhật phòng ban
        [HttpPost]
        public IActionResult Edit(Department department)
        {
            if (!ModelState.IsValid)
                return Content(GetErrorsHTML(), "text/html");

            DepartmentsRepository.UpdateDepartment(department);
            return RedirectToAction(nameof(Index));
        }

        // GET /departments/create — Form thêm mới (trống)
        [HttpGet]
        public IActionResult Create()
        {
            var html = @"
                <h1>Add Department</h1>
                <form method='post' action='/departments/create'>
                    <label>Name: <input type='text' name='Name' /></label><br />
                    <label>Description: <input type='text' name='Description' /></label><br />
                    <br/>
                    <button type='submit'>Add</button>
                    <a href='/departments'>Cancel</a>
                </form>";

            return Content(html, "text/html");
        }

        // POST /departments/create — Lưu phòng ban mới
        [HttpPost]
        public IActionResult Create(Department department)
        {
            if (!ModelState.IsValid)
                return Content(GetErrorsHTML(), "text/html");

            DepartmentsRepository.AddDepartment(department);
            return RedirectToAction(nameof(Index));
        }

        // POST /departments/delete/{id} — Xóa phòng ban
        [HttpPost]
        public IActionResult Delete(int id)
        {
            var department = DepartmentsRepository.GetDepartmentById(id);

            if (department == null)
            {
                ModelState.AddModelError("id", $"Department with id={id} does not exist.");
                return Content(GetErrorsHTML(), "text/html");
            }

            DepartmentsRepository.DeleteDepartment(department);
            return RedirectToAction(nameof(Index));
        }

        // Helper: tổng hợp lỗi ModelState thành HTML
        private string GetErrorsHTML()
        {
            var errorMessages = new List<string>();

            foreach (var value in ModelState.Values)
                foreach (var error in value.Errors)
                    errorMessages.Add(error.ErrorMessage);

            if (errorMessages.Count == 0) return string.Empty;

            return $@"
                <ul>
                    {string.Join("", errorMessages.Select(e => $"<li style='color:red'>{e}</li>"))}
                </ul>";
        }
    }
}

Luồng hoạt động CRUD — Post/Redirect/Get Pattern

  1. Người dùng truy cập GET /departments → thấy danh sách phòng ban.
  2. Click vào phòng ban → GET /departments/details/{id} → form điền sẵn.
  3. Sửa thông tin, click "Update" → POST /departments/edit → validate → RedirectToAction(nameof(Index)) → HTTP 302 → trình duyệt GET lại danh sách.
  4. Click "Add Department" → GET /departments/create → form trống.
  5. Điền và click "Add" → POST /departments/create → thêm vào repository → redirect về danh sách.
  6. Click "Delete" trong trang chi tiết → POST /departments/delete/{id} → xóa → redirect về danh sách.

⚠️ Nút Delete phải dùng form POST, không dùng link GET. Nếu dùng <a href='/departments/delete/1'>, bất kỳ ai (kể cả crawler) có thể vô tình kích hoạt xóa dữ liệu. Dùng <form method='post'> đảm bảo hành động xóa phải là chủ ý của người dùng.

🎯 Bài học quan trọng từ bài tập này: Viết HTML bằng string interpolation trong C# rất cồng kềnh, dễ sai, và khó bảo trì. Đây chính là lý do Razor View ra đời — cho phép viết HTML tự nhiên với C# nhúng vào, thay vì nhúng HTML vào C#. Chương tiếp theo sẽ giải quyết đúng vấn đề này.
