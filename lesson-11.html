MVC — Razor Views

Cú pháp Razor và xây dựng View trong MVC — truyền model, biểu thức ngầm & tường minh, code block, cấu trúc điều kiện & lặp, Razor literal, ViewData/ViewBag, và hoàn thiện CRUD app với Views thực sự.

1 Mô hình MVC

Bài 1 · MVC Pattern

Trước khi viết Razor View, cần hiểu tại sao MVC ra đời. Khi xây dựng ứng dụng CRUD phòng ban ở chương trước, toàn bộ HTML được ghép chuỗi thủ công bên trong Controller — một phương pháp cực kỳ khó bảo trì.

Vấn đề với Controller "chứa HTML"

  • Controller phải xử lý cả logic nghiệp vụ lẫn kết xuất giao diện — vi phạm nguyên tắc Single Responsibility.
  • HTML nhúng trong chuỗi C# không có IntelliSense, dễ sai cú pháp, khó đọc.
  • Không thể tái sử dụng giao diện giữa các action method khác nhau.

MVC giải quyết vấn đề như thế nào

MVC (Model – View – Controller) là kiến trúc tách biệt ba trách nhiệm:

Thành phầnTrách nhiệm
ModelBiểu diễn dữ liệu và logic nghiệp vụ (class Department, DepartmentsRepository).
ViewTạo HTML trả về cho client. Chỉ biết cách hiển thị dữ liệu nhận được.
ControllerĐiều phối: nhận request, gọi Model, chọn View phù hợp, truyền dữ liệu cho View.

🎯Nguyên tắc cốt lõi: Controller không viết HTML. Controller không chứa logic hiển thị. View không truy vấn cơ sở dữ liệu. Ranh giới này giúp code dễ kiểm thử và bảo trì.

Luồng xử lý request trong MVC

  1. Client gửi HTTP request đến server (vd: GET /departments).
  2. Routing ánh xạ request đến Controller và action method tương ứng (DepartmentsController.Index()).
  3. Action method gọi Model để lấy dữ liệu (DepartmentsRepository.GetDepartments()).
  4. Controller gọi View(model) — ASP.NET Core tìm file .cshtml tương ứng và render HTML.
  5. HTML hoàn chỉnh được gửi về client qua HTTP response.

Đăng ký MVC trong Program.cs

Program.cscsharp
var builder = WebApplication.CreateBuilder(args);

// Đăng ký MVC Controllers + Views
builder.Services.AddControllersWithViews();

var app = builder.Build();

app.UseRouting();

app.UseEndpoints(endpoints =>
{
    endpoints.MapControllerRoute(
        name: "default",
        pattern: "{controller=Home}/{action=Index}/{id?}"
    );
});

app.Run();

ℹ️AddControllersWithViews() khác với AddControllers() (chỉ dùng cho Web API). Phương thức này bổ sung hỗ trợ Razor View Engine, ViewResult, ViewData, v.v.

2 Razor View là gì

Bài 2 · Razor View

Thay vì nhúng HTML vào Controller dưới dạng chuỗi, ASP.NET Core cho phép đặt HTML vào các file riêng có phần mở rộng .cshtml — đây là Razor View.

Razor là gì

Razor là template engine của ASP.NET Core. Nó cho phép viết HTML thuần kết hợp với code C# trong cùng một file. Ký tự @ là ranh giới: mọi thứ sau @ được Razor coi là biểu thức hoặc khối code C#.

Quy ước đặt tên và vị trí file

ASP.NET Core tìm View theo convention-over-configuration:

  • View cho DepartmentsController.Index() nằm tại Views/Departments/Index.cshtml.
  • View cho HomeController.Index() nằm tại Views/Home/Index.cshtml.
  • Tên thư mục = tên Controller (bỏ hậu tố Controller). Tên file = tên action method.

Cấu trúc thư mục Views chuẩn

Cây thư mụctext
WebApp/
├── Controllers/
│   ├── HomeController.cs
│   └── DepartmentsController.cs
├── Models/
│   ├── Department.cs
│   └── DepartmentsRepository.cs
├── Views/
│   ├── Home/
│   │   └── Index.cshtml
│   └── Departments/
│       ├── Index.cshtml
│       └── Details.cshtml
└── Program.cs

Razor View đơn giản nhất

Views/Home/Index.cshtmlhtml
<html>
<body>
    <h1>Welcome home!</h1>
</body>
</html>

Controller trả về View này bằng cách gọi phương thức View() không tham số:

Controllers/HomeController.cscsharp
public class HomeController : Controller
{
    public IActionResult Index()
    {
        return View(); // tìm Views/Home/Index.cshtml theo convention
    }
}

💡Trong Visual Studio, bạn có thể tạo View nhanh bằng cách right-click vào tên action method → Add View → chọn Razor View – Empty. Visual Studio sẽ tự tạo thư mục và file đúng convention.

3 Truyền Model & Biểu thức Razor ngầm

Bài 3 · Passing Model to View and Implicit Razor Expression

View chỉ hiển thị dữ liệu — dữ liệu phải được Controller truyền vào. Cơ chế này gọi là passing model to View. Sau khi có model, View dùng biểu thức Razor ngầm (implicit Razor expression) để xuất giá trị ra HTML.

Bước 1 — Controller truyền model

Dùng overload View(object model) của phương thức View():

Controllers/DepartmentsController.cscsharp
public class DepartmentsController : Controller
{
    [HttpGet]
    public IActionResult Details(int id)
    {
        var department = DepartmentsRepository.GetDepartmentById(id);
        if (department == null)
        {
            return Content("<h3 style='color:red'>Department not found.</h3>");
        }

        return View(department); // truyền object Department vào View
    }
}

Bước 2 — View khai báo kiểu model với @model

Ở đầu file .cshtml, dùng directive @model để khai báo kiểu dữ liệu nhận được. Sau đó dùng thuộc tính Model (viết hoa) để truy cập dữ liệu.

Views/Departments/Details.cshtmlhtml
@using WebApp.Models
@model Department

<html>
<body>
    <h1>Department</h1>
    <p>@Model.Id</p>
    <p>@Model.Name</p>
    <p>@Model.Description</p>
</body>
</html>

Biểu thức Razor ngầm (Implicit Expression)

Cú pháp @<biểu_thức> (không có dấu ngoặc đơn) gọi là implicit Razor expression. Razor tự nhận ra kết thúc của biểu thức khi gặp khoảng trắng, dấu chấm câu, hoặc thẻ HTML.

RazorKết quả HTML
@Model.NameGiá trị của thuộc tính Name
@Model.IdGiá trị của thuộc tính Id
@department.Id (trong vòng lặp)Id của phần tử hiện tại
@DateTime.NowNgày giờ hiện tại

ℹ️Razor tự động HTML-encode giá trị xuất ra — ký tự đặc biệt như < > & sẽ được chuyển thành entity an toàn, ngăn chặn XSS. Muốn xuất HTML thô (không encode) phải dùng Html.Raw().

Truyền danh sách làm model

Controllers/DepartmentsController.cscsharp
[HttpGet]
public IActionResult Index()
{
    var departments = DepartmentsRepository.GetDepartments();
    return View(departments); // truyền List<Department>
}
Views/Departments/Index.cshtml (phiên bản đầu)html
@using WebApp.Models
@model List<Department>

<html>
<body>
    <h1>Department List</h1>
    <ul>
        @if (Model is not null && Model.Count > 0)
        {
            foreach (var department in Model)
            {
                <li><a href='/departments/details/@department.Id'>@department.Name - @department.Description</a></li>
            }
        }
    </ul>
    <br/>
    <a href='/departments/create'>Add Department</a>
</body>
</html>

4 Biểu thức Razor tường minh

Bài 4 · Explicit Razor Expression

Implicit expression hoạt động tốt khi biểu thức là một property đơn giản. Nhưng khi biểu thức phức tạp hơn — ví dụ phép tính, gọi constructor với tham số, hoặc nối chuỗi — Razor không thể tự xác định ranh giới. Lúc này dùng explicit Razor expression: @(biểu_thức).

Ví dụ: hiển thị ngày hiện tại

Views/Home/Index.cshtmlhtml
<html>
<body>
    <p>Hôm nay: @DateTime.Today</p>
</body>
</html>

Ví dụ: số ngày đến Giáng sinh (cần explicit)

Biểu thức new DateTime(...) - DateTime.Today có khoảng trắng và toán tử trừ — Razor sẽ nhầm ranh giới nếu dùng implicit. Ta cần bọc trong dấu ngoặc đơn:

Views/Home/Index.cshtmlhtml
<html>
<body>
    <p>Số ngày đến Giáng sinh:
        @(new DateTime(DateTime.Today.Year, 12, 25) - DateTime.Today)
    </p>
</body>
</html>

Khi nào dùng explicit vs implicit

Trường hợpDùngVí dụ
Property đơn giảnImplicit @@Model.Name
Gọi method không tham sốImplicit @@DateTime.Now
Phép tính, constructor, nối chuỗiExplicit @(...)@(a + b)
Biểu thức có khoảng trắng bên trongExplicit @(...)@(Model.Name ?? "N/A")

💡Khi không chắc dùng loại nào, cứ dùng explicit — luôn hoạt động đúng và code rõ ràng hơn.

5 Code block trong Razor

Bài 5 · Code Blocks

Khi cần khai báo biến, tính toán trước rồi mới dùng trong View, Razor cung cấp code block: cú pháp @{ ... }. Bên trong code block là C# thuần, Razor không render ra HTML. Biến khai báo trong code block có thể dùng ở bất kỳ đâu sau đó trong cùng View.

Cú pháp

Cú pháp cơ bảnhtml
@{
    // Code C# thuần — khai báo biến, tính toán, gọi hàm
    var color = "green";
    var message = $"Xin chào, {Model.Name}!";
}

<p style="color: @color">@message</p>

Ví dụ thực tế — tính màu và số ngày

Views/Home/Index.cshtmlhtml
@{
    var days = new DateTime(DateTime.Today.Year, 12, 25) - DateTime.Today;

    if (DateTime.Today > new DateTime(DateTime.Today.Year, 12, 25))
    {
        days = new DateTime(DateTime.Today.Year + 1, 12, 25) - DateTime.Today;
    }

    string color = days.Days % 2 == 0 ? "green" : "red";
}

@if (days.Days <= 10)
{
    <h1>Christmas is around the corner!</h1>
}
else
{
    switch (color)
    {
        case "green":
            <h1 style="color:green">Welcome home!</h1>
            break;
        case "red":
            <h1 style="color:red">Welcome home!</h1>
            break;
    }
}

ℹ️Code block @{ } thường đặt ở đầu file (sau @model) để khai báo tất cả biến cần dùng. Nội dung bên trong không bị render thành HTML — chỉ là C# thuần.

Viết HTML bên trong code block

Muốn xuất HTML từ bên trong code block, bao nội dung trong thẻ HTML bình thường — Razor tự nhận ra và render:

Ví dụhtml
@{
    for (int i = 1; i <= 3; i++)
    {
        <p>Dòng số @i</p>  @* HTML bên trong code block *@
    }
}

💡Comment trong Razor dùng cú pháp @* ... *@ — không bị render ra HTML, khác với comment HTML <!-- ... --> vẫn xuất hiện trong response.

6 Cấu trúc điều kiện

Bài 6 · Conditional Control Structure

Razor cho phép dùng các cấu trúc điều kiện C# (if/else, switch) ngay trong View để kiểm soát việc render phần tử HTML nào tùy điều kiện. Chỉ cần thêm @ trước từ khóa, phần thân bao gồm cả HTML và C# tự do xen kẽ nhau.

if / else

Views/Home/Index.cshtmlhtml
@{
    var days = new DateTime(DateTime.Today.Year, 12, 25) - DateTime.Today;
    if (DateTime.Today > new DateTime(DateTime.Today.Year, 12, 25))
    {
        days = new DateTime(DateTime.Today.Year + 1, 12, 25) - DateTime.Today;
    }
    string color = days.Days % 2 == 0 ? "green" : "red";
}

@if (days.Days <= 10)
{
    <h1>Christmas is around the corner!</h1>
}
else
{
    <h1 style="color: @color">Welcome home!</h1>
}

switch / case

Cú pháp @switch hoạt động như C# bình thường, nhưng bên trong mỗi case có thể viết HTML:

Ví dụ switch trong Razorhtml
@switch (color)
{
    case "green":
        <h1 style="color:green">Welcome home!</h1>
        break;
    case "red":
        <h1 style="color:red">Welcome home!</h1>
        break;
    default:
        <h1>Welcome!</h1>
        break;
}

🎯Điểm mạnh của Razor: bạn có thể xen kẽ HTML và C# tự nhiên bên trong cùng một khối lệnh điều kiện mà không cần ghép chuỗi hay escape thủ công.

Kiểm tra null với @if

Trước khi truy cập thuộc tính của model, luôn kiểm tra null để tránh NullReferenceException:

Kiểm tra null an toànhtml
@if (Model is not null)
{
    <p>@Model.Name</p>
}
else
{
    <p>Không có dữ liệu.</p>
}

7 Cấu trúc lặp

Bài 7 · Iterative Control Structure

Razor hỗ trợ đầy đủ các cấu trúc lặp của C#: for, foreach, while, do-while. Cú pháp: thêm @ trước từ khóa, phần thân xen HTML và C# tự nhiên.

foreach — dùng phổ biến nhất

Views/Departments/Index.cshtmlhtml
@using WebApp.Models
@model List<Department>

<html>
<body>
    <h1>Department List</h1>
    <ul>
        @if (Model is not null && Model.Count > 0)
        {
            foreach (var department in Model)
            {
                <li>
                    <a href='/departments/details/@department.Id'>
                        @department.Name - @department.Description
                    </a>
                </li>
            }
        }
    </ul>
    <br/>
    <a href='/departments/create'>Add Department</a>
</body>
</html>

for — khi cần chỉ số

Ví dụ for với chỉ sốhtml
@for (int i = 0; i < Model.Count; i++)
{
    <p>@(i + 1). @Model[i].Name</p>
}

while — khi điều kiện động

Ví dụ whilehtml
@{
    int i = 0;
}
@while (i < Model.Count)
{
    <p>@Model[i].Name</p>
    i++;
}

⚠️Khi dùng @foreach hoặc @for bên trong HTML, Razor tự nhận biết phần HTML bên trong phần thân vòng lặp mà không cần thêm @ cho từng thẻ. Chỉ dùng @ khi muốn xuất giá trị C# (@department.Name).

    {

8 Bài tập 7 — Error View

Bài 8 · Assignment 7 – Error View

Sau khi học cách tạo View, đây là bài tập thực hành đầu tiên.

Yêu cầu

Trong DepartmentsController, nhiều action method hiện đang trả về lỗi theo kiểu Content(html, "text/html") — HTML lỗi được ghép chuỗi trực tiếp trong Controller. Hãy:

  1. Tạo một Razor View tên Error.cshtml bên trong thư mục Views/Departments/.
  2. View nhận một danh sách chuỗi lỗi (List<string>) làm model.
  3. View hiển thị các lỗi dưới dạng danh sách HTML có màu đỏ.
  4. Controller thay thế tất cả chỗ trả về HTML lỗi thủ công bằng View("Error", errorList).

Phân tích các trường hợp lỗi trong Controller

Action methodTình huống lỗi
Details(int id)Không tìm thấy department theo id
Edit(Department)Model validation thất bại
Create(Department)Model validation thất bại
Delete(int id)Không tìm thấy department

💡Gợi ý: viết một private helper method GetErrors() trả về List<string> bằng cách duyệt ModelState.Values. Dùng method này cho tất cả các trường hợp validation lỗi.

9 Đáp án 7 — Error View

Bài 9 · Assignment 7 (Answer) – Error View

Error.cshtml — View hiển thị lỗi

Views/Departments/Error.cshtmlhtml
@model List<string>

<html>
<head>
    <title>Error</title>
</head>
<body>
    <h1 style="color:red">Errors:</h1>
    <ul>
        @if (Model is not null && Model.Count > 0)
        {
            foreach (var errorMessage in Model)
            {
                <li>@errorMessage</li>
            }
        }
    </ul>
</body>
</html>

Controller sau khi refactor

Controllers/DepartmentsController.cscsharp
public class DepartmentsController : Controller
{
    [HttpGet]
    public IActionResult Details(int id)
    {
        var department = DepartmentsRepository.GetDepartmentById(id);
        if (department == null)
        {
            // Thay thế Content(html) bằng View("Error", list)
            return View("Error", new List<string>() { "Department not found." });
        }
        return View(department);
    }

    [HttpPost]
    public IActionResult Edit(Department department)
    {
        if (!ModelState.IsValid)
        {
            return View("Error", GetErrors());
        }
        DepartmentsRepository.UpdateDepartment(department);
        return RedirectToAction(nameof(Index));
    }

    [HttpPost]
    public IActionResult Create(Department department)
    {
        if (!ModelState.IsValid)
        {
            return View("Error", GetErrors());
        }
        DepartmentsRepository.AddDepartment(department);
        return RedirectToAction(nameof(Index));
    }

    [HttpPost]
    public IActionResult Delete(int id)
    {
        var department = DepartmentsRepository.GetDepartmentById(id);
        if (department == null)
        {
            ModelState.AddModelError("id", "Department not found.");
            return View("Error", GetErrors());
        }
        DepartmentsRepository.DeleteDepartment(department);
        return RedirectToAction(nameof(Index));
    }

    // Helper: trích xuất lỗi từ ModelState
    private List<string> GetErrors()
    {
        var errors = new List<string>();
        foreach (var value in ModelState.Values)
        {
            foreach (var error in value.Errors)
            {
                errors.Add(error.ErrorMessage);
            }
        }
        return errors;
    }
}

🎯View("Error", model) chỉ định tên View tường minh. Khi bạn truyền tên như vậy, ASP.NET Core tìm Views/Departments/Error.cshtml trước, sau đó Views/Shared/Error.cshtml. Không cần phải View cùng tên với action method.

ℹ️Lưu ý cách truyền model: View("Error", GetErrors()) — tham số thứ nhất là tên View (chuỗi), tham số thứ hai là model object. Không nhầm với View(model) chỉ có một tham số.

10 Razor Literal

Bài 10 · Razor Literal

Vấn đề: bên trong @if hay @{ }, nếu muốn xuất văn bản thuần (không phải thẻ HTML), Razor sẽ coi nó là code C# và báo lỗi. Để xuất chuỗi văn bản từ trong code block hoặc cấu trúc điều kiện, dùng Razor literal.

Cách 1 — Bọc trong thẻ HTML

Giải pháp đơn giản nhất: bọc nội dung trong bất kỳ thẻ HTML nào (thường dùng <text> nếu không muốn thêm thẻ):

Bọc trong thẻ HTML thựchtml
@if (days.Days <= 10)
{
    <p>Còn @days.Days ngày nữa đến Giáng sinh!</p>
}

Cách 2 — Thẻ <text> (Razor-only, không render ra HTML)

Thẻ <text> là thẻ đặc biệt của Razor — nó không xuất hiện trong HTML cuối cùng, chỉ dùng để báo Razor "phần này là nội dung văn bản, không phải code":

Dùng <text>html
@if (days.Days <= 10)
{
    <text>Còn @days.Days ngày!</text>
}

Cách 3 — @: (inline text)

Dùng @: để xuất một dòng văn bản thuần từ trong code block:

Dùng @: inline texthtml
@if (days.Days <= 10)
{
    @:Còn @days.Days ngày nữa đến Giáng sinh!
}

Ví dụ đầy đủ từ dự án

Views/Home/Index.cshtmlhtml
@{
    var days = new DateTime(DateTime.Today.Year, 12, 25) - DateTime.Today;
    if (DateTime.Today > new DateTime(DateTime.Today.Year, 12, 25))
    {
        days = new DateTime(DateTime.Today.Year + 1, 12, 25) - DateTime.Today;
    }
    string color = days.Days % 2 == 0 ? "green" : "red";
}

11 ViewData và ViewBag

Bài 11 · ViewData and ViewBag

Ngoài cách truyền model qua View(model), ASP.NET Core còn cung cấp hai cơ chế bổ sung để truyền dữ liệu từ Controller sang View: ViewData và ViewBag.

Tại sao cần ViewData/ViewBag

Đôi khi bạn cần truyền thêm thông tin phụ (ví dụ: tiêu đề trang, thông báo trạng thái) mà không muốn sửa class Model để thêm thuộc tính. ViewData/ViewBag là giải pháp nhanh cho trường hợp này.

ViewData — dictionary kiểu động

ViewData là ViewDataDictionary, một dictionary với key là string và value là object?. Cần ép kiểu khi đọc trong View.

Controller — gán ViewDatacsharp
[HttpGet]
public IActionResult Details(int id)
{
    ViewData["PageTitle"] = "Chi tiết phòng ban";
    ViewData["ShowTimestamp"] = true;

    var department = DepartmentsRepository.GetDepartmentById(id);
    return View(department);
}
View — đọc ViewDatahtml
@model WebApp.Models.Department

<html>
<head>
    <title>@ViewData["PageTitle"]</title>
</head>
<body>
    <h1>@ViewData["PageTitle"]</h1>
    <p>@Model.Name</p>
    @if ((bool?)ViewData["ShowTimestamp"] == true)
    {
        <p>Xem lúc: @DateTime.Now</p>
    }
</body>
</html>

ViewBag — wrapper dynamic của ViewData

ViewBag là một dynamic wrapper bọc quanh ViewData. Không cần ép kiểu, dùng cú pháp property trực tiếp. Bên dưới, ViewBag.PageTitle và ViewData["PageTitle"] cùng trỏ đến một giá trị.

Controller — gán ViewBagcsharp
[HttpGet]
public IActionResult Index()
{
    ViewBag.PageTitle = "Danh sách phòng ban";
    ViewBag.TotalCount = DepartmentsRepository.GetDepartments().Count;

    var departments = DepartmentsRepository.GetDepartments();
    return View(departments);
}
View — đọc ViewBaghtml
@model List<WebApp.Models.Department>

<html>
<body>
    <h1>@ViewBag.PageTitle</h1>
    <p>Tổng số: @ViewBag.TotalCount phòng ban</p>
    <ul>
        @foreach (var dept in Model)
        {
            <li>@dept.Name</li>
        }
    </ul>
</body>
</html>

So sánh ba cách truyền dữ liệu

CáchType-safetyIntelliSenseKhi nào dùng
Model (strongly typed)CóĐầy đủDữ liệu chính của View — luôn ưu tiên
ViewDataKhông (object)KhôngMetadata phụ, cần ép kiểu khi đọc
ViewBagKhông (dynamic)KhôngMetadata phụ, cú pháp ngắn hơn ViewData

⚠️ViewData và ViewBag là weakly typed — lỗi tên key hay sai kiểu chỉ phát hiện lúc runtime, không compile-time. Chỉ dùng khi thực sự cần thiết; ưu tiên strongly-typed model cho dữ liệu chính của View.

Bạn có sẵn project DepartsCRUD với DepartmentsRepository và DepartmentsController đang hoạt động. Vấn đề hiện tại: HTML được ghép chuỗi trong từng action method. File Error.cshtml đã được tạo ở bài 9.

Model cần dùng

Models/Department.cscsharp
using System.ComponentModel.DataAnnotations;

namespace WebApp.Models
{
    public class Department
    {
        public Department() { }

        public Department(int id, string name, string? description = "")
        {
            Id = id;
            Name = name;
            Description = description;
        }

        public int Id { get; set; }

        [Required]
        public string? Name { get; set; }

        [StringLength(500)]
        public string? Description { get; set; }
    }
}

Controller — action Details và Edit

Controllers/DepartmentsController.cscsharp
[HttpGet]
public IActionResult Details(int id)
{
    var department = DepartmentsRepository.GetDepartmentById(id);
    if (department == null)
    {
        return View("Error", new List<string>() { "Department not found." });
    }
    return View(department); // truyền Department vào Details.cshtml
}

[HttpPost]
public IActionResult Edit(Department department)
{
    if (!ModelState.IsValid)
    {
        return View("Error", GetErrors());
    }
    DepartmentsRepository.UpdateDepartment(department);
    return RedirectToAction(nameof(Index));
}

🎯Lưu ý quan trọng: trường <input type='hidden' name='Id' value='@Model.Id' /> là bắt buộc. Khi form POST lên /departments/edit, ASP.NET Core dùng Model Binding để điền vào object Department — nếu không có hidden field, Id sẽ bằng 0 và Update sẽ thất bại.
