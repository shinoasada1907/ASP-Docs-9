Course Project — Quản lý Nhân viên với Razor Pages

Dự án thực hành xây dựng ứng dụng quản lý nhân viên hoàn chỉnh (full CRUD) bằng Razor Pages — từ thiết lập cấu trúc dự án, tổ chức model/repository, đến tích hợp ViewComponent, Partial View, tìm kiếm động bằng JavaScript và xác nhận xóa an toàn.

1 Khởi động dự án

Bài 1 · Start working on the project

Dự án thực hành này xây dựng trên nền tảng dự án Department đã có từ các chương trước. Mục tiêu là bổ sung đầy đủ tính năng CRUD (Create, Read, Update, Delete) cho nhân viên (Employee) bằng cách sử dụng Razor Pages — trong khi phần Department vẫn dùng MVC Controller truyền thống.

2 Cập nhật model Employee & Repository

Bài 2 · Updated Employee and Repository classes

Trước khi bắt đầu xây dựng các trang Razor Pages, cần cập nhật class Employee để phản ánh mối quan hệ với Department, và bổ sung đầy đủ phương thức CRUD vào EmployeesRepository.

Model Employee — quan hệ với Department

Class Employee có thêm hai thuộc tính mới: DepartmentId (khóa ngoại) và Department (navigation property — giúp truy cập thông tin phòng ban liên quan mà không cần join thủ công).

Model/Employee.cscsharp
using System.ComponentModel.DataAnnotations;

namespace WebApp.Models
{
    public class Employee
    {
        public int Id { get; set; }

        [Required]
        public string? Name { get; set; }

        [Required]
        public string? Position { get; set; }

        public double? Salary { get; set; }

        // Khóa ngoại liên kết tới Department
        public int DepartmentId { get; set; }

        // Navigation property — truy cập thông tin phòng ban liên quan
        public Department? Department { get; set; }

        public Employee() { }

        public Employee(int id, string name, string position, double salary, int departmentId)
        {
            Id = id;
            Name = name;
            Position = position;
            Salary = salary;
            DepartmentId = departmentId;
        }
    }
}

EmployeesRepository — In-memory CRUD

Repository sử dụng danh sách tĩnh (static List<Employee>) để lưu dữ liệu trong bộ nhớ, cung cấp đầy đủ các phương thức: lấy danh sách (có lọc), lấy theo ID, thêm, cập nhật, và xóa.

Model/EmployeesRepository.cscsharp
namespace WebApp.Models
{
    public static class EmployeesRepository
    {
        private static List<Employee> _employees = new List<Employee>
        {
            new Employee(1, "John Doe",       "Engineer",    60000, 1),
            new Employee(2, "Jane Smith",     "Manager",     75000, 1),
            new Employee(3, "Sam Brown",      "Technician",  50000, 1),
            new Employee(4, "Alice Johnson",  "Analyst",     55000, 2),
            new Employee(5, "Bob Lee",        "Developer",   65000, 2),
            // ... thêm nhân viên khác
        };

        public static List<Employee> GetEmployees(string? filter = null, int? departmentId = null)
        {
            // Gắn Department navigation property cho mỗi nhân viên
            foreach (Employee emp in _employees)
                emp.Department = DepartmentsRepository.GetDepartmentById(emp.DepartmentId);

            if (departmentId.HasValue)
                return _employees.Where(x => x.DepartmentId == departmentId.Value).ToList();

            if (!string.IsNullOrWhiteSpace(filter))
                return _employees.Where(x => x.Name is not null
                    && x.Name.ToLower().Contains(filter.ToLower())).ToList();

            return _employees;
        }

        public static Employee? GetEmployeeById(int id)
            => _employees.FirstOrDefault(x => x.Id == id);

        public static void AddEmployee(Employee? employee)
        {
            if (employee is not null)
            {
                employee.Id = _employees.Max(x => x.Id) + 1;
                _employees.Add(employee);
            }
        }

        public static bool UpdateEmployee(Employee? employee)
        {
            if (employee is not null)
            {
                var emp = _employees.FirstOrDefault(x => x.Id == employee.Id);
                if (emp is not null)
                {
                    emp.Name = employee.Name;
                    emp.Position = employee.Position;
                    emp.Salary = employee.Salary;
                    emp.DepartmentId = employee.DepartmentId;
                    return true;
                }
            }
            return false;
        }

        public static bool DeleteEmployee(Employee? employee)
        {
            if (employee is not null)
            {
                _employees.Remove(employee);
                return true;
            }
            return false;
        }
    }
}

ℹ️Trong dự án thực tế, repository sẽ sử dụng Entity Framework Core để tương tác với database. Ở đây dùng danh sách tĩnh trong bộ nhớ để tập trung vào việc học Razor Pages mà không phụ thuộc database.

3 Thêm hỗ trợ Razor Pages

Bài 3 · Add Razor Pages support

Để Razor Pages hoạt động trong một ứng dụng ASP.NET Core đã có MVC, cần thực hiện hai thay đổi trong Program.cs: đăng ký dịch vụ Razor Pages và ánh xạ endpoint Razor Pages vào pipeline.

Program.cscsharp
var builder = WebApplication.CreateBuilder(args);

// Đăng ký cả MVC Controllers+Views lẫn Razor Pages
builder.Services.AddControllersWithViews();
builder.Services.AddRazorPages();              // ← THÊM MỚI

var app = builder.Build();

app.UseRouting();

app.UseEndpoints(endpoints =>
{
    // Endpoint cho MVC Controller (Department)
    endpoints.MapControllerRoute(
        name: "default",
        pattern: "{controller=Home}/{action=Index}/{id?}"
    );

    // Endpoint cho Razor Pages (Employee)
    endpoints.MapRazorPages();                 // ← THÊM MỚI
});

app.Run();

Cấu hình _ViewStart.cshtml cho Razor Pages

Tạo file Pages/_ViewStart.cshtml để chỉ định layout mặc định cho tất cả Razor Pages. Đây là file tương đương với Views/_ViewStart.cshtml trong MVC — và cả hai có thể trỏ đến cùng một file _Layout.cshtml.

Pages/_ViewStart.cshtmlrazor
@{
    Layout = "_Layout";
}

Cấu hình _ViewImports.cshtml cho Razor Pages

File Pages/_ViewImports.cshtml khai báo namespace dùng chung cho tất cả Razor Pages, tránh phải lặp lại @using ở mỗi trang.

Pages/_ViewImports.cshtmlrazor
@using WebApp.Models
@using WebApp.ViewModels

💡Dùng chung Layout: File _Layout.cshtml trong Views/Shared/ có thể được dùng chung cho cả MVC lẫn Razor Pages. ASP.NET Core tìm kiếm layout theo thứ tự: thư mục hiện tại → Shared/ → Pages/Shared/ → Views/Shared/.

Page Model — Index.cshtml.cs

Page Model kế thừa PageModel. Phương thức OnGet() xử lý GET request, còn OnGetSearchEmployeeResult() là Named Handler — được gọi khi có query string ?handler=SearchEmployeeResult — trả về kết quả ViewComponent dạng HTML fragment.

Pages/Employees/Index.cshtml.cscsharp
using Microsoft.AspNetCore.Mvc;
using Microsoft.AspNetCore.Mvc.RazorPages;
using WebApp.Models;

namespace WebApp.Pages.Employees
{
    public class IndexModel : PageModel
    {
        public List<Employee>? Employees { get; set; }

        public void OnGet()
        {
            // Không tải trực tiếp — danh sách được tải bằng AJAX
        }

        // Named Handler: GET /employees?filter=...&handler=SearchEmployeeResult
        public IActionResult OnGetSearchEmployeeResult(string? filter)
        {
            return ViewComponent("EmployeeList", new { filter });
        }
    }
}

🎯Named Handler trong Razor Pages: Thay vì dùng nhiều Action Method như MVC, Razor Pages dùng quy ước đặt tên OnGet[HandlerName] / OnPost[HandlerName]. Gọi bằng query string ?handler=HandlerName hoặc attribute asp-page-handler trên form/link.

5 ViewComponent & Partial View cho danh sách nhân viên

Bài 5 · Componentize with ViewComponent and PartialView

Để tái sử dụng và hỗ trợ render động (AJAX), danh sách nhân viên được tách thành một ViewComponent và một Partial View. Cách tổ chức này giống hệt những gì đã làm với Department trong MVC — điểm khác biệt là thư mục đặt ViewComponent nằm trong Pages/Shared/ thay vì Views/Shared/.

Cấu trúc thư mục ViewComponent

Cấu trúc thư mụctext
Pages/
└── Shared/
    └── Components/
        └── EmployeeList/
            ├── EmployeeListViewComponent.cs   ← Logic
            └── Default.cshtml                 ← View của component

EmployeeListViewComponent

ViewComponent kế thừa ViewComponent, phương thức Invoke() nhận các tham số lọc và trả về View với danh sách nhân viên đã lọc.

Pages/Shared/Components/EmployeeList/EmployeeListViewComponent.cscsharp
using Microsoft.AspNetCore.Mvc;
using WebApp.Models;

namespace WebApp.Pages.Shared.Components.EmployeeList
{
    public class EmployeeListViewComponent : ViewComponent
    {
        public IViewComponentResult Invoke(string? filter, int? departmentId)
        {
            return View(EmployeesRepository.GetEmployees(filter, departmentId));
        }
    }
}

ℹ️Partial View vs ViewComponent: Partial View phù hợp khi chỉ cần render một đoạn HTML đơn giản từ model có sẵn. ViewComponent phù hợp khi cần logic riêng (truy vấn dữ liệu, lọc, xử lý) trước khi render — đây là lý do EmployeeList được triển khai dưới dạng ViewComponent.

6 Thêm tính năng tìm kiếm động

Bài 6 · Add interactivity

Sau khi tách danh sách nhân viên thành ViewComponent, thanh tìm kiếm cần được kết nối lại để hoạt động: khi người dùng nhấn nút Search, JavaScript sẽ gọi Named Handler OnGetSearchEmployeeResult qua fetch(), nhận về HTML fragment và cập nhật phần <tbody> mà không tải lại toàn bộ trang.

Vấn đề với Section và Script trong Layout

Khi đặt JavaScript vào @section scripts, script này được render trong _Layout.cshtml sau khi trang đã load xong. Đây là thực hành đúng để đảm bảo tất cả DOM element đã tồn tại trước khi script chạy.

Pages/Employees/Index.cshtml — phần scriptsrazor
@section scripts
{
    <script>
        // Tải danh sách khi trang khởi động
        loadEmployees('');

        // Gắn sự kiện click cho nút Search
        document.querySelector("#search-button").addEventListener("click", async () => {
            var filter = document.querySelector("#search-filter").value;
            loadEmployees(filter);
        });

        // Hàm gọi Named Handler và cập nhật tbody
        async function loadEmployees(filter) {
            var response = await fetch(`/employees?filter=${filter}&handler=SearchEmployeeResult`);
            var html = await response.text();
            document.querySelector("#employee-list").innerHTML = html;
        }
    </script>
}

Luồng hoạt động tìm kiếm

  1. Trang load lần đầu → JavaScript gọi loadEmployees('') → fetch /employees?filter=&handler=SearchEmployeeResult.
  2. Server nhận request GET với handler=SearchEmployeeResult → gọi OnGetSearchEmployeeResult(filter).
  3. Named Handler trả về ViewComponent("EmployeeList", new { filter }) — chỉ là HTML của các <tr>.
  4. JavaScript lấy HTML fragment và đặt vào #employee-list (tbody).
  5. Người dùng nhập từ khóa, nhấn Search → lặp lại từ bước 1 với filter mới.

💡Pattern AJAX + ViewComponent: Đây là pattern rất phổ biến trong ASP.NET Core — thay vì tải lại toàn bộ trang, chỉ render một phần nhỏ (fragment) từ server và cập nhật DOM. Pattern này không cần framework frontend nặng như React/Angular.

Page Model

Thuộc tính DepartmentId được đánh dấu [BindProperty(SupportsGet = true)] để nhận giá trị từ route parameter. OnGet() dùng DepartmentId để tra cứu tên phòng ban.

Pages/Employees/DepartmentEmployees.cshtml.cscsharp
using Microsoft.AspNetCore.Mvc;
using Microsoft.AspNetCore.Mvc.RazorPages;
using WebApp.Models;

namespace WebApp.Pages.Employees
{
    public class DepartmentEmployeesModel : PageModel
    {
        public string? DepartmentName { get; set; }

        // SupportsGet = true cho phép bind từ GET request (route/query string)
        [BindProperty(SupportsGet = true)]
        public int? DepartmentId { get; set; }

        public void OnGet()
        {
            if (DepartmentId.HasValue)
            {
                var department = DepartmentsRepository.GetDepartmentById(DepartmentId.Value);
                DepartmentName = department?.Name;
            }
        }
    }
}

ℹ️[BindProperty(SupportsGet = true)]: Mặc định, [BindProperty] chỉ hoạt động với POST. Thêm SupportsGet = true để binding cũng hoạt động với GET request — hữu ích khi cần nhận tham số từ URL (route hoặc query string) vào thuộc tính của Page Model.

Pages/Employees/Create.cshtml.cscsharp
using Microsoft.AspNetCore.Mvc;
using Microsoft.AspNetCore.Mvc.RazorPages;
using WebApp.Helpers;
using WebApp.Models;
using WebApp.ViewModels;

namespace WebApp.Pages.Employees
{
    public class CreateModel : PageModel
    {
        // [BindProperty] cho phép binding dữ liệu POST vào thuộc tính này
        [BindProperty]
        public EmployeeViewModel? EmployeeViewModel { get; set; }

        public void OnGet()
        {
            EmployeeViewModel = new EmployeeViewModel
            {
                Employee = new Employee(),
                Departments = DepartmentsRepository.GetDepartments()
            };
        }

        public IActionResult OnPost()
        {
            if (!ModelState.IsValid)
            {
                var errors = ModelStateHelper.GetErrors(ModelState);
                return RedirectToPage("/Error", new { errors });
            }

            if (EmployeeViewModel?.Employee is not null)
                EmployeesRepository.AddEmployee(EmployeeViewModel.Employee);

            return RedirectToPage("Index");
        }
    }
}

🎯Antiforgery Token: @Html.AntiForgeryToken() tự động sinh một hidden field chứa token bảo mật. Khi form được submit, ASP.NET Core kiểm tra token này — nếu không khớp (hoặc thiếu), request bị từ chối với lỗi 400. Điều này ngăn chặn tấn công CSRF (Cross-Site Request Forgery). Razor Pages tự động xác thực antiforgery token mà không cần cấu hình thêm.

9 Trang báo lỗi (Error Razor Page)

Bài 9 · Error razor page

Khi validation thất bại trong Create hoặc Edit, ứng dụng chuyển hướng người dùng đến trang /Error kèm danh sách thông báo lỗi. Trang Error là một Razor Page đặt tại Pages/Error.cshtml.

Helper trích xuất lỗi từ ModelState

ModelStateHelper duyệt qua ModelStateDictionary và gom tất cả lỗi thành List<string> để truyền qua redirect.

Helpers/ModelStateHelper.cscsharp
using Microsoft.AspNetCore.Mvc.ModelBinding;

namespace WebApp.Helpers
{
    public static class ModelStateHelper
    {
        public static List<string> GetErrors(ModelStateDictionary modelState)
        {
            var errorMessages = new List<string>();
            foreach (var value in modelState.Values)
                foreach (var error in value.Errors)
                    errorMessages.Add(error.ErrorMessage);
            return errorMessages;
        }
    }
}

using Microsoft.AspNetCore.Mvc;
using Microsoft.AspNetCore.Mvc.RazorPages;
using WebApp.Helpers;
using WebApp.Models;
using WebApp.ViewModels;

namespace WebApp.Pages.Employees
{
    public class EditModel : PageModel
    {
        [BindProperty]
        public EmployeeViewModel? EmployeeViewModel { get; set; }

        // id được nhận từ route parameter "{id:int}"
        public void OnGet(int id)
        {
            EmployeeViewModel = new EmployeeViewModel
            {
                Employee    = EmployeesRepository.GetEmployeeById(id),
                Departments = DepartmentsRepository.GetDepartments()
            };
        }

        public IActionResult OnPost()
        {
            if (!ModelState.IsValid)
            {
                var errors = ModelStateHelper.GetErrors(ModelState);
                return RedirectToPage("/Error", new { errors });
            }

            if (EmployeeViewModel?.Employee is not null)
                EmployeesRepository.UpdateEmployee(EmployeeViewModel.Employee);

            return RedirectToPage("Index");
        }
    }
}

11 Xóa nhân viên

Bài 11 · Delete employee

Chức năng xóa nhân viên được tích hợp trực tiếp trong trang Edit. Thay vì tạo một trang riêng, một form DELETE được đặt bên dưới form Edit — khi người dùng nhấn nút Delete, form này được submit đến Named Handler OnPostDeleteEmplopyee.

Form Delete trong Edit.cshtml

Form Delete cần thuộc tính action trỏ đến đúng Named Handler. Lưu ý cú pháp: ?handler=DeleteEmplopyee (đúng với tên method OnPostDeleteEmplopyee).

Pages/Employees/Edit.cshtml — phần Delete formrazor

<form name="deleteEmployheeForm"
      method="post"
      action="/employees/edit/@Model.EmployeeViewModel.Employee.Id?handler=DeleteEmplopyee">
    @Html.AntiForgeryToken()
    <button type="submit" id="delete-employee" class="btn btn-danger">Delete</button>
</form>

Named Handler xóa trong EditModel

Named Handler OnPostDeleteEmplopyee nhận id từ route, tìm nhân viên, xóa và redirect về trang danh sách. Nếu không tìm thấy nhân viên, redirect đến trang Error.

Pages/Employees/Edit.cshtml.cs — OnPostDeleteEmplopyeecsharp
// Named Handler: POST /employees/edit/{id}?handler=DeleteEmplopyee
public IActionResult OnPostDeleteEmplopyee(int id)
{
    var employee = EmployeesRepository.GetEmployeeById(id);
    if (employee == null)
    {
        ModelState.AddModelError("id", "Employee not found");
        var errors = ModelStateHelper.GetErrors(ModelState);
        return RedirectToPage("/Error", new { errors });
    }

    EmployeesRepository.DeleteEmployee(employee);
    return RedirectToPage("Index");
}

ℹ️Tại sao dùng POST thay vì GET để xóa? Theo nguyên tắc HTTP, GET không được phép có tác dụng phụ (side effect) như xóa dữ liệu. Nếu dùng link GET để xóa, các crawler/bot và trình duyệt prefetch có thể vô tình xóa dữ liệu. Luôn dùng POST (hoặc DELETE) cho các thao tác thay đổi trạng thái.

12 Xác nhận xóa bằng JavaScript

Bài 12 · Delete confirmation with JavaScript

Xóa nhân viên là thao tác không thể hoàn tác. Để tránh người dùng lỡ tay xóa, một hộp thoại xác nhận JavaScript (confirm()) được hiển thị khi nhấn nút Delete. Chỉ khi người dùng xác nhận, form mới thực sự được submit.

Script xác nhận xóa

Script lắng nghe sự kiện click trên nút Delete, ngăn hành vi submit mặc định (event.preventDefault()), hiển thị hộp thoại confirm() với tên nhân viên, và chỉ submit form khi người dùng xác nhận.

Pages/Employees/Edit.cshtml — @section scriptsrazor
@section scripts
{
    <script>
        document.querySelector("#delete-employee").addEventListener("click", (event) => {
            // Ngăn form submit ngay lập tức
            event.preventDefault();

            // Lấy tên nhân viên từ server-side model (render lúc tải trang)
            const employeeName = '@Model.EmployeeViewModel.Employee.Name';

            // Hiển thị hộp thoại xác nhận
            const userConfirmation = confirm(
                `Are you sure you want to delete the employee: ${employeeName}`
            );

            // Chỉ submit nếu người dùng nhấn OK
            if (userConfirmation) {
                document.forms["deleteEmployheeForm"].submit();
            }
        });
    </script>
}

Luồng hoạt động xóa có xác nhận

  1. Người dùng nhấn nút Delete → JavaScript bắt sự kiện click, gọi event.preventDefault() — form chưa submit.
  2. Hộp thoại confirm() hiện ra với tên nhân viên.
  3. Nếu người dùng nhấn Cancel → userConfirmation = false, không làm gì thêm.
  4. Nếu người dùng nhấn OK → document.forms["deleteEmployheeForm"].submit() gửi POST request đến OnPostDeleteEmplopyee.
  5. Server xóa nhân viên và redirect về /Employees.

💡Nhúng giá trị Razor vào JavaScript: Cú pháp const employeeName = '@Model.EmployeeViewModel.Employee.Name' cho phép nhúng giá trị C# vào chuỗi JavaScript. Razor render giá trị này phía server khi tạo HTML, trình duyệt nhận được chuỗi JavaScript đã có giá trị thật. Lưu ý: cần escape nếu tên nhân viên có dấu nháy đơn.
