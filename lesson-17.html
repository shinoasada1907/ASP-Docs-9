MVC & Razor Pages — Dependency Injection

Constructor injection trong controller và page model, view injection (@inject). Học cách đăng ký service vào DI container và để ASP.NET Core tự động tiêm phụ thuộc khi khởi tạo controller, Razor Page model và ViewComponent.

1 Chuẩn bị: tách interface & chuyển repository về instance

Bài 1 · Constructor Injection 1 — Preparing Repositories

Trước khi áp dụng Dependency Injection (DI), các repository trong dự án dùng phương thức static — nghĩa là dữ liệu và logic được gọi trực tiếp mà không qua instance. DI yêu cầu chúng ta làm việc với interface và đối tượng instance, do đó bước đầu tiên là tái cấu trúc repository.

Tại sao cần interface?

DI container của ASP.NET Core hoạt động theo nguyên tắc ánh xạ interface → implementation. Khi một class khai báo phụ thuộc vào IDepartmentsRepository, container sẽ tra cứu xem interface đó được đăng ký với implementation nào, sau đó tạo instance và truyền vào. Điều này giúp code phụ thuộc vào hợp đồng (contract) thay vì implementation cụ thể, dễ dàng thay thế hoặc mock khi viết unit test.

Định nghĩa interface cho repository

Visual Studio hỗ trợ tính năng Extract Interface (nhấn Ctrl+. trên tên class → Extract interface) để tạo interface tự động. Kết quả cuối cùng như sau:

Model/IDepartmentsRepository.cscsharp
namespace WebApp.Models
{
    public interface IDepartmentsRepository
    {
        void AddDepartment(Department? Department);
        bool DeleteDepartment(Department? Department);
        Department? GetDepartmentById(int id);
        List<Department> GetDepartments(string? filter = null);
        bool UpdateDepartment(Department? Department);
    }
}
Model/IEmployeesRepository.cscsharp
namespace WebApp.Models
{
    public interface IEmployeesRepository
    {
        void AddEmployee(Employee? employee);
        bool DeleteEmployee(Employee? employee);
        Employee? GetEmployeeById(int id);
        List<Employee> GetEmployees(string? filter = null, int? departmentId = null);
        bool UpdateEmployee(Employee? employee);
    }
}

Implementation: DepartmentsRepository

Sau khi có interface, class DepartmentsRepository implement nó. Lưu ý các phương thức không còn là static — chúng là instance method bình thường:

Model/DepartmentsRepository.cscsharp
namespace WebApp.Models
{
    public class DepartmentsRepository : IDepartmentsRepository
    {
        private List<Department> _departments = new List<Department>
        {
            new Department(1, "Sales", "Sales Department"),
            new Department(2, "Engineering", "Engineering Department"),
            new Department(3, "QA", "Quality Assurance")
        };

        public List<Department> GetDepartments(string? filter = null)
        {
            if (string.IsNullOrWhiteSpace(filter)) return _departments;
            return _departments
                .Where(x => x.Name != null && x.Name.ToLower().Contains(filter.ToLower()))
                .ToList();
        }

        public Department? GetDepartmentById(int id)
            => _departments.FirstOrDefault(x => x.Id == id);

        public void AddDepartment(Department? department)
        {
            if (department is not null)
            {
                department.Id = _departments.Max(x => x.Id) + 1;
                _departments.Add(department);
            }
        }

        public bool UpdateDepartment(Department? department)
        {
            if (department is not null)
            {
                var existing = _departments.FirstOrDefault(x => x.Id == department.Id);
                if (existing is not null)
                {
                    existing.Name = department.Name;
                    existing.Description = department.Description;
                    return true;
                }
            }
            return false;
        }

        public bool DeleteDepartment(Department? department)
        {
            if (department is not null) { _departments.Remove(department); return true; }
            return false;
        }
    }
}

EmployeesRepository phụ thuộc vào DepartmentsRepository

Repository nhân viên cần biết phòng ban để gắn thông tin Department vào mỗi Employee. Đây là ví dụ điển hình của service phụ thuộc vào service khác: EmployeesRepository nhận IDepartmentsRepository qua constructor — DI container sẽ tự giải quyết chuỗi phụ thuộc này.

Model/EmployeesRepository.cs (trích)csharp
namespace WebApp.Models
{
    public class EmployeesRepository : IEmployeesRepository
    {
        private readonly IDepartmentsRepository departmentsRepository;

        // DI container sẽ tự inject IDepartmentsRepository khi tạo EmployeesRepository
        public EmployeesRepository(IDepartmentsRepository departmentsRepository)
        {
            this.departmentsRepository = departmentsRepository;
        }

        private List<Employee> _employees = new List<Employee>
        {
            new Employee(1, "John Doe",       "Engineer",   60000, 1),
            new Employee(2, "Jane Smith",      "Manager",    75000, 1),
            new Employee(4, "Alice Johnson",   "Analyst",    55000, 2),
            new Employee(5, "Bob Lee",         "Developer",  65000, 2),
            new Employee(9, "Franklin Zhang",  "Architect",  80000, 3),
            // ...
        };

        public List<Employee> GetEmployees(string? filter = null, int? departmentId = null)
        {
            // Gắn thông tin phòng ban cho mỗi nhân viên
            foreach (var emp in _employees)
                emp.Department = departmentsRepository.GetDepartmentById(emp.DepartmentId);

            if (departmentId.HasValue)
                return _employees.Where(x => x.DepartmentId == departmentId.Value).ToList();

            if (!string.IsNullOrWhiteSpace(filter))
                return _employees.Where(x => x.Name != null
                    && x.Name.ToLower().Contains(filter.ToLower())).ToList();

            return _employees;
        }

        // ... các phương thức Add, Update, Delete tương tự
    }
}

ℹ️Tại sao dùng Singleton? Cả hai repository đang lưu dữ liệu in-memory trong danh sách private. Nếu đăng ký là Scoped hoặc Transient, mỗi request sẽ tạo instance mới và dữ liệu thêm/sửa/xóa sẽ bị mất ngay. Dùng Singleton đảm bảo cùng một instance tồn tại suốt vòng đời ứng dụng. Sau này khi chuyển sang database thật (EF Core), ta thường dùng Scoped.

2 Constructor Injection trong Controller, PageModel & ViewComponent

Bài 2 · Constructor Injection 2 — Controllers, Pages & ViewComponents

Sau khi đã có interface và implementation, bước tiếp theo là: đăng ký service vào DI container trong Program.cs, rồi khai báo phụ thuộc qua constructor ở từng class cần dùng.

Đăng ký service trong Program.cs

Mọi service phải được đăng ký vào builder.Services trước khi gọi builder.Build(). ASP.NET Core sẽ dùng thông tin đăng ký này để biết interface nào tương ứng với implementation nào.

Program.cscsharp
using WebApp.Models;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllersWithViews();
builder.Services.AddRazorPages();

// Đăng ký repository vào DI container dưới dạng Singleton
builder.Services.AddSingleton<IDepartmentsRepository, DepartmentsRepository>();
builder.Services.AddSingleton<IEmployeesRepository, EmployeesRepository>();

var app = builder.Build();

app.UseStaticFiles();
app.UseRouting();

app.UseEndpoints(endpoints =>
{
    endpoints.MapControllerRoute(
        name: "default",
        pattern: "{controller=Home}/{action=Index}/{id?}");

    endpoints.MapRazorPages();
});

app.Run();

🎯Cú pháp đăng ký: services.AddSingleton<TInterface, TImplementation>() — khai báo rằng bất cứ khi nào DI container thấy yêu cầu TInterface, nó sẽ cung cấp instance của TImplementation. Đây là bước bắt buộc; nếu thiếu, runtime sẽ ném ngoại lệ InvalidOperationException khi cố khởi tạo class có phụ thuộc chưa đăng ký.

Constructor Injection trong MVC Controller

Khi một HTTP request đến, ASP.NET Core routing xác định controller và action method cần thực thi, sau đó tạo instance controller. Lúc này framework kiểm tra constructor của controller, thấy tham số kiểu IDepartmentsRepository, tra DI container, lấy instance đã đăng ký và truyền vào. Toàn bộ quá trình này hoàn toàn tự động.

Controllers/DepartmentsController.cscsharp
using Microsoft.AspNetCore.Mvc;
using WebApp.Models;

namespace WebApp.Controllers
{
    public class DepartmentsController : Controller
    {
        // 1. Khai báo field readonly để lưu service được inject
        private readonly IDepartmentsRepository departmentsRepository;

        // 2. Constructor nhận service qua tham số — DI container tự truyền vào
        public DepartmentsController(IDepartmentsRepository departmentsRepository)
        {
            this.departmentsRepository = departmentsRepository;
        }

        [HttpGet]
        public IActionResult Index()
        {
            return View();
        }

        [Route("/department-list/{filter?}")]
        public IActionResult SearchDepartments(string? filter)
        {
            // 3. Dùng service đã inject như bình thường
            return ViewComponent("DepartmentList", new { filter });
        }

        [HttpGet]
        public IActionResult Details(int id)
        {
            var department = departmentsRepository.GetDepartmentById(id);
            if (department == null)
                return View("Error", new List<string> { "Department not found." });

            return View(department);
        }

        [HttpPost]
        public IActionResult Create(Department department)
        {
            if (!ModelState.IsValid)
                return View("Error", new List<string> { "Invalid data." });

            departmentsRepository.AddDepartment(department);
            return RedirectToAction(nameof(Index));
        }

        [HttpPost]
        public IActionResult Delete(int id)
        {
            var department = departmentsRepository.GetDepartmentById(id);
            if (department == null)
                return View("Error", new List<string> { "Department not found." });

            departmentsRepository.DeleteDepartment(department);
            return RedirectToAction(nameof(Index));
        }
    }
}

💡Shortcut Visual Studio: đặt con trỏ vào tên tham số constructor, nhấn Ctrl+. → Create and assign field. IDE sẽ tự tạo field private readonly và dòng gán trong constructor.

Constructor Injection trong Razor Pages PageModel

Razor Pages không có controller riêng — thay vào đó mỗi page có một Page Model (class kế thừa PageModel). Cơ chế inject hoàn toàn giống controller: ASP.NET Core tạo instance Page Model, nhìn vào constructor và inject các service đã đăng ký.

Page Create Employee cần cả hai service — một ví dụ inject nhiều phụ thuộc:

Pages/Employees/Create.cshtml.cscsharp
using Microsoft.AspNetCore.Mvc;
using Microsoft.AspNetCore.Mvc.RazorPages;
using WebApp.Models;
using WebApp.ViewModels;

namespace WebApp.Pages.Employees
{
    public class CreateModel : PageModel
    {
        private readonly IDepartmentsRepository departmentsRepository;
        private readonly IEmployeesRepository employeesRepository;

        // Inject nhiều service: DI container tự giải quyết thứ tự
        public CreateModel(
            IDepartmentsRepository departmentsRepository,
            IEmployeesRepository employeesRepository)
        {
            this.departmentsRepository = departmentsRepository;
            this.employeesRepository = employeesRepository;
        }

        [BindProperty]
        public EmployeeViewModel? EmployeeViewModel { get; set; }

        public void OnGet()
        {
            EmployeeViewModel = new EmployeeViewModel
            {
                Employee    = new Employee(),
                Departments = departmentsRepository.GetDepartments()
            };
        }

        public IActionResult OnPost()
        {
            if (!ModelState.IsValid)
                return RedirectToPage("/Error");

            if (EmployeeViewModel?.Employee is not null)
                employeesRepository.AddEmployee(EmployeeViewModel.Employee);

            return RedirectToPage("Index");
        }
    }
}

Page Edit Employee tương tự — cũng inject hai service và dùng chúng trong cả handler GET và POST:

Pages/Employees/Edit.cshtml.cs (trích)csharp
public class EditModel : PageModel
{
    private readonly IEmployeesRepository employeesRepository;
    private readonly IDepartmentsRepository departmentsRepository;

    public EditModel(
        IEmployeesRepository employeesRepository,
        IDepartmentsRepository departmentsRepository)
    {
        this.employeesRepository   = employeesRepository;
        this.departmentsRepository = departmentsRepository;
    }

    [BindProperty]
    public EmployeeViewModel? EmployeeViewModel { get; set; }

    public void OnGet(int id)
    {
        EmployeeViewModel = new EmployeeViewModel
        {
            Employee    = employeesRepository.GetEmployeeById(id),
            Departments = departmentsRepository.GetDepartments()
        };
    }

    public IActionResult OnPost()
    {
        if (!ModelState.IsValid) return RedirectToPage("/Error");
        if (EmployeeViewModel?.Employee is not null)
            employeesRepository.UpdateEmployee(EmployeeViewModel.Employee);
        return RedirectToPage("Index");
    }

    public IActionResult OnPostDeleteEmplopyee(int id)
    {
        var employee = employeesRepository.GetEmployeeById(id);
        if (employee == null) return RedirectToPage("/Error");
        employeesRepository.DeleteEmployee(employee);
        return RedirectToPage("Index");
    }
}

Constructor Injection trong ViewComponent

ViewComponent cũng là một class bình thường — nó cũng được DI container tạo ra và cũng hỗ trợ constructor injection. Đây là điểm mạnh của DI: cơ chế nhất quán cho mọi loại class trong ASP.NET Core.

Views/Shared/Components/DepartmentList/DepartmentListViewComponent.cscsharp
using Microsoft.AspNetCore.Mvc;
using WebApp.Models;

namespace WebApp.Views.Shared.Components.DepartmentList
{
    [ViewComponent]
    public class DepartmentListViewComponent : ViewComponent
    {
        private readonly IDepartmentsRepository departmentsRepository;

        public DepartmentListViewComponent(IDepartmentsRepository departmentsRepository)
        {
            this.departmentsRepository = departmentsRepository;
        }

        public IViewComponentResult Invoke(string? filter)
        {
            var departments = departmentsRepository.GetDepartments(filter);
            return View(departments);
        }
    }
}
Pages/Shared/Components/EmployeeList/EmployeeListViewComponent.cscsharp
using Microsoft.AspNetCore.Mvc;
using WebApp.Models;

namespace WebApp.Pages.Shared.Components.EmployeeList
{
    public class EmployeeListViewComponent : ViewComponent
    {
        private readonly IEmployeesRepository employeesRepository;

        public EmployeeListViewComponent(IEmployeesRepository employeesRepository)
        {
            this.employeesRepository = employeesRepository;
        }

        public IViewComponentResult Invoke(string? filter, int? departmentId)
        {
            return View(employeesRepository.GetEmployees(filter, departmentId));
        }
    }
}

Luồng hoạt động tổng thể

Sơ đồ dưới đây minh họa cách DI container giải quyết toàn bộ chuỗi phụ thuộc khi một HTTP request đến:

Bước Điều xảy ra
1. Request đến Routing xác định DepartmentsController hoặc Razor Page Model cần xử lý.
2. DI khởi tạo Framework yêu cầu DI container tạo instance. Container đọc constructor, thấy IDepartmentsRepository.
3. Giải quyết phụ thuộc Container tra bảng đăng ký, tìm thấy DepartmentsRepository là Singleton — trả về instance đã có (hoặc tạo mới lần đầu).
4. Chuỗi phụ thuộc EmployeesRepository cần IDepartmentsRepository — container tự giải quyết tiếp, không cần can thiệp thủ công.
5. Thực thi Action method / handler / Invoke() chạy với service đã sẵn sàng.

View Injection với @inject

Ngoài constructor injection, Razor Views còn hỗ trợ view injection bằng directive @inject. Kỹ thuật này phù hợp khi view cần một service nhỏ (ví dụ: helper tính toán, formatter) mà không muốn truyền qua ViewModel.

Views/Departments/Index.cshtml (ví dụ @inject)html
@* Inject service trực tiếp vào view *@
@inject WebApp.Models.IDepartmentsRepository DeptRepo

@{
    ViewData["Title"] = "Departments";
    // Có thể gọi service ngay trong view nếu cần
    var count = DeptRepo.GetDepartments().Count;
}

<h1>Danh sách phòng ban (@count phòng ban)</h1>

⚠️Khi nào KHÔNG dùng @inject: View injection phù hợp cho các service phụ trợ nhỏ (format, localization…). Tránh dùng @inject để gọi logic nghiệp vụ hoặc truy vấn dữ liệu phức tạp — những tác vụ đó nên nằm trong controller/page model để đảm bảo nguyên tắc Separation of Concerns.

Tổng kết điểm khác biệt: Minimal API vs MVC/Razor Pages

Đặc điểm Minimal API MVC / Razor Pages
Vị trí inject Tham số của lambda handler (method injection) Constructor của Controller / PageModel / ViewComponent
Scope Chỉ trong handler đó Dùng được trong toàn bộ class (mọi action method)
View injection Không áp dụng Hỗ trợ qua @inject trong .cshtml
Đăng ký service Giống nhau — builder.Services.AddXxx<TInterface, TImpl>()

🎯Nguyên tắc ghi nhớ: Luôn phụ thuộc vào interface, không phải class cụ thể. Đăng ký đúng lifetime (Singleton / Scoped / Transient). DI container sẽ lo phần còn lại — bạn chỉ cần khai báo "tôi cần cái này" trong constructor.
