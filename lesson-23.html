Web API — Documentation & Versioning

Tài liệu hóa Web API controller-based với OpenAPI/Swagger UI, bổ sung thông tin chi tiết cho endpoint, và quản lý nhiều phiên bản API bằng Asp.Versioning.Mvc.

1 Dùng OpenAPI để tài liệu hóa Web API

Bài 1 · Use OpenAPI to document Web API

Trong chương về Minimal API, chúng ta đã tài liệu hóa endpoint bằng OpenAPI tích hợp sẵn của .NET 9. Với Controller-based Web API, quy trình hoàn toàn tương tự — chỉ cần thêm một NuGet package bổ sung và khai báo HTTP verb trên action method.

Tạo project

Tạo project mới từ template ASP.NET Core Web API (hoặc dùng CLI), bỏ chọn tùy chọn "Use controllers" tự động để tự cấu hình thủ công cho dễ nhớ:

Terminalbash
dotnet new webapi -n LearnApiDocument --no-openapi
cd LearnApiDocument

Cài NuGet package

Package duy nhất cần thiết cho bước này là Microsoft.AspNetCore.OpenApi — đây là package OpenAPI tích hợp của .NET 9, đã có sẵn nếu bạn dùng template mặc định:

Terminalbash
dotnet add package Microsoft.AspNetCore.OpenApi

Cấu hình Program.cs

Đăng ký dịch vụ OpenAPI và map endpoint tài liệu. Endpoint /openapi/v1.json sẽ trả về JSON spec theo chuẩn OpenAPI 3:

Program.cscsharp
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllers();
builder.Services.AddOpenApi(); // đăng ký OpenAPI document mặc định (v1)

var app = builder.Build();

if (app.Environment.IsDevelopment())
{
    app.MapOpenApi(); // phục vụ /openapi/v1.json
}

app.MapControllers();
app.Run();

Controller mẫu

Để OpenAPI có thể khám phá endpoint, mỗi action method bắt buộc phải được trang trí bằng HTTP verb attribute ([HttpGet], [HttpPost], …). Nếu thiếu, endpoint sẽ không xuất hiện trong tài liệu:

Controllers/WeatherForecastController.cscsharp
[ApiController]
[Route("[controller]")]
public class WeatherForecastController : ControllerBase
{
    private static readonly string[] Summaries = new[]
    {
        "Freezing", "Bracing", "Chilly", "Cool", "Mild",
        "Warm", "Balmy", "Hot", "Sweltering", "Scorching"
    };

    [HttpGet]
    public IEnumerable<WeatherForecast> GetAll()
    {
        return Enumerable.Range(1, 5).Select(i => new WeatherForecast
        {
            Date      = DateOnly.FromDateTime(DateTime.Now.AddDays(i)),
            TemperatureC = Random.Shared.Next(-20, 55),
            Summary   = Summaries[Random.Shared.Next(Summaries.Length)]
        });
    }

    [HttpPut("{id}")]
    public IActionResult Update(int id) => NoContent();

    [HttpDelete("{id}")]
    public IActionResult Delete(int id) => NoContent();
}

Chạy ứng dụng và truy cập https://localhost:{port}/openapi/v1.json để xem OpenAPI JSON spec.

💡 So sánh với Minimal API: cách dùng AddOpenApi() / MapOpenApi() hoàn toàn giống nhau. Điểm khác biệt là với controller, bạn phải khai báo [HttpGet] / [HttpPost] … rõ ràng — framework mới biết đây là endpoint HTTP.

2 Dùng Swagger UI để hiển thị tài liệu

Bài 2 · Use SwaggerUI to visualize our document

File JSON OpenAPI hữu ích cho công cụ tự động, nhưng không thân thiện với người dùng. Swagger UI là giao diện web giúp hiển thị tài liệu dạng trực quan và cho phép gọi thử API ngay trên trình duyệt.

Cài Swashbuckle

Package Swashbuckle.AspNetCore là thư viện phổ biến nhất để tích hợp Swagger UI vào ASP.NET Core. Đây là package duy nhất cần thêm:

Terminalbash
dotnet add package Swashbuckle.AspNetCore

Cấu hình Swagger UI trong Program.cs

Swagger UI của Swashbuckle cần biết URL của OpenAPI JSON document để hiển thị. URL mặc định mà MapOpenApi() phục vụ là /openapi/v1.json:

Program.cscsharp
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllers();
builder.Services.AddOpenApi();

var app = builder.Build();

if (app.Environment.IsDevelopment())
{
    app.MapOpenApi(); // phục vụ /openapi/v1.json

    app.UseSwaggerUI(options =>
    {
        // trỏ Swagger UI đến OpenAPI document do .NET 9 sinh ra
        options.SwaggerEndpoint("/openapi/v1.json", "API V1");
    });
}

app.MapControllers();
app.Run();

ℹ️ Mặc định Swagger UI phục vụ tại /swagger. Chạy ứng dụng, mở http://localhost:{port}/swagger để thấy giao diện tương tác với danh sách tất cả endpoint.

Kết quả

Swagger UI hiển thị tất cả controller và action method, nhóm theo controller. Bạn có thể:

  • Xem HTTP method, URL pattern, và schema request/response.
  • Bấm Try it out → Execute để gọi trực tiếp API và xem response ngay trên giao diện.

🔑 Hai thành phần: Microsoft.AspNetCore.OpenApi (của Microsoft, tích hợp .NET 9) sinh ra OpenAPI JSON spec; Swashbuckle.AspNetCore (bên thứ ba) đọc spec đó và render Swagger UI. Chúng hoạt động độc lập nhưng phối hợp với nhau.

3 Bổ sung chi tiết cho tài liệu

Bài 3 · Provide more details to the document

Tài liệu OpenAPI mặc định chỉ liệt kê endpoint và HTTP method. Chúng ta có thể làm phong phú thêm bằng cách thêm: nhóm (tags), tóm tắt, mô tả, tên operation, và thông tin response type.

Nhóm endpoint bằng Tags

Trong controller-based API, dùng attribute [Tags] để gán nhóm cho action. Swagger UI hiển thị theo từng nhóm thay vì theo tên controller:

Controllers/WeatherForecastController.cscsharp
[ApiController]
[Route("[controller]")]
public class WeatherForecastController : ControllerBase
{
    [HttpGet]
    [Tags("Reading")]                         // nhóm "Reading"
    [EndpointSummary("Lấy tất cả dự báo thời tiết")]
    [EndpointDescription("Trả về danh sách dự báo thời tiết trong 5 ngày tới.")]
    [ProducesResponseType<IEnumerable<WeatherForecast>>(StatusCodes.Status200OK)]
    public IEnumerable<WeatherForecast> GetAll()
    {
        return Enumerable.Range(1, 5).Select(i => new WeatherForecast
        {
            Date         = DateOnly.FromDateTime(DateTime.Now.AddDays(i)),
            TemperatureC = Random.Shared.Next(-20, 55),
            Summary      = Summaries[Random.Shared.Next(Summaries.Length)]
        });
    }

    [HttpPut("{id}")]
    [Tags("Changing")]                        // nhóm "Changing"
    [EndpointName("UpdateForecast")]          // operation ID trong OpenAPI spec
    [ProducesResponseType(StatusCodes.Status204NoContent)]
    [ProducesResponseType<ProblemDetails>(StatusCodes.Status404NotFound)]
    public IActionResult Update(int id)
    {
        // giả lập: nếu id <= 0 thì không tìm thấy
        if (id <= 0) return NotFound();
        return NoContent();
    }

    [HttpDelete("{id}")]
    [Tags("Changing")]                        // cùng nhóm "Changing"
    [ProducesResponseType(StatusCodes.Status204NoContent)]
    [ProducesResponseType<ProblemDetails>(StatusCodes.Status404NotFound)]
    public IActionResult Delete(int id)
    {
        if (id <= 0) return NotFound();
        return NoContent();
    }
}

Giải thích các attribute

AttributeÝ nghĩa
[Tags("Tên")]Gán nhóm hiển thị trong Swagger UI. Nhiều action có thể cùng tag.
[EndpointSummary("...")]Dòng tóm tắt ngắn, hiển thị ngay bên cạnh tên endpoint.
[EndpointDescription("...")]Mô tả dài hơn, hiển thị khi mở rộng endpoint.
[EndpointName("...")]Đặt operationId trong OpenAPI spec (dùng cho client code generation).
[ProducesResponseType<T>(statusCode)]Khai báo kiểu dữ liệu và status code mà action có thể trả về. Giúp OpenAPI sinh schema response chính xác.

💡 [ProducesResponseType] rất quan trọng khi action có thể trả về nhiều status code khác nhau (200, 204, 404, …). Không khai báo thì OpenAPI không thể biết được schema response đầy đủ.

Dùng XML Comments

Ngoài attribute, bạn có thể dùng XML doc comments (<summary>, <param>, <returns>) để mô tả. Cần bật XML documentation trong file .csproj:

LearnApiDocument.csprojxml
<PropertyGroup>
  <GenerateDocumentationFile>true</GenerateDocumentationFile>
  <NoWarn>$(NoWarn);1591</NoWarn>
</PropertyGroup>
Controllers/WeatherForecastController.cscsharp
/// <summary>
/// Lấy tất cả dự báo thời tiết trong 5 ngày tới.
/// </summary>
/// <returns>Danh sách <see cref="WeatherForecast"/></returns>
[HttpGet]
[ProducesResponseType<IEnumerable<WeatherForecast>>(StatusCodes.Status200OK)]
public IEnumerable<WeatherForecast> GetAll() { /* ... */ }

⚠️ Khi dùng XML comments, Swashbuckle cần được cấu hình thêm để đọc file XML. Với Microsoft.AspNetCore.OpenApi thuần (.NET 9), XML comments được hỗ trợ tự động khi bật GenerateDocumentationFile.

4 Versioning cho Web API

Bài 4 · Web API Versioning

API Versioning cho phép duy trì nhiều phiên bản của cùng một endpoint song song, giúp client cũ không bị ảnh hưởng khi API thay đổi. Với controller-based API, cần cài hai NuGet package riêng (khác với Minimal API).

Cài NuGet packages

Terminalbash
dotnet add package Asp.Versioning.Mvc          # versioning cho controller
dotnet add package Asp.Versioning.Mvc.ApiExplorer # tích hợp với API Explorer

⚠️ Với Minimal API dùng Asp.Versioning.Http. Với Controller-based API phải dùng Asp.Versioning.Mvc — đây là điểm khác biệt then chốt.

Cấu trúc thư mục

Tổ chức controller theo thư mục version để rõ ràng:

Cấu trúc projecttext
Controllers/
  V1/
    WeatherForecastController.cs   // namespace ...V1
  V2/
    WeatherForecastController.cs   // namespace ...V2

Đánh dấu phiên bản trên Controller

Controllers/V1/WeatherForecastController.cscsharp
namespace LearnApiDocument.Controllers.V1;

[ApiController]
[Route("[controller]")]
[ApiVersion("1.0")]
public class WeatherForecastController : ControllerBase
{
    [HttpGet]
    public IEnumerable<WeatherForecast> GetAll()
    {
        // phiên bản 1: dự báo từ ngày mai, 5 ngày
        return Enumerable.Range(1, 5).Select(i => new WeatherForecast
        {
            Date         = DateOnly.FromDateTime(DateTime.Now.AddDays(i)),
            TemperatureC = Random.Shared.Next(-20, 55),
            Summary      = "V1 - " + Summaries[Random.Shared.Next(Summaries.Length)]
        });
    }
}
Controllers/V2/WeatherForecastController.cscsharp
namespace LearnApiDocument.Controllers.V2;

[ApiController]
[Route("[controller]")]
[ApiVersion("2.0")]
public class WeatherForecastController : ControllerBase
{
    [HttpGet]
    public IEnumerable<WeatherForecast> GetAll()
    {
        // phiên bản 2: dự báo từ ngày 6 trở đi
        return Enumerable.Range(6, 5).Select(i => new WeatherForecast
        {
            Date         = DateOnly.FromDateTime(DateTime.Now.AddDays(i)),
            TemperatureC = Random.Shared.Next(-20, 55),
            Summary      = "V2 - " + Summaries[Random.Shared.Next(Summaries.Length)]
        });
    }
}

Cấu hình versioning trong Program.cs

Program.cscsharp
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllers();
builder.Services.AddOpenApi();

builder.Services.AddApiVersioning(options =>
{
    options.AssumeDefaultVersionWhenUnspecified = true;
    options.DefaultApiVersion = new ApiVersion(1, 0);
    options.ReportApiVersions = true;
    // đọc version từ query string: ?api-version=1.0
    options.ApiVersionReader = new QueryStringApiVersionReader("api-version");
})
.AddMvc(); // BẮT BUỘC với controller — thiếu dòng này versioning không hoạt động

var app = builder.Build();

if (app.Environment.IsDevelopment())
{
    app.MapOpenApi();
    app.UseSwaggerUI(o => o.SwaggerEndpoint("/openapi/v1.json", "API V1"));
}

app.MapControllers();
app.Run();

🔑 Bắt buộc gọi .AddMvc() sau AddApiVersioning() khi dùng controller. Đây là điểm khác biệt duy nhất so với Minimal API. Thiếu lời gọi này, versioning sẽ hoàn toàn không hoạt động dù không có lỗi compile.

Kiểm tra

Gọi API với query string api-version:

Terminalbash
# Gọi phiên bản 1 — dự báo từ ngày mai
GET /WeatherForecast?api-version=1.0

# Gọi phiên bản 2 — dự báo từ ngày 6 trở đi
GET /WeatherForecast?api-version=2.0

ℹ️ Ngoài query string, bạn có thể dùng URL segment (/v1/weatherforecast) hoặc HTTP header để truyền version number, bằng cách thay QueryStringApiVersionReader bằng UrlSegmentApiVersionReader hoặc HeaderApiVersionReader.

5 Tài liệu hóa nhiều phiên bản

Bài 5 · Document multiple versions

Sau khi đã cấu hình versioning, bước tiếp theo là làm cho Swagger UI hiển thị tài liệu riêng biệt cho từng phiên bản API. Quy trình gồm ba phần: cấu hình nhiều OpenAPI document, tích hợp ApiExplorer, và gán group name cho controller.

Bước 1 — Cấu hình nhiều OpenAPI document

Mỗi phiên bản cần một OpenAPI document riêng, được phân biệt bằng documentName. Group name phải khớp với tên document:

Program.cscsharp
builder.Services.AddOpenApi("v1"); // document cho version 1
builder.Services.AddOpenApi("v2"); // document cho version 2

Bước 2 — Tích hợp ApiExplorer với versioning

Gọi .AddApiExplorer() để ApiExplorer nhận biết các phiên bản API đã đăng ký:

Program.cscsharp
builder.Services.AddApiVersioning(options =>
{
    options.AssumeDefaultVersionWhenUnspecified = true;
    options.DefaultApiVersion = new ApiVersion(1, 0);
    options.ReportApiVersions = true;
    options.ApiVersionReader = new QueryStringApiVersionReader("api-version");
})
.AddMvc()
.AddApiExplorer(options =>
{
    // đặt format tên group: 'v' + major version
    options.GroupNameFormat = "'v'VVV";
    options.SubstituteApiVersionInUrl = true;
});

Bước 3 — Gán group name cho Controller

Dùng attribute [ApiExplorerSettings(GroupName = "...")] để chỉ định controller này thuộc document nào. Group name phải khớp với tên document truyền vào AddOpenApi("v1"):

Controllers/V1/WeatherForecastController.cscsharp
[ApiController]
[Route("[controller]")]
[ApiVersion("1.0")]
[ApiExplorerSettings(GroupName = "v1")] // gán vào document "v1"
public class WeatherForecastController : ControllerBase
{
    // ...
}
Controllers/V2/WeatherForecastController.cscsharp
[ApiController]
[Route("[controller]")]
[ApiVersion("2.0")]
[ApiExplorerSettings(GroupName = "v2")] // gán vào document "v2"
public class WeatherForecastController : ControllerBase
{
    // ...
}

Bước 4 — Map OpenAPI endpoints và cấu hình Swagger UI nhiều version

Program.cs (hoàn chỉnh)csharp
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllers();
builder.Services.AddOpenApi("v1");
builder.Services.AddOpenApi("v2");

builder.Services.AddApiVersioning(options =>
{
    options.AssumeDefaultVersionWhenUnspecified = true;
    options.DefaultApiVersion = new ApiVersion(1, 0);
    options.ReportApiVersions = true;
    options.ApiVersionReader = new QueryStringApiVersionReader("api-version");
})
.AddMvc()
.AddApiExplorer(options =>
{
    options.GroupNameFormat = "'v'VVV";
    options.SubstituteApiVersionInUrl = true;
});

var app = builder.Build();

if (app.Environment.IsDevelopment())
{
    app.MapOpenApi("/openapi/{documentName}.json"); // pattern chứa {documentName}

    app.UseSwaggerUI(options =>
    {
        options.SwaggerEndpoint("/openapi/v1.json", "API V1");
        options.SwaggerEndpoint("/openapi/v2.json", "API V2");
    });
}

app.MapControllers();
app.Run();

Kết quả

Swagger UI sẽ có dropdown để chọn phiên bản. Mỗi phiên bản hiển thị tập endpoint tương ứng:

  • API V1: endpoint GET /WeatherForecast thuộc controller V1 (dự báo từ ngày mai).
  • API V2: endpoint GET /WeatherForecast thuộc controller V2 (dự báo từ ngày 6).

Gọi thử trong Swagger UI sẽ yêu cầu nhập api-version=1 hoặc api-version=2.

🔑 Ba điều không được quên khi tài liệu hóa nhiều phiên bản controller:

  1. Gọi .AddMvc() sau AddApiVersioning() — bắt buộc với controller.
  2. Gọi .AddApiExplorer() — để ApiExplorer nhận biết phiên bản.
  3. Đặt [ApiExplorerSettings(GroupName = "vN")] trên mỗi controller — để endpoint xuất hiện đúng document.

Thiếu bất kỳ bước nào, document tương ứng sẽ trống rỗng dù không có lỗi runtime.
