Tạo Web API với Controllers

Xây dựng Web API theo mô hình MVC với ControllerBase — attribute routing, token replacement, [ApiController], content negotiation, và so sánh với Minimal API.

1 MVC Pattern trong Web API

Bài 1 · MVC pattern in creating Web API with controllers

Khi xây dựng API bằng controller trong ASP.NET Core, mô hình MVC vẫn được áp dụng hoàn toàn — chỉ khác ở một điểm: phần View được thay thế bằng Output Formatter.

Sơ đồ luồng xử lý

Một request đến Web API đi qua pipeline như sau:

  1. Client gửi HTTP request → đi qua Middleware Pipeline.
  2. Endpoint Middleware chọn controller action phù hợp.
  3. Controller sử dụng service để lấy dữ liệu (từ database hoặc nguồn khác).
  4. Action method trả về một model object (thay vì View).
  5. Output Formatter định dạng model thành JSON/XML và ghi vào HTTP response.

ℹ️ Trong MVC truyền thống, bước 5 là Result Execution (render Razor View). Trong Web API, bước này được thay bằng Output Formatter — kết quả là một luồng bytes (JSON mặc định) thay vì HTML. Về cơ chế filter pipeline, mọi thứ hoàn toàn giống nhau.

Filter pipeline vẫn hoạt động đầy đủ

Web API với controller kế thừa toàn bộ filter pipeline của MVC: Authorization → Resource → Action → Exception → Result. Điều này cho phép tái sử dụng các filter đã viết cho MVC mà không cần thay đổi.

🎯Điểm mấu chốt: MVC pattern vẫn áp dụng ở cả cấp độ cao (Controller → Service → Model → Output Formatter) lẫn cấp độ thấp (filter pipeline). Hiểu điều này giúp bạn tận dụng kiến thức MVC để xây dựng API hiệu quả.

2 Tạo Project Web API bằng Visual Studio Template

Bài 2 · Create Web API with controllers using VS template

Visual Studio cung cấp template ASP.NET Core Web API để nhanh chóng khởi tạo project API với đầy đủ cấu trúc. Hiểu rõ những gì template tạo ra giúp bạn tự xây dựng từ đầu khi cần.

Cấu trúc project do template tạo ra

  • Program.cs — đăng ký services và cấu hình middleware pipeline.
  • Controllers/WeatherForecastController.cs — controller mẫu.
  • WeatherForecast.cs — model class (nên chuyển vào thư mục Models/).

💡 Template đặt WeatherForecast.cs ở thư mục gốc. Trong dự án thực tế, luôn tạo thư mục Models/ để tổ chức các model class — đặc biệt khi project có nhiều model.

Program.cs của Web API project

Program.cscsharp
var builder = WebApplication.CreateBuilder(args);

// Đăng ký controller services (không có View support)
builder.Services.AddControllers();

var app = builder.Build();

app.UseHttpsRedirection();

app.UseAuthorization();

// Map tất cả controller action thành endpoint
// Không dùng MapControllerRoute (conventional routing)
// mà dùng MapControllers (attribute routing)
app.MapControllers();

app.Run();

ControllerBase vs Controller

Web API controller kế thừa ControllerBase, không phải Controller:

Base classDùng choCó View support?
ControllerBase Web API (chỉ trả JSON/XML) Không — nhẹ hơn
Controller MVC (trả về View/Razor) Có — kế thừa ControllerBase + thêm View(), PartialView(), ViewData…

Controller mẫu

Controllers/WeatherForecastController.cscsharp
[ApiController]
[Route("[controller]")]
public class WeatherForecastController : ControllerBase
{
    private static readonly string[] Summaries = new[]
    {
        "Freezing", "Bracing", "Chilly", "Cool", "Mild",
        "Warm", "Balmy", "Hot", "Sweltering", "Scorching"
    };

    [HttpGet]
    public IEnumerable<WeatherForecast> Get()
    {
        return Enumerable.Range(1, 5).Select(index => new WeatherForecast
        {
            Date = DateOnly.FromDateTime(DateTime.Now.AddDays(index)),
            TemperatureC = Random.Shared.Next(-20, 55),
            Summary = Summaries[Random.Shared.Next(Summaries.Length)]
        });
    }
}

MapControllers vs MapControllerRoute

Phương thứcRouting styleDùng khi
app.MapControllers() Attribute routing Web API — route được chỉ định trực tiếp trên controller/action
app.MapControllerRoute() Conventional routing MVC — pattern {controller}/{action}/{id?}

3 Attribute Routing — Route Attribute

Bài 3 · Attribute Routing — Route Attribute

Web API hầu như chỉ dùng attribute routing. Conventional routing (pattern {controller}/{action}) ít phù hợp vì API cần URL có ngữ nghĩa rõ ràng và kiểm soát chính xác hơn.

Cách dùng [Route] trên controller và action

Controllers/DepartmentsController.cscsharp
[ApiController]
[Route("departments")]           // route gốc: /departments
public class DepartmentsController : ControllerBase
{
    // GET /departments
    [HttpGet]
    public ActionResult<IEnumerable<Department>> GetAll() { ... }

    // GET /departments/active  — route thêm vào phía sau route gốc
    [HttpGet]
    [Route("active")]
    public ActionResult<IEnumerable<Department>> GetActive() { ... }

    // GET /overridden  — dấu "/" đầu tiên OVERRIDE hoàn toàn route gốc
    [HttpGet]
    [Route("/overridden")]
    public ActionResult<string> AbsoluteRoute() => Ok("absolute");
}

Quy tắc kết hợp route

Route trên controllerRoute trên actionURL kết quả
[Route("departments")](không có)/departments
[Route("departments")][Route("active")]/departments/active
[Route("departments")][Route("/overridden")]/overridden

⚠️ Khi route trên action bắt đầu bằng /, nó sẽ ghi đè hoàn toàn route trên controller — không có sự kết hợp nào xảy ra.

Một action có thể có nhiều [Route]

Controllers/DepartmentsController.cscsharp
// Cùng một action nhưng có thể truy cập qua hai URL khác nhau
[HttpGet]
[Route("all")]
[Route("/api/departments")]
public ActionResult<IEnumerable<Department>> GetAll() { ... }

ℹ️ Tất cả tính năng của route template đã học (route parameters, constraints, optional parameters) đều áp dụng hoàn toàn trong attribute routing: [Route("{id:int}")], [Route("{name?}")]…

4 Attribute Routing — Token Replacement

Bài 4 · Attribute Routing — Token Replacement

Token replacement là cơ chế cho phép dùng các token đặc biệt trong route template để tự động lấy tên controller hoặc action, tránh hard-code chuỗi dễ sai khi đổi tên class.

Hai token chính

TokenĐược thay bằngVí dụ
[controller] Tên controller bỏ suffix "Controller" DepartmentsController → departments
[action] Tên action method Method GetAll → getall (case-insensitive)

ℹ️ Token replacement không phân biệt hoa thường — [Controller], [controller], [CONTROLLER] đều như nhau.

Cách dùng token

Controllers/DepartmentsController.cscsharp
[ApiController]
[Route("api/[controller]")]   // → "api/departments"
public class DepartmentsController : ControllerBase
{
    // URL: GET /api/departments
    [HttpGet]
    public ActionResult<IEnumerable<Department>> GetAll() { ... }

    // URL: GET /api/departments/42
    [HttpGet("{id:int}")]
    public ActionResult<Department> GetById(int id) { ... }

    // Token [action] → method name "Search"
    // URL: GET /api/departments/search?name=HR
    [HttpGet("[action]")]
    public ActionResult<IEnumerable<Department>> Search(string name) { ... }
}

Kết hợp token với chuỗi cố định

Ví dụ kết hợpcsharp
// Token [controller] + chuỗi cố định "api/" + parameter {city}
[Route("api/[controller]/{city}")]
public class WeatherForecastController : ControllerBase
{
    // URL: GET /api/weatherforecast/hanoi
    [HttpGet]
    public ActionResult<WeatherForecast> Get(string city) { ... }
}

💡 Dùng [Route("api/[controller]")] làm route chuẩn cho Web API — prefix api/ giúp phân biệt endpoint API với các trang web thông thường trong cùng project.

5 Attribute Routing — HttpMethod Attribute

Bài 5 · Attribute Routing — HttpMethod attribute

Để một route hoạt động, cần xác định hai yếu tố: route template (URL) và HTTP method. Các attribute [HttpGet], [HttpPost], [HttpPut], [HttpDelete], [HttpPatch] vừa khai báo HTTP method vừa có thể mang route template.

Lý do cần HttpMethod attribute

Xét hai endpoint cùng URL /departments nhưng khác nhau về HTTP method:

Controllers/DepartmentsController.cscsharp
[ApiController]
[Route("api/[controller]")]
public class DepartmentsController : ControllerBase
{
    // GET /api/departments  — lấy danh sách
    [HttpGet]
    public ActionResult<IEnumerable<Department>> GetAll() { ... }

    // POST /api/departments  — tạo mới
    // Cùng route template nhưng khác HTTP method → không conflict
    [HttpPost]
    public ActionResult<Department> Create(Department department) { ... }
}

⚠️ Nếu hai action có cùng route template và cùng HTTP method mà không có gì phân biệt thêm, sẽ xảy ra route conflict — framework báo lỗi khi khởi động hoặc khi gọi endpoint.

Dùng HttpMethod attribute như Route attribute

HttpMethod attribute có thể nhận route template trực tiếp, không cần dùng thêm [Route]:

Controllers/DepartmentsController.cscsharp
[ApiController]
[Route("api/[controller]")]
public class DepartmentsController : ControllerBase
{
    // GET /api/departments
    [HttpGet]
    public ActionResult<IEnumerable<Department>> GetAll() { ... }

    // GET /api/departments/42
    [HttpGet("{id:int}")]
    public ActionResult<Department> GetById(int id) { ... }

    // POST /api/departments
    [HttpPost]
    public ActionResult<Department> Create(Department department) { ... }

    // PUT /api/departments/42
    [HttpPut("{id:int}")]
    public ActionResult<Department> Update(int id, Department department) { ... }

    // DELETE /api/departments/42
    [HttpDelete("{id:int}")]
    public IActionResult Delete(int id) { ... }
}

Kết hợp token trong HttpMethod attribute

Ví dụ token trong HttpGetcsharp
// Token [controller] dùng được bên trong HttpMethod attribute
// URL: GET /api/departments/active
[HttpGet("api/[controller]/active")]
public ActionResult<IEnumerable<Department>> GetActive() { ... }

💡 Quy ước phổ biến: đặt route gốc trên controller bằng [Route("api/[controller]")], rồi chỉ thêm phần phân biệt vào [HttpGet(...)], [HttpPost(...)] trên action. Giữ code gọn và dễ đọc.

6 ApiController Attribute

Bài 6 · ApiController Attribute

[ApiController] là một attribute đặc biệt áp dụng một tập hợp convention (quy ước mặc định) phù hợp với Web API, giúp giảm code boilerplate và đảm bảo response nhất quán.

Ba convention chính của [ApiController]

1. Tự động validate model và trả về 400 Bad Request

Trong MVC/Razor Pages, bạn phải tự kiểm tra ModelState.IsValid. Với [ApiController], nếu validation thất bại, framework tự động trả về 400 với ProblemDetails — action method không được gọi.

Models/Department.cscsharp
public class Department
{
    public int Id { get; set; }

    [Required]
    [StringLength(100)]
    public string Name { get; set; } = string.Empty;

    [Required]
    public string Location { get; set; } = string.Empty;
}
Controllers/DepartmentsController.cscsharp
[ApiController]
[Route("api/[controller]")]
public class DepartmentsController : ControllerBase
{
    // Không cần kiểm tra ModelState.IsValid
    // Nếu Name hoặc Location bị thiếu → framework tự trả 400
    [HttpPost]
    public ActionResult<Department> Create(Department department)
    {
        // Code chỉ chạy khi model hợp lệ
        // ... lưu vào DB
        return CreatedAtAction(nameof(GetById), new { id = department.Id }, department);
    }
}

Response khi validation thất bại (ProblemDetails tự động):

HTTP 400 Responsejson
{
  "type": "https://tools.ietf.org/html/rfc9110#section-15.5.1",
  "title": "One or more validation errors occurred.",
  "status": 400,
  "errors": {
    "Name": ["The Name field is required."],
    "Location": ["The Location field is required."]
  }
}

2. Tự động wrap kết quả lỗi thành ProblemDetails

Khi action trả về các kết quả lỗi như NotFound(), BadRequest()… mà không kèm message, [ApiController] tự động tạo ProblemDetails với đầy đủ thông tin.

Controllers/DepartmentsController.cscsharp
[HttpGet("{id:int}")]
public ActionResult<Department> GetById(int id)
{
    var dept = _repository.GetById(id);
    if (dept is null)
        return NotFound();  // Tự động → ProblemDetails 404

    return dept;
}

3. Tự động áp dụng [FromBody] cho complex type

Không cần khai báo [FromBody] tường minh — khi parameter là complex type, [ApiController] tự suy ra binding từ request body.

Controllers/DepartmentsController.cscsharp
// Không cần [FromBody] trước Department — [ApiController] tự áp dụng
[HttpPost]
public ActionResult<Department> Create(Department department)
{
    // department được bind từ request body (JSON)
    ...
}

// Tương đương với:
// public ActionResult<Department> Create([FromBody] Department department)

Cấu hình tùy chỉnh hành vi [ApiController]

Có thể tắt từng convention nếu không muốn dùng:

Program.cscsharp
builder.Services.AddControllers()
    .ConfigureApiBehaviorOptions(options =>
    {
        // Tắt tự động validate ModelState (tự kiểm tra trong action)
        options.SuppressModelStateInvalidFilter = false;

        // Tắt tự động wrap lỗi thành ProblemDetails
        options.SuppressMapClientErrors = false;

        // Tắt tự động suy [FromBody] cho complex type
        options.SuppressInferBindingSourcesForParameters = false;
    });

🎯Luôn dùng [ApiController] khi viết Web API với controller. Attribute này giúp API nhất quán, an toàn hơn và giảm đáng kể code lặp lại.

7 Content Negotiation

Bài 7 · Content Negotiation

Content negotiation là quá trình server và client "thương lượng" về định dạng dữ liệu trong response. Client khai báo định dạng mong muốn qua HTTP header Accept, server dựa vào danh sách Output Formatter để quyết định định dạng trả về.

Cơ chế hoạt động

  1. Client gửi request kèm header Accept: application/xml.
  2. Server duyệt danh sách Output Formatter theo thứ tự đã đăng ký.
  3. Nếu tìm thấy formatter hỗ trợ media type đó → dùng formatter đó.
  4. Nếu không tìm thấy → fallback về formatter đầu tiên (mặc định là JSON).
Mặc định — chỉ có JSON formattertext
Formatter list: [JsonOutputFormatter]

Accept: application/json  → dùng JsonOutputFormatter  ✓
Accept: application/xml   → không tìm thấy → fallback → JsonOutputFormatter
Accept: text/plain        → không tìm thấy → fallback → JsonOutputFormatter

Thêm XML formatter

Program.cscsharp
builder.Services.AddControllers()
    // Thêm XML Serializer formatter vào danh sách
    .AddXmlSerializerFormatters();

// Sau khi thêm, formatter list: [JsonOutputFormatter, XmlOutputFormatter]
// Accept: application/xml  → dùng XmlOutputFormatter  ✓
// Accept: application/json → dùng JsonOutputFormatter ✓

Thứ tự ưu tiên khi client chấp nhận nhiều định dạng

Client có thể khai báo nhiều media type kèm quality value (q) để chỉ mức ưu tiên:

HTTP Request Headertext
# q mặc định = 1.0 nếu không khai báo; giá trị càng cao ưu tiên càng cao
Accept: application/xml;q=1.0, application/json;q=0.5
# → Server ưu tiên XML (q=1.0) hơn JSON (q=0.5)

Accept: application/json;q=1.0, application/xml;q=0.5
# → Server ưu tiên JSON (q=1.0) hơn XML (q=0.5)

Tạo custom Output Formatter

Có thể tạo formatter tùy chỉnh bằng cách kế thừa TextOutputFormatter:

CustomCsvOutputFormatter.cscsharp
public class CustomCsvOutputFormatter : TextOutputFormatter
{
    public CustomCsvOutputFormatter()
    {
        SupportedMediaTypes.Add("text/csv");
        SupportedEncodings.Add(Encoding.UTF8);
    }

    public override bool CanWriteType(Type? type)
        => type?.IsGenericType == true
           && type.GetGenericTypeDefinition() == typeof(IEnumerable<>);

    public override async Task WriteResponseBodyAsync(
        OutputFormatterWriteContext context,
        Encoding selectedEncoding)
    {
        var response = context.HttpContext.Response;
        var data = context.Object as IEnumerable<object>;
        // ... ghi CSV vào response
        await response.WriteAsync("csv content here", selectedEncoding);
    }
}
Program.cs — đăng ký custom formattercsharp
builder.Services.AddControllers(options =>
{
    // Chèn vào đầu danh sách → ưu tiên cao nhất
    options.OutputFormatters.Insert(0, new CustomCsvOutputFormatter());
});

ℹ️ Trong thực tế, hầu hết Web API chỉ cần JSON (mặc định). Thêm XML formatter khi cần tương thích với hệ thống cũ (legacy SOAP/XML). Custom formatter phù hợp khi tích hợp với client yêu cầu định dạng đặc biệt (CSV, vCard…).

8 Minimal API vs Web API với Controllers

Bài 8 · Minimal API vs Web API with controllers

ASP.NET Core .NET 9 hỗ trợ hai cách xây dựng API. Không có lựa chọn nào "tốt hơn tuyệt đối" — quyết định phụ thuộc vào yêu cầu dự án và sở thích team.

Bảng so sánh chi tiết

Tiêu chí Minimal API Web API với Controllers
Cú pháp Ngắn gọn, inline trong Program.cs hoặc tách file Class ControllerBase, method riêng biệt
Filter pipeline Chỉ có endpoint filter (AddEndpointFilter), không đầy đủ Đầy đủ: Authorization → Resource → Action → Exception → Result filter
Model validation Cần cài thêm MiniValidation hoặc package ngoài Tích hợp sẵn qua [ApiController], tự động 400
Content Negotiation Không hỗ trợ — chỉ JSON Hỗ trợ đầy đủ, cấu hình output formatter linh hoạt
Hiệu năng Nhỉnh hơn (ít layer hơn) Rất tốt, nhưng qua nhiều layer hơn một chút
Tổ chức code Dễ bị lộn xộn khi nhiều endpoint Cấu trúc rõ ràng, dễ bảo trì khi project lớn
Attribute Routing Không cần — URL khai báo trực tiếp Bắt buộc — dùng [Route], [HttpGet]…
Phù hợp với Microservice nhỏ, prototype, API đơn giản API phức tạp, team lớn, nhiều logic tầng filter

Ví dụ cùng chức năng — hai cách viết

Program.cs — Minimal APIcsharp
var builder = WebApplication.CreateBuilder(args);
var app = builder.Build();

// GET /api/departments
app.MapGet("/api/departments", (IDepartmentService svc) =>
    Results.Ok(svc.GetAll()));

// GET /api/departments/{id}
app.MapGet("/api/departments/{id:int}", (int id, IDepartmentService svc) =>
{
    var dept = svc.GetById(id);
    return dept is null ? Results.NotFound() : Results.Ok(dept);
});

// POST /api/departments
app.MapPost("/api/departments", (Department dept, IDepartmentService svc) =>
{
    svc.Add(dept);
    return Results.Created($"/api/departments/{dept.Id}", dept);
});

app.Run();
Controllers/DepartmentsController.cs — Controller APIcsharp
[ApiController]
[Route("api/[controller]")]
public class DepartmentsController : ControllerBase
{
    private readonly IDepartmentService _svc;

    public DepartmentsController(IDepartmentService svc) => _svc = svc;

    // GET /api/departments
    [HttpGet]
    public ActionResult<IEnumerable<Department>> GetAll()
        => Ok(_svc.GetAll());

    // GET /api/departments/{id}
    [HttpGet("{id:int}")]
    public ActionResult<Department> GetById(int id)
    {
        var dept = _svc.GetById(id);
        return dept is null ? NotFound() : Ok(dept);
    }

    // POST /api/departments
    [HttpPost]
    public ActionResult<Department> Create(Department dept)
    {
        _svc.Add(dept);
        return CreatedAtAction(nameof(GetById), new { id = dept.Id }, dept);
    }
}

🎯Gợi ý lựa chọn: Dùng Minimal API cho microservice nhỏ hoặc khi team quen functional style. Dùng Controller API khi project lớn, cần filter pipeline phức tạp, content negotiation, hoặc khi team đã quen với MVC pattern. Hai cách có thể cùng tồn tại trong một project ASP.NET Core.
