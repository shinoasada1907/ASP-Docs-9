CHƯƠNG 16

Tag Helpers

Tag Helpers trong Razor — sinh URL tự động, sửa thuộc tính form/input/option, kích hoạt validation phía client, thêm version number cho tài nguyên tĩnh, và tải Partial View cùng View Component theo cú pháp HTML thuần.

1 Tag Helpers là gì

Bài 1 · What are Tag Helpers

Tag Helper là các thành phần phía server cho phép chúng ta thêm các thuộc tính tùy chỉnh vào phần tử HTML trong file Razor (.cshtml). Khi Razor Engine xử lý view, nó nhận ra các thuộc tính đặc biệt này (ví dụ asp-controller, asp-for…) và tự động sinh ra HTML tương ứng dựa trên dữ liệu của ứng dụng.

Tại sao cần Tag Helpers?

Khi viết HTML thuần, chúng ta thường phải tự điền các URL, tên field, kiểu dữ liệu… bằng tay. Điều này dễ gây lỗi khi route hoặc tên property thay đổi. Tag Helpers giải quyết điều này bằng cách kéo thông tin từ server (controller, action, model class…) để sinh HTML chính xác.

Vấn đề với HTML thuầnGiải pháp với Tag Helper
Viết cứng URL trong hrefasp-controller + asp-action tự sinh URL
Tự điền name, id, type cho <input>asp-for đọc từ property của model
Tự viết anti-forgery tokenForm Tag Helper tự thêm hidden token
Tự thêm query string version cho ảnh/JSasp-append-version tự tính hash

Cách hoạt động

Tag Helper thực chất là một thuộc tính HTML — nhưng không phải thuộc tính HTML chuẩn, mà là thuộc tính mà Razor Engine hiểu. Ví dụ:

Views/Home/Index.cshtmlrazor
<!-- Razor (Tag Helper) -->
<a asp-controller="Home" asp-action="Index">Trang chủ</a>

<!-- HTML được sinh ra -->
<a href="/Home/Index">Trang chủ</a>

Thuộc tính asp-controller không phải HTML chuẩn — đây là Tag Helper. Razor đọc nó, tra cứu route tương ứng với controller Home và action Index, rồi sinh ra href đúng.

Kích hoạt Tag Helpers

Để dùng Tag Helpers trong tất cả view của một vùng, thêm dòng sau vào file _ViewImports.cshtml:

Views/_ViewImports.cshtmlrazor
@addTagHelper *, Microsoft.AspNetCore.Mvc.TagHelpers

ℹ️ Cú pháp @addTagHelper *, AssemblyName có nghĩa là: đăng ký tất cả Tag Helper (*) trong assembly chỉ định. Template ASP.NET Core MVC tự thêm dòng này vào _ViewImports.cshtml nên thường đã có sẵn.

2 Sinh URL với Tag Helpers

Bài 2 · Generating URL with Tag Helpers

Tag Helpers phổ biến nhất dùng để sinh URL là nhóm thuộc tính trên thẻ <a>. Một URL trong ASP.NET Core MVC thường có cấu trúc:

/{controller}/{action}/{id}?queryKey=value

Ba phần tương ứng với ba loại thuộc tính Tag Helper:

Phần URLTag HelperVí dụ
Controllerasp-controllerasp-controller="Employee"
Actionasp-actionasp-action="Edit"
Route parameterasp-route-{name}asp-route-id="@Model.Id"
Query stringasp-route-{key}asp-route-page="2"

Ví dụ thực tế

Views/Employee/Index.cshtmlrazor
@model IEnumerable<Employee>

<table>
@foreach (var emp in Model)
{
    <tr>
        <td>@emp.Name</td>
        <td>
            <!-- Sinh URL: /Employee/Edit/5 -->
            <a asp-controller="Employee"
               asp-action="Edit"
               asp-route-id="@emp.Id">Sửa</a>

            <!-- Sinh URL: /Employee/Details/5 -->
            <a asp-controller="Employee"
               asp-action="Details"
               asp-route-id="@emp.Id">Chi tiết</a>
        </td>
    </tr>
}
</table>

<!-- Link không có route param: /Employee/Create -->
<a asp-controller="Employee" asp-action="Create"
   class="btn btn-primary">Thêm mới</a>

HTML được sinh ra tự động đúng theo route table — nếu sau này bạn thay đổi route template, chỉ cần cập nhật một nơi, tất cả link tự cập nhật theo.

💡 Khi dùng Razor Pages thay vì MVC, thay asp-controller/asp-action bằng asp-page (tên Razor Page) và asp-page-handler (tên handler method). Ví dụ: asp-page="/Employees/Edit" asp-route-id="@emp.Id".

So sánh: viết tay vs Tag Helper

Trước (viết tay)html
<a href="/Employee/Edit/@emp.Id">Sửa</a>
Sau (Tag Helper)razor
<a asp-controller="Employee" asp-action="Edit" asp-route-id="@emp.Id">Sửa</a>

Với Tag Helper, nếu route thay đổi (ví dụ thêm tiền tố /app), URL sinh ra vẫn luôn đúng vì nó được tính từ route table chứ không phải chuỗi cứng.

3 Sửa thẻ <form> với Tag Helpers

Bài 3 · Modify form tag with Tag Helpers

Form Tag Helper hoạt động tương tự Anchor Tag Helper nhưng áp dụng lên thẻ <form>. Nó có hai chức năng chính:

  1. Sinh thuộc tính action (URL đích) dựa trên controller/action hoặc Razor Page.
  2. Tự động chèn anti-forgery hidden token vào bên trong form để chống tấn công CSRF.

Cú pháp — MVC Controller

Views/Employee/Edit.cshtmlrazor
@model Employee

<!-- Form Tag Helper cho MVC -->
<form asp-controller="Employee"
      asp-action="Edit"
      asp-route-id="@Model.Id"
      method="post">

    <!-- Các input bên trong -->
    <input asp-for="Name" class="form-control" />

    <button type="submit">Lưu</button>
</form>

<!-- HTML được sinh ra: -->
<!--
<form action="/Employee/Edit/5" method="post">
    <input type="text" id="Name" name="Name" value="John Doe" class="form-control" />
    <input name="__RequestVerificationToken" type="hidden" value="..." />
    <button type="submit">Lưu</button>
</form>
-->

Cú pháp — Razor Pages

Pages/Employees/Edit.cshtmlrazor
@model EditModel

<!-- asp-page: tên Razor Page; asp-page-handler: tên method xử lý -->
<form asp-page="/Employees/Edit"
      asp-page-handler="Save"
      asp-route-id="@Model.Employee.Id"
      method="post">

    <input asp-for="Employee.Name" class="form-control" />
    <button type="submit">Lưu</button>
</form>

<!-- Form xóa (gọi handler Delete) -->
<form asp-page="/Employees/Edit"
      asp-page-handler="Delete"
      asp-route-id="@Model.Employee.Id"
      method="post">
    <button type="submit" class="btn btn-danger">Xóa</button>
</form>

🔐Anti-forgery token tự động: Khi dùng Form Tag Helper, ASP.NET Core tự chèn <input type="hidden" name="__RequestVerificationToken" /> vào bên trong form. Bạn không cần gọi @Html.AntiForgeryToken() nữa. Nếu dùng MVC với [ValidateAntiForgeryToken] trên action, token này sẽ được kiểm tra tự động.

⚠️ Khi chuyển từ HTML thủ công sang Form Tag Helper, hãy xóa dòng @Html.AntiForgeryToken() (nếu có) để tránh token bị thêm hai lần trong form.

4 Sửa thẻ <input> với Tag Helpers

Bài 4 · Modify Input element with Tag Helpers

Input Tag Helper dùng thuộc tính asp-for để liên kết một <input> với một property của model. Chỉ cần một thuộc tính này, Tag Helper tự sinh ra đầy đủ type, name, id và value.

Quy tắc sinh thuộc tính từ asp-for

Thuộc tính HTMLNguồn gốc
nameTên property (dùng cho model binding khi post)
idTên property (dùng để label liên kết)
typeSuy ra từ kiểu dữ liệu C# hoặc Data Annotation (string → text, int → number, [EmailAddress] → email…)
valueGiá trị hiện tại của property trong model
data-val-*Attributes phục vụ validation phía client (xem Bài 6)

Ví dụ — Form nhập thông tin nhân viên

Models/Employee.cscsharp
public class Employee
{
    public int Id { get; set; }

    [Required, MaxLength(100)]
    public string Name { get; set; } = string.Empty;

    [Range(0, 1_000_000)]
    public decimal Salary { get; set; }

    [EmailAddress]
    public string? Email { get; set; }

    public int DepartmentId { get; set; }
}
Views/Employee/_EmployeeDetail.cshtmlrazor
@model Employee

<!-- asp-for="Name" sinh ra:
     type="text", name="Name", id="Name", value="@Model.Name"
     + data-val-required, data-val-maxlength -->
<div class="mb-3">
    <label asp-for="Name" class="form-label"></label>
    <input asp-for="Name" class="form-control" />
    <span asp-validation-for="Name" class="text-danger"></span>
</div>

<!-- asp-for="Salary" sinh type="number" vì kiểu decimal -->
<div class="mb-3">
    <label asp-for="Salary" class="form-label"></label>
    <input asp-for="Salary" class="form-control" />
    <span asp-validation-for="Salary" class="text-danger"></span>
</div>

<!-- asp-for="Email" sinh type="email" nhờ [EmailAddress] -->
<div class="mb-3">
    <label asp-for="Email" class="form-label"></label>
    <input asp-for="Email" class="form-control" />
    <span asp-validation-for="Email" class="text-danger"></span>
</div>

💡 Label Tag Helper (<label asp-for="Name">) cũng tự điền thuộc tính for="Name" và lấy tên hiển thị từ [Display(Name="...")] nếu có. Kết hợp Label + Input + ValidationMessage Tag Helpers cho một trải nghiệm form chuẩn và nhất quán.

So sánh: viết tay vs Tag Helper

Trước (viết tay)html
<input type="text"
       name="Name"
       id="Name"
       value="@Model.Name"
       class="form-control" />
Sau (Tag Helper)razor
<input asp-for="Name" class="form-control" />

Ngoài ra khi đổi tên property trong C# (ví dụ rename Name → FullName), Razor sẽ báo lỗi compile ngay thay vì để lỗi âm thầm ở runtime.

5 Sinh <option> với asp-items

Bài 5 · Generate Option elements with Tag Helpers

Để điền danh sách lựa chọn cho thẻ <select>, Tag Helper cung cấp thuộc tính asp-items. Nó nhận vào một IEnumerable<SelectListItem> và tự sinh toàn bộ các thẻ <option>.

Lớp SelectListItem

SelectListItem là class có sẵn trong Microsoft.AspNetCore.Mvc.Rendering với các property quan trọng:

PropertyÝ nghĩa
TextNội dung hiển thị của option
ValueGiá trị gửi lên server khi submit (thường là ID)
SelectedCó phải option đang được chọn không

Ví dụ — Chọn phòng ban cho nhân viên

Controllers/EmployeeController.cscsharp
public IActionResult Edit(int id)
{
    var employee = _repo.GetById(id);
    var departments = _repo.GetAllDepartments();

    // Chuyển danh sách Department thành IEnumerable<SelectListItem>
    ViewBag.DepartmentList = departments.Select(d => new SelectListItem
    {
        Text     = d.Name,
        Value    = d.Id.ToString(),
        Selected = d.Id == employee.DepartmentId
    });

    return View(employee);
}
Views/Employee/Edit.cshtmlrazor
@model Employee

<form asp-controller="Employee" asp-action="Edit"
      asp-route-id="@Model.Id" method="post">

    <div class="mb-3">
        <label asp-for="DepartmentId" class="form-label"></label>

        <!-- asp-for="DepartmentId" liên kết model binding
             asp-items    nhận danh sách option từ ViewBag -->
        <select asp-for="DepartmentId"
                asp-items="ViewBag.DepartmentList"
                class="form-select">
            <!-- Option mặc định "Chọn phòng ban" -->
            <option value="">-- Chọn phòng ban --</option>
        </select>

        <span asp-validation-for="DepartmentId" class="text-danger"></span>
    </div>

    <button type="submit" class="btn btn-primary">Lưu</button>
</form>

ℹ️ Khi bạn khai báo một <option> thủ công bên trong thẻ <select> có asp-items, Tag Helper không ghi đè option đó mà nối thêm sau nó. Nhờ vậy có thể thêm option "mặc định" (placeholder) dễ dàng.

Sử dụng SelectList để đơn giản hóa

Thay vì LINQ thủ công, có thể dùng constructor SelectList:

Controllercsharp
// SelectList(items, dataValueField, dataTextField, selectedValue)
ViewBag.DepartmentList = new SelectList(
    departments,
    nameof(Department.Id),
    nameof(Department.Name),
    employee.DepartmentId
);

6 Validation phía client

Bài 6 · Client side form validations

Khi dùng asp-for trên một <input>, Tag Helper tự động đọc các Data Annotations ([Required], [Range], [MaxLength]…) từ model và thêm vào input các thuộc tính data-val-*. Thư viện JavaScript jQuery Unobtrusive Validation đọc các thuộc tính này để kiểm tra dữ liệu ngay trên trình duyệt mà không cần gửi request lên server.

Các Tag Helpers liên quan đến validation

Tag HelperHTML sinh raMục đích
asp-for trên <input>data-val="true" + data-val-required, data-val-range…Gắn rules validation vào input
asp-validation-for trên <span><span data-valmsg-for="Name">Hiển thị thông báo lỗi cho từng field
asp-validation-summary trên <div><div data-valmsg-summary="true">Hiển thị tóm tắt tất cả lỗi

Cấu hình đầy đủ

Models/Employee.cscsharp
public class Employee
{
    public int Id { get; set; }

    [Required(ErrorMessage = "Tên không được để trống")]
    [MaxLength(100, ErrorMessage = "Tên tối đa 100 ký tự")]
    public string Name { get; set; } = string.Empty;

    [Range(0, 1_000_000, ErrorMessage = "Lương phải từ 0 đến 1.000.000")]
    public decimal Salary { get; set; }

    [Required]
    public int DepartmentId { get; set; }
}
Views/Employee/_EmployeeDetail.cshtmlrazor
@model Employee

<!-- Tóm tắt tất cả lỗi (hiện khi có lỗi server-side) -->
<div asp-validation-summary="ModelOnly" class="text-danger"></div>

<div class="mb-3">
    <label asp-for="Name" class="form-label"></label>
    <input asp-for="Name" class="form-control" />
    <!-- Hiện lỗi tức thì phía client cho field Name -->
    <span asp-validation-for="Name" class="text-danger"></span>
</div>

<div class="mb-3">
    <label asp-for="Salary" class="form-label"></label>
    <input asp-for="Salary" class="form-control" />
    <span asp-validation-for="Salary" class="text-danger"></span>
</div>

Bật thư viện JavaScript validation

Validation phía client cần hai thư viện JS. Thêm chúng vào layout hoặc partial view dành riêng:

Views/Shared/_ValidationScriptsPartial.cshtmlrazor
<script src="~/lib/jquery-validation/dist/jquery.validate.min.js"></script>
<script src="~/lib/jquery-validation-unobtrusive/jquery.validate.unobtrusive.min.js"></script>
Views/Employee/Edit.cshtmlrazor
@section Scripts {
    @await Html.PartialAsync("_ValidationScriptsPartial")
}

🔑Lưu ý quan trọng: Validation phía client chỉ là UX bổ sung — nó có thể bị bỏ qua. Luôn phải kiểm tra ModelState.IsValid trong action phía server trước khi xử lý dữ liệu.

ℹ️ Giá trị của asp-validation-summary có thể là: "None" (tắt), "ModelOnly" (chỉ lỗi model-level, không lặp lại lỗi field), hoặc "All" (hiện tất cả).

7 Thêm version number với asp-append-version

Bài 7 · Append Version Number with Tag Helpers

Trình duyệt thường cache các tài nguyên tĩnh (ảnh, CSS, JS) để tránh tải lại không cần thiết. Điều này gây ra vấn đề: khi bạn cập nhật file (ví dụ sửa app.js), trình duyệt vẫn dùng bản cũ từ cache vì tên file không đổi.

Giải pháp truyền thống — thêm query string thủ công

Thủ công (dễ quên cập nhật)html
<script src="/js/app.js?v=2"></script>
<link rel="stylesheet" href="/css/site.css?v=3" />

Giải pháp tốt hơn — asp-append-version

Tag Helper asp-append-version="true" tự động tính hash của nội dung file và thêm vào URL dưới dạng query string. Mỗi khi file thay đổi, hash thay đổi → browser buộc phải tải lại.

Views/Shared/_Layout.cshtmlrazor
<!-- CSS -->
<link rel="stylesheet"
      href="~/css/site.css"
      asp-append-version="true" />

<!-- JavaScript -->
<script src="~/js/app.js"
        asp-append-version="true"></script>

<!-- Hình ảnh -->
<img src="~/images/logo.png"
     asp-append-version="true"
     alt="Logo" />

<!-- HTML được sinh ra (hash thay đổi mỗi khi file đổi): -->
<!--
<link rel="stylesheet" href="/css/site.css?v=sha256-AbCdEf12..." />
<script src="/js/app.js?v=sha256-XyZ789..."></script>
<img src="/images/logo.png?v=sha256-MnOpQr..." alt="Logo" />
-->

Điều kiện để asp-append-version hoạt động

  1. File tài nguyên phải nằm trong thư mục wwwroot/.
  2. Middleware UseStaticFiles() phải được đăng ký trong Program.cs.
  3. Đường dẫn trong src/href phải bắt đầu bằng ~/ (ký hiệu wwwroot).
Program.cscsharp
var builder = WebApplication.CreateBuilder(args);
builder.Services.AddControllersWithViews();

var app = builder.Build();

// Bắt buộc phải có để phục vụ file tĩnh từ wwwroot
app.UseStaticFiles();

app.UseRouting();
app.MapControllerRoute(
    name: "default",
    pattern: "{controller=Home}/{action=Index}/{id?}");

app.Run();

💡 Hash được tính dựa trên nội dung file, không phải thời gian sửa đổi. Nếu hai file có nội dung giống nhau, chúng sẽ có cùng hash — đây là hành vi đúng vì browser không cần tải lại.

8 Tải Partial View và View Component

Bài 8 · Load Partial Views and ViewComponents with Tag Helpers

Trước đây chúng ta tải Partial View và View Component bằng HTML Helpers (@Html.Partial(), @await Component.InvokeAsync()). Tag Helpers cung cấp cú pháp HTML-like gọn hơn và quen thuộc hơn với designer.

Partial Tag Helper

Thay vì @await Html.PartialAsync("_EmployeeDetail", Model), dùng thẻ <partial>:

Views/Employee/Create.cshtmlrazor
@model Employee

<h2>Thêm nhân viên mới</h2>

<form asp-controller="Employee" asp-action="Create" method="post">

    <!-- Cách cũ (HTML Helper) -->
    @* @await Html.PartialAsync("_EmployeeDetail", Model) *@

    <!-- Cách mới (Tag Helper) -->
    <partial name="_EmployeeDetail" model="Model" />

    <button type="submit" class="btn btn-primary">Tạo mới</button>
</form>

<!-- Partial không có model (chỉ cần name) -->
<partial name="_SearchBar" />
Thuộc tínhBắt buộcMô tả
nameCóTên partial view (áp dụng quy tắc discovery giống HTML Helper)
modelKhôngObject truyền vào partial view; bỏ qua nếu partial không cần model
view-dataKhôngViewDataDictionary bổ sung truyền vào partial

View Component Tag Helper

View Component được gọi bằng thẻ có tiền tố vc:. Tên component dùng kebab-case (mỗi chữ hoa thêm dấu gạch ngang).

⚠️ Để dùng vc: Tag Helper, bạn phải thêm assembly của project vào _ViewImports.cshtml tương ứng:

Views/_ViewImports.cshtmlrazor
@using MyApp
@addTagHelper *, Microsoft.AspNetCore.Mvc.TagHelpers
@* Thêm assembly của project để dùng vc: Tag Helper *@
@addTagHelper *, MyApp
ViewComponents/EmployeeListViewComponent.cscsharp
public class EmployeeListViewComponent : ViewComponent
{
    private readonly IEmployeeRepository _repo;

    public EmployeeListViewComponent(IEmployeeRepository repo)
        => _repo = repo;

    public async Task<IViewComponentResult> InvokeAsync(int departmentId)
    {
        var employees = await _repo.GetByDepartmentAsync(departmentId);
        return View(employees);
    }
}
Pages/Departments/Edit.cshtml (Razor Page)razor
@model EditModel

<h2>@Model.Department.Name</h2>

<!-- Cách cũ (HTML Helper) -->
@* @await Component.InvokeAsync("EmployeeList", new { departmentId = Model.Department.Id }) *@

<!-- Cách mới (Tag Helper): EmployeeList -> employee-list -->
<vc:employee-list department-id="Model.Department.Id"></vc:employee-list>

<!-- View Component không có tham số -->
<vc:navigation-menu></vc:navigation-menu>

<!-- View Component với nhiều tham số -->
<vc:employee-list
    department-id="Model.Department.Id"
    page-size="10">
</vc:employee-list>

Quy tắc đặt tên vc: Tag Helper

Tên ViewComponent (C#)Tag Helper
EmployeeList<vc:employee-list>
NavigationMenu<vc:navigation-menu>
departmentId (param)department-id="..."
pageSize (param)page-size="..."

🎯Tại sao dùng Tag Helper thay vì HTML Helper? Tag Helpers có cú pháp HTML thuần — designer có thể đọc và chỉnh sửa view mà không cần hiểu C#. IntelliSense trong Visual Studio cũng hoạt động tốt hơn với cú pháp Tag Helper. Tuy nhiên cả hai đều hợp lệ — bạn có thể trộn lẫn trong cùng một project.
